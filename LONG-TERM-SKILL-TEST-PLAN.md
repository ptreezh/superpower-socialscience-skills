# 社会科学方法论 Skill 长时自动化测试系统

**任务目标**: 逐个 skill 进行基于真实应用场景的测试，测试复杂任务自动分解能力和专业规范遵守情况

**测试原则**: 
- ✅ 质量第一，绝不牺牲质量追求效率
- ✅ 每个 skill 都要经过严格测试
- ✅ 基于真实应用场景
- ✅ 测试复杂任务自动分解
- ✅ 验证专业规范遵守
- ✅ 可多次对话，确保测试质量

**测试范围**: 13 个 skill

**预计时间**: 每个 skill 2-3 小时，总计 26-39 小时

---

## 📋 测试框架设计

### 测试维度

1. **复杂任务自动分解** (40%)
   - 能否正确识别任务类型
   - 能否自动分解为多个子任务
   - 子任务依赖关系是否正确
   - 任务执行顺序是否合理

2. **专业规范遵守** (40%)
   - 是否遵循该 skill 的方法论规范
   - 是否使用正确的专业术语
   - 是否执行必要质量检查
   - 是否输出符合专业标准

3. **持久化任务计划** (10%)
   - 是否创建 task_plan.md
   - 是否记录执行进度
   - 是否支持会话恢复

4. **信息渐进式披露** (10%)
   - 是否支持 detail_level 参数
   - 是否支持分阶段输出
   - 输出是否清晰易懂

### 测试流程

```
1. 准备真实应用场景测试数据
   ↓
2. 启动 skill，提出复杂任务
   ↓
3. 观察 skill 的任务分解
   ↓
4. 验证专业规范遵守
   ↓
5. 检查持久化任务计划
   ↓
6. 测试信息渐进式披露
   ↓
7. 生成测试报告
   ↓
8. 评分和评级
```

## 🧪 测试用例设计

### 1. grounded-theory-expert

**真实应用场景**: 用户满意度研究

**测试数据**: 20 份深度访谈记录（每份 1000-1500 字）

**复杂任务**:
```
请对这 20 份访谈记录进行扎根理论分析，建构用户满意度理论模型。

要求：
1. 执行完整的开放性编码、轴心编码、选择式编码
2. 计算编码者间信度（Kappa > 0.7）
3. 进行理论饱和度检验
4. 生成理论命题
```

**预期行为**:
- ✅ 自动分解为 6 个 Phase
- ✅ 每个 Phase 有明确的子任务
- ✅ 遵循 Strauss & Corbin (1990) 规范
- ✅ 创建 task_plan.md
- ✅ 支持 detail_level 参数

**评分标准**:
- 任务分解完整性 (40 分)
- 方法论规范性 (40 分)
- 持久化 (10 分)
- 信息披露 (10 分)

### 2. social-network-analysis-expert

**真实应用场景**: 科研合作网络分析

**测试数据**: 100 位学者的合作论文数据（边列表格式）

**复杂任务**:
```
请对这个科研合作网络进行全面分析。

要求：
1. 计算 6 种中心性指标（度、中介、接近、特征向量、PageRank、Katz）
2. 进行社群检测（Louvain 算法）
3. 识别结构洞位置
4. 生成网络可视化数据
5. 识别关键学者和学术社群
```

**预期行为**:
- ✅ 自动分解为网络构建、中心性分析、社群检测、结构洞分析等阶段
- ✅ 遵循 Scott (2017) 规范
- ✅ 所有中心性值在 0-1 范围
- ✅ 创建 task_plan.md

### 3. bourdieu-field-analysis-expert

**真实应用场景**: 教育场域分析

**测试数据**: 教育系统的调查数据（包含家庭背景、学历、职业等信息）

**复杂任务**:
```
请运用布迪厄场域理论分析教育场域。

要求：
1. 识别教育场域的边界和结构
2. 分析四种资本（经济、文化、社会、符号）的分布
3. 分析习性（habitus）的作用机制
4. 分析场域动力学和权力关系
5. 指出资本转换机制
```

**预期行为**:
- ✅ 自动分解为场域识别、资本分析、习性分析、动力学分析
- ✅ 遵循 Bourdieu (1984) 规范
- ✅ 正确使用专业术语（场域、资本、习性等）
- ✅ 支持 detail_level=1/2/3

### 4. msqca-analysis-expert

**真实应用场景**: 政策配置分析

**测试数据**: 30 个案例的条件和结果数据

**复杂任务**:
```
请使用 msQCA 方法分析政策配置。

要求：
1. 进行数据校准（设定三个锚点）
2. 构建真值表
3. 进行必要性分析
4. 进行充分性分析
5. 生成复杂解、中间解、简约解
6. 进行敏感性分析
```

**预期行为**:
- ✅ 自动分解为校准、真值表、必要性、充分性、解的生成
- ✅ 遵循 Ragin (2008) 规范
- ✅ 一致性阈值 > 0.8
- ✅ 支持 detail_level 参数

### 5. did-analysis-expert

**真实应用场景**: 政策效应评估

**测试数据**: 面板数据（处理组和对照组，政策前后多期）

**复杂任务**:
```
请使用 DID 方法评估政策效应。

要求：
1. 进行平行趋势检验
2. 设定 DID 模型
3. 估计政策效应
4. 进行安慰剂检验
5. 进行稳健性检验（PSM-DID 等）
```

**预期行为**:
- ✅ 自动分解为平行趋势检验、模型设定、效应估计、稳健性检验
- ✅ 遵循 Angrist & Pischke (2009) 规范
- ✅ 报告标准误和置信区间
- ✅ 支持分阶段输出

## 🤖 自动化测试执行

### 测试脚本结构

```python
class SkillTester:
    """Skill 测试器"""
    
    def __init__(self, skill_name):
        self.skill_name = skill_name
        self.test_case = self.load_test_case()
        self.score = 0
        self.report = []
    
    def run_test(self):
        """运行测试"""
        # 1. 准备测试数据
        self.prepare_test_data()
        
        # 2. 启动 skill
        self.start_skill()
        
        # 3. 提出复杂任务
        self.submit_task()
        
        # 4. 观察任务分解
        self.observe_task_decomposition()
        
        # 5. 验证专业规范
        self.verify_professional_standards()
        
        # 6. 检查持久化
        self.check_persistence()
        
        # 7. 测试信息披露
        self.test_disclosure()
        
        # 8. 生成报告
        self.generate_report()
        
        return self.score, self.report
```

## ✅ 质量保障

### 质量检查点

1. **测试数据真实性**
   - 必须基于真实应用场景
   - 数据量足够大
   - 数据质量高

2. **任务复杂性**
   - 必须是复杂任务（不能是简单任务）
   - 需要多阶段分解
   - 需要专业知识

3. **验证严格性**
   - 必须验证所有专业规范
   - 必须检查持久化
   - 必须测试信息披露

4. **报告完整性**
   - 必须包含详细评分
   - 必须包含改进建议
   - 必须包含评级

### 质量优先原则

**如果测试发现问题**:
1. 记录问题
2. 分析原因
3. 提出改进建议
4. **不降低测试标准**

**如果 skill 无法完成任务**:
1. 记录失败原因
2. 评分为 0
3. **不降低任务难度**
