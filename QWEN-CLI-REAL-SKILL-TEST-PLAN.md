# Qwen CLI 内 Skill 真实环境测试计划

**测试目标**: 在真实 Qwen CLI 环境中测试 13 个 skill 的实际表现

**测试原则**: 
- ✅ 质量第一，绝不妥协
- ✅ 真实 CLI 环境测试
- ✅ 每个 skill 都要严格测试
- ✅ 可多次对话，确保质量
- ✅ 测试后 skill 必须能够独立使用

**测试范围**: 13 个 skill

**预计时间**: 每个 skill 1-2 小时对话测试 + 实际运行时间

---

## 🎯 测试核心内容

### 1. 复杂任务自动分解测试
- 给 skill 一个复杂任务
- 观察是否自动分解为多个子任务
- 子任务是否有合理的依赖关系
- 是否创建 task_plan.md

### 2. 专业规范遵守测试
- 是否遵循该 skill 的方法论规范
- 是否使用正确的专业术语
- 是否执行必要质量检查
- 输出是否符合专业标准

### 3. 进化机制测试
- 任务完成后是否记录教训到 lesson-memory.md
- 成功案例是否积累到 case-library/
- 每 10 次会话是否触发定期进化

### 4. CLI 任务计划集成测试
- 是否可以通过 `/task list` 查看任务
- 是否可以通过 `/task start` 启动任务
- 任务状态是否同步到 CLI 对话
- 会话重启后是否恢复任务

### 5. 信息渐进式披露测试
- 是否支持 detail_level=1/2/3
- 是否支持 phased_output
- 输出是否清晰易懂

## 📋 测试流程（每个 skill）

### 步骤 1: 部署 skill 到 Qwen CLI

```bash
# 复制 skill 到 Qwen CLI
cp -r {skill-name} ~/.qwen/skills/

# 验证部署
ls ~/.qwen/skills/{skill-name}/
# 应该看到：soul.md, lesson-memory.md, case-library/, tools/
```

### 步骤 2: 启动 Qwen CLI 并加载 skill

```
qwen

# 应该看到：
# [SkillEvolution] 进化引擎初始化...
# [SkillEvolution] 已加载 {skill-name}
# [SkillTaskIntegration] 任务集成扩展初始化...
```

### 步骤 3: 测试 skill 加载

```
你是 {skill-name} 吗？请介绍一下你的角色和能力。

# 预期：skill 应该根据 soul.md 介绍自己
```

### 步骤 4: 提出复杂任务（真实应用场景）

**示例（grounded-theory-expert）**:
```
我正在进行一项用户满意度研究，收集了 20 份深度访谈记录。

请你对这些访谈数据进行扎根理论分析，建构用户满意度理论模型。

要求：
1. 执行完整的开放性编码、轴心编码、选择式编码
2. 计算编码者间信度（Kappa > 0.7）
3. 进行理论饱和度检验
4. 生成理论命题

请自动分解这个任务，并创建任务计划。
```

### 步骤 5: 观察任务分解

**观察点**:
- ✅ 是否自动分解为多个 Phase
- ✅ 每个 Phase 是否有子任务
- ✅ 是否创建 task_plan.md
- ✅ 任务分解是否合理

### 步骤 6: 启动任务执行

```
/task start {skill-name} 1

# 或者直接说：开始执行 Phase 1
```

### 步骤 7: 观察执行过程

**观察点**:
- ✅ 是否遵循专业规范
- ✅ 是否使用正确术语
- ✅ 是否执行质量检查
- ✅ 输出是否专业

### 步骤 8: 测试信息渐进式披露

```
请用摘要模式（detail_level=1）输出当前结果。

请用标准模式（detail_level=2）输出。

请用详细模式（detail_level=3）输出。
```

### 步骤 9: 测试进化机制

```
查看教训记忆。

# 应该看到 lesson-memory.md 的内容

查看案例库。

# 应该看到 case-library/的内容
```

### 步骤 10: 测试任务管理

```
/task list
/task progress
/task show {skill-name}
```

### 步骤 11: 生成测试报告

记录：
- 任务分解是否成功
- 专业规范是否遵守
- 进化机制是否工作
- CLI 任务集成是否正常
- 信息披露是否正常

## 🧪 13 个 skill 的测试任务

### 1. grounded-theory-expert

**真实任务**: 用户满意度研究（20 份访谈）

**复杂任务描述**:
```
我正在进行一项用户满意度研究，收集了 20 份深度访谈记录（每份 1000-1500 字）。

请你对这些访谈数据进行扎根理论分析，建构用户满意度理论模型。

要求：
1. 执行完整的开放性编码、轴心编码、选择式编码
2. 计算编码者间信度（Cohen's Kappa > 0.7）
3. 进行理论饱和度检验（多维度）
4. 生成理论命题
5. 生成研究报告

请自动分解这个任务，创建任务计划，并执行分析。
```

**预期行为**:
- ✅ 分解为 6 个 Phase
- ✅ 遵循 Strauss & Corbin (1990) 规范
- ✅ 创建 task_plan.md
- ✅ 记录教训到 lesson-memory.md
- ✅ 支持 detail_level 参数

### 2. social-network-analysis-expert

**真实任务**: 科研合作网络分析（100 位学者）

**复杂任务描述**:
```
我有 100 位学者的科研合作数据（边列表格式，约 500 条合作记录）。

请你对这个科研合作网络进行全面分析。

要求：
1. 构建合作网络
2. 计算 6 种中心性指标（度、中介、接近、特征向量、PageRank、Katz）
3. 进行社群检测（Louvain 算法）
4. 识别结构洞位置（约束、有效规模）
5. 识别关键学者和学术社群
6. 生成网络可视化数据

请自动分解这个任务，创建任务计划，并执行分析。
```

**预期行为**:
- ✅ 分解为网络构建、中心性分析、社群检测、结构洞分析
- ✅ 遵循 Scott (2017) 规范
- ✅ 所有中心性值在 0-1 范围
- ✅ 创建 task_plan.md

### 3. bourdieu-field-analysis-expert

**真实任务**: 教育场域分析（调查数据）

**复杂任务描述**:
```
我有教育系统的调查数据，包含家庭背景、学历、职业、收入等信息（N=1000）。

请运用布迪厄场域理论分析教育场域。

要求：
1. 识别教育场域的边界和结构
2. 分析四种资本（经济、文化、社会、符号）的分布
3. 分析习性（habitus）的作用机制
4. 分析场域动力学和权力关系
5. 指出资本转换机制

请自动分解这个任务，创建任务计划，并执行分析。
```

**预期行为**:
- ✅ 分解为场域识别、资本分析、习性分析、动力学分析
- ✅ 遵循 Bourdieu (1984) 规范
- ✅ 正确使用专业术语

### 4. msqca-analysis-expert

**真实任务**: 政策配置分析（30 个案例）

**复杂任务描述**:
```
我有 30 个政策案例的数据，包含 5 个条件变量和 1 个结果变量。

请使用 msQCA 方法分析政策配置。

要求：
1. 进行数据校准（设定三个锚点：25%, 50%, 75%）
2. 构建真值表
3. 进行必要性分析
4. 进行充分性分析（一致性阈值 0.8）
5. 生成复杂解、中间解、简约解
6. 进行敏感性分析

请自动分解这个任务，创建任务计划，并执行分析。
```

**预期行为**:
- ✅ 分解为校准、真值表、必要性、充分性、解的生成
- ✅ 遵循 Ragin (2008) 规范

### 5. did-analysis-expert

**真实任务**: 政策效应评估（面板数据）

**复杂任务描述**:
```
我有面板数据，包含处理组和对照组在政策前后 5 期的数据（N=500, T=5）。

请使用 DID 方法评估政策效应。

要求：
1. 进行平行趋势检验
2. 设定 DID 模型
3. 估计政策效应
4. 进行安慰剂检验
5. 进行稳健性检验（PSM-DID 等）
6. 报告标准误和置信区间

请自动分解这个任务，创建任务计划，并执行分析。
```

**预期行为**:
- ✅ 分解为平行趋势检验、模型设定、效应估计、稳健性检验
- ✅ 遵循 Angrist & Pischke (2009) 规范

## ✅ 质量保障

### 质量检查点

1. **测试环境真实性**
   - 必须在真实 Qwen CLI 中测试
   - 不能是模拟环境

2. **任务复杂性**
   - 必须是复杂任务
   - 需要多阶段分解
   - 需要专业知识

3. **验证严格性**
   - 必须验证所有专业规范
   - 必须检查持久化
   - 必须测试信息披露

4. **报告完整性**
   - 必须包含详细评分
   - 必须包含改进建议
   - 必须包含评级

### 质量优先原则

**如果 skill 无法完成任务**:
1. 记录失败原因
2. 评分为实际得分（不人为提高）
3. 提出改进建议
4. **不降低测试标准**
