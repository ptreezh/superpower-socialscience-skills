## 📋 更新策略建议

### 方案 A: 保持现有结构，补充子页面

**优点**:
- 保持网站一致性
- 用户导航习惯不变
- SEO 友好

**缺点**:
- 需要创建多个子页面
- 工作量较大

**执行**:
```
/agent
  ├── /agent/theory (理论研究智能体)
  ├── /agent/digital (数字理论家)
  └── /agent/methodology (方法论智能体)

/methods
  ├── /methods/grounded-theory
  ├── /methods/sna
  └── ... (其他 10 种)

/tools
  ├── /tools/soul-agent-creator
  ├── /tools/autonomous-evolution
  └── ...
```

### 方案 B: 整合到现有页面

**优点**:
- 工作量小
- 快速上线

**缺点**:
- 页面可能过长
- 信息架构混乱

**执行**:
- 将所有方法论整合到 /methods 单页
- 将工具整合到 /tools 单页

### 方案 C: 重建信息架构（推荐）

**优点**:
- 清晰的信息架构
- 便于扩展
- 用户体验好

**缺点**:
- 需要重新设计
- URL 变更影响 SEO

**执行**:
```
/agents/ (所有智能体)
  ├── /agents/theory/ (理论研究)
  ├── /agents/digital/ (数字理论家)
  └── /agents/methods/ (方法论)

/methods/ (方法论详解)
  ├── /methods/grounded-theory/
  ├── /methods/sna/
  └── ...

/tools/ (工具)
  ├── /tools/soul-agent-creator/
  └── ...

/resources/ (资源)
  ├── /resources/guides/ (使用指南)
  ├── /resources/blog/ (博客)
  └── /resources/whitepaper/ (白皮书)
```
