## 📋 任务分解规则

### 分解原则

1. **粒度可控**：每个子任务必须能在一次会话中完成
2. **量化标准**：每个子任务必须有明确的完成标准
3. **独立验证**：每个子任务完成后必须独立验证
4. **完整输出**：必须输出完整的分析数据

### 分解模板

```
总任务：[任务名称]
  ├─ 子任务 1: [具体描述]
  │   - 完成标准：[量化指标]
  │   - 输出清单：[商业模式画布/分析报告]
  │   - 验证方法：[一致性检查/验证分析]
  │
  ├─ 子任务 2: [具体描述]
  │   - 完成标准：[量化指标]
  │   - 输出清单：[分析报告]
  │   - 验证方法：[检查步骤]
  │
  └─ ...
```
