## 成功经验

### 1. 螺旋式迭代的重要性
**经验**: 行动研究的核心在于螺旋式迭代，单轮循环往往不足以解决问题。
**证据**: 多个案例显示，3轮以上循环才能达到稳定效果。
**建议**: 默认规划3轮循环，每轮都要有明确的改进目标。

### 2. 利益相关者参与的深度
**经验**: 参与式行动研究中，利益相关者的深度参与是成功关键。
**证据**: 参与度>60%的项目成功率显著高于参与度<40%的项目。
**建议**: 在计划阶段就明确参与机制和决策流程。

### 3. 反思的批判性
**经验**: 反思不应只是描述性的，而应具有批判性。
**证据**: 批判性反思能够识别深层假设和结构性问题。
**建议**: 使用"为什么-如何-什么"三层次反思框架。

### 4. 数据三角验证
**经验**: 单一数据源容易产生偏误。
**证据**: 使用3种以上数据收集方法的研究更可信。
**建议**: 每轮循环至少使用访谈+观察+文档三种数据源。

## 失败教训

### 1. 过早进入行动阶段
**教训**: 跳过深入诊断直接行动往往导致干预无效。
**原因**: 问题理解不深入，干预措施不针对根本原因。
**预防**: 确保诊断阶段收集足够数据，识别真正的问题根源。

### 2. 忽视组织政治
**教训**: 行动研究发生在组织情境中，政治因素不可忽视。
**原因**: 未识别关键利益相关者的利益和权力关系。
**预防**: 在计划阶段进行利益相关者分析和权力映射。

### 3. 反思流于表面
**教训**: 描述性反思无法产生深度学习。
**原因**: 缺乏反思框架和时间投入。
**预防**: 使用结构化反思工具，预留充足反思时间。

### 4. 研究者角色混淆
**教训**: 研究者与参与者角色不清会导致伦理问题。
**原因**: 未明确角色边界和权力关系。
**预防**: 在项目开始时明确角色分工和决策机制。

## 模式库

### 模式1: 诊断-干预-评估循环
```
问题诊断 → 根因分析 → 干预设计 → 实施行动 → 效果评估 → 反思调整
```
**适用**: 组织改进、教学改进、流程优化

### 模式2: 参与式协作循环
```
共同界定问题 → 共同设计方案 → 共同实施行动 → 共同评估效果 → 共同提炼知识
```
**适用**: 社区发展、健康促进、社会变革

### 模式3: 实践者自我研究
```
自我观察 → 自我反思 → 自我调整 → 自我评估 → 知识外化
```
**适用**: 教师研究、临床实践、专业发展

## 方法论知识

### Kemmis & McTaggart 四阶段模型
1. **计划(Plan)**: 识别问题、收集基线数据、制定行动计划
2. **行动(Act)**: 实施干预、灵活调整、记录过程
3. **观察(Observe)**: 收集行动数据、多方法观察、监控变化
4. **反思(Reflect)**: 分析数据、评估效果、修改计划

### 行动研究质量标准
| 标准 | 描述 | 检验方法 |
|------|------|---------|
| 实践相关性 | 解决实际问题 | 问题解决程度 |
| 参与深度 | 利益相关者参与 | 参与程度评估 |
| 循环完整 | 多轮循环迭代 | 循环记录 |
| 反思深度 | 批判性反思 | 反思日志质量 |
| 知识贡献 | 理论/实践知识 | 可迁移性评估 |
