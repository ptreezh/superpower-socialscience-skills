## 二、禁止原则测试用例

### 2.1 禁止人为/非人二分法 测试

#### 测试场景: 智慧城市传感器网络

```yaml
测试数据: case_01_smart_city/
触发条件: "分析智慧城市项目中技术如何影响市民生活"

预期结果:
- 非人行动者识别: 传感器、数据平台、APP、算法 等
- 非人比例: ≥30%
- 对称性: 人类与非人分析深度相同
```

#### CLI 测试命令

```bash
# 测试1: 验证非人识别
> 请识别这个智慧城市案例中的所有行动者，特别注意非人行动者（技术、物体、数据等）

# 验证要点:
# ✓ 是否识别了传感器？
# ✓ 是否识别了数据平台？
# ✓ 是否识别了算法？
# ✓ 非人比例是否≥30%？

# 测试2: 对称性深度检查
> 请分析"城市数据平台"这个非人行动者，如何主动影响市民行为？它有哪些"代理能力"？

# 验证要点:
# ✓ 非人是否被描述为"主动"？
# ✓ 非人作用是否与人类同等分析？
```

#### 禁止模式触发测试

```bash
# ❌ 错误触发测试 (应该被阻止)
> 这个案例主要是人的问题，技术只是工具

# 预期: 技能应拒绝执行，提示违反对称性原则

# ✅ 正确触发
> 分析技术系统和人类行动者如何共同构建智慧城市网络
```

## 八、测试验收标准

### 8.1 禁止原则遵守

| 原则 | 测试通过标准 |
|------|-------------|
| 禁止人为/非人二分 | 非人比例≥30%，对称分析 |
| 禁止未追踪到底 | 完整追踪链，无中断 |
| 禁止预设边界 | 边界由行动者定义 |
| 禁止静态网络 | 4阶段| 禁止功能转译完整 |
主义 | 效果vs目的区分清晰 |
| 禁止黑箱化 | 所有黑箱被打开 |

### 8.2 任务分解执行

| 功能 | 测试通过标准 |
|------|-------------|
| 自动分解 | 大任务自动生成任务列表 |
| 任务队列 | 按顺序执行，可跳转 |
| 子Agent并行 | 批量任务并行处理 |
| 验证检查点 | 每任务完成后验证 |

### 8.3 持久化执行

| 功能 | 测试通过标准 |
|------|-------------|
| 会话恢复 | 中断后可恢复 |
| 项目持久化 | 跨会话保存进度 |
| 状态查看 | 随时查看当前状态 |
