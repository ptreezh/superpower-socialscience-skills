## 三、6大禁止原则测试

### 3.1 禁止原则清单

| # | 原则 | 测试要点 | 状态 |
|---|------|---------|------|
| 1 | 禁止人为/非人二分法 | 非人行动者≥30%，完全对称 | ⚠️ |

| 5 | 禁止功能主义解释 | 效果vs目的 | ⚠️ |
| 6 | 禁止黑箱化 | 所有黑箱打开 | ⚠️ |

### 3.2 原则详解 (来自 SKILL.md)

#### 原则1: 禁止人为/非人二分法

```
❌ 绝对禁止:
- 只关注人类行动者，忽略技术、物体、自然
- 将非人行动者视为"背景"或"环境"
- 优先分析人类，非人作为补充

✅ 正确做法:
- 人与非人行动者完全对称对待
- 技术、物体、自然与人类同等重要
- 追踪所有行动者的作用和影响
- 不预设哪类行动者更重要
- 非人行动者可以是积极代理人

📊 量化标准:
- 行动者类型覆盖: 人+非人都必须有
- 非人行动者比例: ≥30%
- 非人行动者分析深度: 与人类相同
```

#### 原则2: 禁止未追踪到底就停止

```
❌ 绝对禁止:
- 追踪到某个环节就停止
- 认为某个行动者"不重要"而不追踪
- 因为数据难获取就跳过某些链条
- 以"时间不足"为由缩短追踪
- 只看表面现象，不深入转译过程

✅ 正确做法:
- 追根溯源: 每个行动者都要追溯到其起源
- 顺藤摸瓜: 每个影响都要追踪到其后果
- 打开黑箱: 不接受"黑箱"作为终点
- 持续追踪: 跨越时间和空间的限制
- 每条转译链条都必须完整
```

#### 原则3: 禁止预设网络边界

```
❌ 绝对禁止:
- 事先定义研究范围（如"只看组织内部"）
- 排除"外部"行动者
- 认为某些行动者"不相关"
- 以学科或领域边界限制追踪

✅ 正确做法:
- 跟随行动者自己: 让行动者定义网络边界
- 开放边界: 追踪到哪里，网络就到哪里
- 包含所有相关者: 包括边缘、意外的行动者
- 跨学科追踪: 不受领域限制
- 接受网络的复杂性和异质性
```

#### 原则4: 禁止静态网络观

```
❌ 绝对禁止:
- 将网络视为固定的结构
- 忽略网络的持续构建过程
- 只看"结果"，不关注"过程"
- 认为网络一旦形成就稳定
- 用"状态"代替"过程"

✅ 正确做法:
- 网络即过程: 网络是持续转译的结果
- 动态追踪: 关注网络的变化和重组
- 过程优先: 过程比结果更重要
- 不稳定性: 网络永远不稳定
- 持续重构: 网络在不断重新定义
```

#### 原则5: 禁止功能主义解释

```
❌ 绝对禁止:
- 用"功能"、"目的"解释行动者行为
- 认为技术物有"预定功能"
- 用"效率"、"优化"作为解释框架
- 假设行动者有"意图"或"目标"
- 事后理性化（成功后有"目的"）

✅ 正确做法:
- 效果而非目的: 关注行动者的效果，而非预设目的
- 转译而非功能: 关注如何被转译，而非"功能"
- 过程而非结果: 关注如何达成，而非"为什么"
- 意外后果: 关注非预期的效果
- 情境化: 在具体情境中理解行动者
```

#### 原则6: 禁止黑箱化

```
❌ 绝对禁止:
- 接受"黑箱"作为分析终点
- 不打开技术、制度、理论的黑箱
- 认为"过于复杂"无法打开
- 用专业术语掩盖无知
- 接受专家的"权威解释"

✅ 正确做法:
- 打开黑箱: 每个黑箱都要打开
- 追踪内部: 黑箱内部的行动者也要追踪
- 重新组合: 打开后如何重新黑箱化
- 持续怀疑: 不接受"就是这样"
- 透明化: 让黑箱内部的转译可见
```
