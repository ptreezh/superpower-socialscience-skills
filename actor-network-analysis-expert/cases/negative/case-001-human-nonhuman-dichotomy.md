# 负面案例001: 人为/非人二分的错误

## 案例描述

研究者分析某在线教育平台的实施，声称使用ANT方法，但实际上只关注人类行动者。

---

## ❌ 错误做法

### 1. 行动者识别严重不对称

**研究者声称识别了所有行动者**：

**人类行动者**（25人，98%！）：
- 教师、学生、管理员、技术支持、家长...

**非人行动者**（1个，2%❌）：
- 平台软件（仅作为"工具"提及）

**对称性检查**：
- 非人行动者比例：2% （要求≥30%❌）
- 分析深度：人类行动者详细，非人行动者仅一句话

### 2. 非人行动者被简化为"工具"

**研究者如何"分析"平台软件**：

> "平台软件作为工具，支持教师和学生进行在线教学。"

**为什么错误**：
- 软件不是主动代理人，而是"工具"
- 软件如何塑造教学实践？未分析
- 软件如何约束或使能行为？未分析
- 软件如何与其他行动者互动？未分析

### 3. 研究结论严重偏差

**错误结论**：
> "在线教育的成功主要依赖于教师的适应性和学生的学习动机。平台软件只是一个工具，只要易用即可。"

**为什么错误**：
- 将成功归因于人类意图和动机
- 忽略了软件如何重新定义"教学"
- 忽略了算法如何塑造学习过程
- 忽略了界面设计如何影响互动

---

## ✅ 正确做法

### 1. 对称性识别行动者

**正确做法**：

**人类行动者**（15人，40%）：
- 教师、学生、管理员...

**非人行动者**（22个，60%✅）：
- 平台软件、视频会议工具、聊天功能、文件共享、白板、评分系统、学习分析算法、推荐引擎、通知系统、日历、成绩单、证书、隐私设置、用户协议、网络带宽、电脑、手机、麦克风、摄像头、家庭空间、学校制度、教育标准...

**对称性分析**：
- 平台软件：不是"工具"，而是重新定义教学的核心行动者
- 算法：不是"辅助"，而是塑造学习路径的主动力量
- 家庭空间：不是"背景"，而是影响学习参与的关键因素

### 2. 分析非人行动者的主动作用

**正确分析平台软件**：

> "平台软件不只是工具，而是教学实践的重构者。界面设计决定了哪些互动被鼓励（如实时聊天）和哪些被抑制（如深度讨论）。评分系统重新定义了'学习成功'的标准。学习分析算法将学生转化为数据点，将教学转化为数据优化。软件不是中立的，它有自己'意图'——设计者的意图+商业逻辑+技术约束的混合。"

### 3. 对称性分析

**正确结论**：
> "在线教育的成功不依赖于'教师+工具'的简单模型，而是人类（教师、学生）与非人（软件、算法、家庭空间）共同构建的异质网络。在这个网络中，平台软件、算法、家庭空间等非人行动者与人类行动者同样主动，共同塑造了在线教学的实践。成功不是人类适应技术，而是人类与非人共同演化的结果。"

---

## 🎯 关键教训

### 1. 对称性是ANT的基础
❌ **错误**：非人行动者比例<10%，仅作为"工具"提及
✅ **正确**：非人行动者比例≥30%，与人类行动者同等分析

### 2. 非人行动者是主动代理人
❌ **错误**：技术是"工具"、"背景"、"环境"
✅ **正确**：技术有能动性，塑造和定义实践

### 3. 对称性检查是必须的
❌ **错误**：不检查对称性，自称使用ANT
✅ **正确**：强制检查，非人比例≥30%

### 4. 不对称导致结论偏差
❌ **错误**：成功归因于人类意图和动机
✅ **正确**：成功是人类与非人共同演化的结果

## 防范措施

**行动者识别自检**：
- [ ] 行动者清单中非人比例≥30%？
- [ ] 非人行动者有独立能动性分析？
- [ ] 非人行动者有转译过程记录？
- [ ] 是否使用了"工具"、"背景"等不对称词汇？
- [ ] 结论中人类与非人是否对称？

**对称性强制检查**：
```python
def check_symmetry(actors_list):
    human_count = count(actors_list, type="human")
    nonhuman_count = count(actors_list, type="nonhuman")
    total = human_count + nonhuman_count
    symmetry_index = nonhuman_count / total

    if symmetry_index < 0.3:
        return "FAIL: Need more nonhuman actors"
    else:
        return "PASS: Symmetry maintained"
```

---

**对称性原则**：ANT的核心，不可妥协
**对称性检查**：必须执行，不可跳过
**对称性后果**：不对称导致网络不完整、结论偏差
