## 高频分析模式

### Pattern 1: 技术系统实施追踪

**频率**: 高频 (8次/月)
**成功率**: 92%
**平均时长**: 8-12小时

#### 标准任务序列

```yaml
Phase 1: 行动者识别 (1.5-2小时)
  Task 1.1: 开放式行动者识别 (30-45分钟)
    - 输出: 初步行动者清单
    - 验证: 人+非人都包含

  Task 1.2: 对称性检查 (30分钟)
    - 输出: 对称性分析报告
    - 验证: 非人行动者比例≥30%

  Task 1.3: 行动者清单制定 (30-45分钟)
    - 输出: 完整行动者清单
    - 验证: 每个行动者有初步描述

Phase 2: 转译过程追踪 (4-5小时)
  Task 2.1: 阶段1-问题化 (1小时)
    - 输出: 问题化分析报告
    - 验证: 核心问题和OPP识别

  Task 2.2: 阶段2-利益赋予 (1小时)
    - 输出: 利益赋予分析报告
    - 验证: 利益和身份重新定义

  Task 2.3: 阶段3-招募 (1小时)
    - 输出: 招募过程分析报告
    - 验证: 招募策略和抵抗记录

  Task 2.4: 阶段4-动员 (1小时)
    - 输出: 动员过程分析报告
    - 验证: 代表关系和网络稳定化

Phase 3: 对称性分析 (1小时)
  Task 3.1: 人与非人的对称性 (30分钟)
    - 输出: 对称性分析
    - 验证: 完全对称，无功能主义

  Task 3.2: 行动者角色动态分析 (30分钟)
    - 输出: 角色变化分析
    - 验证: 过程追踪完整

Phase 4: 争议和黑箱分析 (1.5-2小时)
  Task 4.1: 争议识别 (30-45分钟)
    - 输出: 争议清单
    - 验证: 所有争议记录

  Task 4.2: 黑箱打开 (1-1.5小时)
    - 输出: 黑箱打开报告
    - 验证: 所有黑箱被打开

Phase 5: 转译叙事撰写 (1-2小时)
  Task 5.1: 数据整合 (30-60分钟)
    - 输出: 整合数据清单
    - 验证: 所有追踪数据完整

  Task 5.2: 转译叙事撰写 (30-60分钟)
    - 输出: 完整转译叙事报告
    - 验证: 过程优先，无功能主义
```

#### 质量检查清单

- [ ] **对称性遵守**: 非人行动者比例≥30%
- [ ] **行动者完整性**: 人+非人都完整识别
- [ ] **4阶段完整**: 问题化→利益赋予→招募→动员
- [ ] **黑箱打开**: 技术/制度/理论都打开
- [ ] **争议记录**: 包含所有争议和失败
- [ ] **过程优先**: 叙事强调过程，非结果
- [ ] **无功能主义**: 不用"功能"、"目的"解释

#### 常见变化

- **简单技术** (如单个软件): 减少到6-8小时
- **复杂技术** (如系统集成): 增加到12-15小时
- **高度争议**: 增加争议分析时间
- **跨组织**: 增加边界追踪

#### 成功因素

✅ **对称性优先**: 先识别非人行动者
✅ **4阶段完整**: 不跳过任何阶段
✅ **黑箱系统化**: 分类打开（技术/制度/理论）
✅ **争议敏感**: 主动寻找争议和失败

#### 常见陷阱

❌ 只关注人类行动者
❌ 跳过某些转译阶段
❌ 接受"黑箱"作为终点
❌ 只看成功，忽略失败

### Pattern 2: 政策推行过程分析

**频率**: 中频 (5次/月)
**成功率**: 88%
**平均时长**: 6-10小时

#### 标准任务序列

```yaml
Phase 1: 政策文本分析 (1-2小时)
  Task 1.1: 政策行动者识别 (30分钟)
  Task 1.2: 政策作为行动者分析 (30分钟)
  Task 1.3: 政策网络构建 (30-60分钟)

Phase 2: 转译过程追踪 (3-4小时)
  Task 2.1: 问题化: 政策问题如何定义 (1小时)
  Task 2.2: 利益赋予: 利益如何被重新定义 (1小时)
  Task 2.3: 招募: 行动者如何被招募 (1小时)
  Task 2.4: 动员: 政策如何被实施 (1小时)

Phase 3: 对称性分析 (1小时)
  Task 3.1: 政策vs人类行动者 (30分钟)
  Task 3.2: 政策的主动性 (30分钟)

Phase 4: 争议分析 (1-2小时)
  Task 4.1: 政策争议识别 (30-60分钟)
  Task 4.2: 实施争议分析 (30-60分钟)

Phase 5: 政策网络撰写 (1小时)
  Task 5.1: 数据整合 (30分钟)
  Task 5.2: 政策网络叙事 (30分钟)
```

#### 特殊考虑

- **政策作为行动者**: 政策本身是行动者，不只是工具
- **多层级追踪**: 从政策制定到实施
- **文本分析**: 政策文本的转译效果
- **时空跨越**: 政策如何跨越时间和空间

### Pattern 3: 跨域网络构建研究

**频率**: 低频 (3次/月)
**成功率**: 85%
**平均时长**: 10-15小时

#### 标准任务序列

```yaml
Phase 1: 跨界识别 (2-3小时)
  Task 1.1: 多域行动者识别 (1小时)
  Task 1.2: 边界追踪 (1小时)
  Task 1.3: 跨界点识别 (1小时)

Phase 2: 转译过程追踪 (4-5小时)
  Task 2.1: 问题化: 跨界问题如何定义 (1小时)
  Task 2.2: 利益赋予: 跨界利益如何协调 (1小时)
  Task 2.3: 招募: 跨界行动者如何招募 (1.5小时)
  Task 2.4: 动员: 跨界网络如何稳定 (1.5小时)

Phase 3: 对称性分析 (1.5小时)
  Task 3.1: 域间对称性 (45分钟)
  Task 3.2: 边界行动者分析 (45分钟)

Phase 4: 争议和黑箱 (2-3小时)
  Task 4.1: 跨界争议识别 (1小时)
  Task 4.2: 边界黑箱打开 (1-2小时)

Phase 5: 网络叙事撰写 (1.5-2小时)
  Task 5.1: 跨域数据整合 (1小时)
  Task 5.2: 跨域网络叙事 (1小时)
```

#### 特殊考虑

- **边界开放性**: 不预设边界
- **异质性**: 接受网络的异质性
- **跨学科追踪**: 不受领域限制
- **翻译过程**: 不同域的"翻译"过程

## ANT特定模式

### Pattern 4: 转译过程四阶段

**核心模式**: 所有ANT分析的必经之路

#### 阶段1: 问题化 (Problematization)

**关键问题**:
- 谁在定义问题？
- 问题如何被定义？
- 必要路径（OPP）是什么？
- 谁被定义为"无关"？

**验证点**:
- [ ] 核心问题识别
- [ ] OPP识别
- [ ] 行动者角色定义
- [ ] "他者"识别

**常见错误**:
- ❌ 只有一个问题定义
- ❌ 忽视"他者"的构建
- ❌ 忽视OPP的识别

#### 阶段2: 利益赋予 (Interessement)

**关键问题**:
- 利益如何被重新定义？
- 身份如何被重新定义？
- 使用什么策略？
- 有哪些竞争策略？

**验证点**:
- [ ] 利益重新定义追踪
- [ ] 身份重新定义追踪
- [ ] 策略识别
- [ ] 竞争策略识别

**常见错误**:
- ❌ 认为利益是固定的
- ❌ 忽视身份变化
- ❌ 只关注成功策略

#### 阶段3: 招募 (Enrollment)

**关键问题**:
- 行动者如何被招募？
- 有哪些抵抗？
- 如何处理抵抗？
- 谁被成功招募？

**验证点**:
- [ ] 招募策略追踪
- [ ] 抵抗记录
- [ ] 抵抗处理追踪
- [ ] 招募成功/失败记录

**常见错误**:
- ❌ 只看成功招募
- ❌ 忽视抵抗
- ❌ 认为招募是最终状态

#### 阶段4: 动员 (Mobilization)

**关键问题**:
- 谁代表谁？
- 代表关系如何形成？
- 网络如何稳定？
- 黑箱如何形成？

**验证点**:
- [ ] 代表关系识别
- [ ] 网络稳定化追踪
- [ ] 黑箱形成过程
- [ ] 动员成功/失败记录

**常见错误**:
- ❌ 认为动员是最终状态
- ❌ 忽视代表关系的变化
- ❌ 接受黑箱作为终点

### Pattern 5: 对称性分析

**核心原则**: 人与非人完全对称

#### 对称性检查清单

```yaml
行动者识别:
  - [ ] 人类行动者: 完整识别
  - [ ] 非人行动者: 完整识别
  - [ ] 非人比例: ≥30%
  - [ ] 对称性: 无偏向

行动者分析:
  - [ ] 人类行动者: 详细分析
  - [ ] 非人行动者: 相同详细程度
  - [ ] 主动角色: 非人行动者有主动角色
  - [ ] 效果分析: 非人行动者的效果追踪

网络分析:
  - [ ] 连接对称: 人-人、人-非、非-非都追踪
  - [ ] 影响对称: 所有关系都追踪
  - [ ] 权力对称: 不预设权力关系

叙事撰写:
  - [ ] 语言对称: 不用"工具"、"背景"等词
  - [ ] 解释对称: 不用功能主义解释非人
  - [ ] 重要性对称: 不预设哪类更重要
```

#### 常见对称性违反

**违反1: 只关注人类**
- 错误: "EHR系统被医生使用"
- 正确: "EHR系统重新定义了医生实践"

**违反2: 非人作为工具**
- 错误: "隐私法规约束了行为"
- 正确: "隐私法规塑造了实践的边界"

**违反3: 非人作为背景**
- 错误: "在技术环境下，医生..."
- 正确: "技术环境与医生共同构成实践"

### Pattern 6: 黑箱打开

**核心原则**: 所有黑箱都要打开

#### 黑箱类型和打开策略

```yaml
技术黑箱:
  类型: 软件系统、技术设备、基础设施
  打开策略:
    - 算法如何定义问题？
    - 界面如何塑造行为？
    - 数据结构如何重新定义对象？

  示例: EHR系统
    - 算法: 如何定义"标准化"？
    - 界面: 如何塑造工作流程？
    - 数据: 如何重新定义"患者"？

制度黑箱:
  类型: 法律、法规、政策、标准
  打开策略:
    - 如何被制定？
    - 如何塑造行为？
    - 如何与实施者互动？

  示例: 隐私法规
    - 制定: 利益相关者如何参与？
    - 塑造: 如何约束行为？
    - 互动: 如何被解释和应用？

理论黑箱:
  类型: 科学理论、方法论、专业知识
  打开策略:
    - 如何被构建？
    - 如何塑造研究？
    - 如何与实践互动？

  示例: 循证医学
    - 构建: "证据"如何被定义？
    - 塑造: 如何约束医疗实践？
    - 互动: 如何与医生经验互动？
```

#### 黑箱打开检查清单

- [ ] 技术黑箱: 所有技术都打开
- [ ] 制度黑箱: 所有制度都打开
- [ ] 理论黑箱: 所有理论都打开
- [ ] 内部行动者: 黑箱内部的行动者也追踪
- [ ] 重新黑箱化: 打开后如何重新黑箱化

## 成功模式

### Success Pattern 1: 对称性优先策略

**模式**: 先识别非人行动者，确保对称性

**实施步骤**:
1. 第一轮: 专门识别非人行动者
2. 第二轮: 识别人类行动者
3. 第三轮: 检查对称性（非人比例≥30%）
4. 第四轮: 如不达标，补充非人行动者

**效果**:
- 对称性达标率: 100%
- 分析质量: 显著提升

**案例**: EHR系统实施研究
- 非人行动者: 25个（55.6%）
- 人类行动者: 20个（44.4%）
- 对称性: 完全对称

---

### Success Pattern 2: 4阶段完整追踪

**模式**: 不跳过任何转译阶段

**实施步骤**:
1. 阶段1-问题化: 问题如何被定义？
2. 阶段2-利益赋予: 利益如何被重新定义？
3. 阶段3-招募: 行动者如何被招募？
4. 阶段4-动员: 网络如何稳定？

**验证点**:
- 每阶段独立验证
- 阶段间连接清晰
- 不跳过任何阶段

**效果**:
- 转译过程完整性: 100%
- 网络变化清晰可见

---

### Success Pattern 3: 黑箱系统化打开

**模式**: 分类打开所有黑箱

**实施步骤**:
1. 识别所有黑箱（技术/制度/理论）
2. 技术黑箱: 算法、界面、数据结构
3. 制度黑箱: 制定、塑造、互动
4. 理论黑箱: 构建、塑造、实践
5. 追踪黑箱内部的行动者

**效果**:
- 黑箱打开率: 100%
- 网络深度: 显著提升

## 常见错误模式

### Error Pattern 1: 只关注人类行动者

**错误表现**:
- 行动者清单中人类占90%以上
- 非人行动者只是"工具"、"环境"
- 对称性检查缺失

**后果**:
- 网络不完整
- 转译过程不准确
- 理论深度不足

**纠正**:
1. 重新识别非人行动者
2. 确保非人比例≥30%
3. 对称性分析

---

### Error Pattern 2: 转译过程不完整

**错误表现**:
- 只看"采用"或"实施"
- 跳过某些转译阶段
- 只关注成功

**后果**:
- 过程信息丢失
- 争议和失败被忽略
- 网络变化不清晰

**纠正**:
1. 完整追踪4阶段
2. 记录所有争议
3. 包含失败案例

---

### Error Pattern 3: 接受黑箱

**错误表现**:
- "EHR系统就是这样的"
- "政策规定..."
- "科学研究表明..."

**后果**:
- 网络简化
- 理解不深入
- 关键行动者被遗漏

**纠正**:
1. 识别所有黑箱
2. 打开黑箱
3. 追踪内部行动者

## 优化建议

### 短期优化（下次执行）

1. **对称性前置**
   - 在任务分解时就明确对称性检查
   - 第一轮专门识别非人行动者

2. **4阶段模板化**
   - 为每个阶段创建问题清单
   - 确保不遗漏任何阶段

3. **黑箱清单**
   - 预先列出常见黑箱类型
   - 系统化打开

### 长期优化（技能更新）

1. **创建专门子技能**

2. **自动化工具**
   - 行动者识别工具（自然语言处理）

3. **案例库扩展**
