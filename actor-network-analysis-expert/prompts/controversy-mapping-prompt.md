# Controversy Mapping Prompt - 争议映射

## 任务描述

识别并映射行动者网络中的**争议（Controversies）**和冲突。

## 争议类型

### 1. 技术争议

- 技术选择争议
- 技术效果争议
- 技术可靠性争议

### 2. 社会争议

- 利益冲突
- 价值观冲突
- 权力争议

### 3. 伦理争议

- 隐私争议
- 公平性争议
- 责任归属争议

## 追踪要点

### 争议识别

- 争议如何产生？
- 争议如何发展？
- 争议如何解决/僵持？

### 争议方

- 谁支持？谁反对？
- 联盟如何形成？
- 立场如何变化？

### 争议结果

- 胜利者/失败者
- 网络如何重组
- 新的行动者出现？

## 输出格式

```yaml
争议映射:

争议1: 技术选择
  类型: 技术争议
  参与方:
    - 行动者A: 支持技术X
    - 行动者B: 反对技术X
  过程:
    - 阶段1: ...
    - 阶段2: ...
  结果:
    - 解决/僵持/搁置
    - 网络变化: ...

争议2: 隐私担忧
  类型: 伦理争议
  ...

争议3: 利益分配
  类型: 社会争议
  ...

整体评估:
  争议总数: X
  已解决: Y
  僵持: Z
  失败记录: 有/无
```

## 禁止事项

- ❌ 只关注"成功"路径
- ❌ 忽略争议和冲突
- ❌ 忽略失败案例
- ❌ 简化争议为"功能"问题
