# Translation Process Prompt - 转译过程追踪

## 任务描述

追踪行动者网络中**转译（Translation）**的4个阶段，理解网络如何形成和稳定。

## 4个转译阶段

### 阶段1: 问题化 (Problematization)

**核心问题**: 谁定义了问题？问题如何被界定？

**必须追踪**:
- 核心问题是什么？谁定义的？
- 哪些行动者被识别为"相关"？
- obligatory passage point (OPP) 如何形成？
- 谁是"不可或缺"的行动者？

**分析要点**:
- 问题定义权在谁手中？
- 如何排除其他问题？
- 哪些行动者被"问题化"？

### 阶段2: 利益赋予 (Interest Assignment)

**核心问题**: 各行动者被赋予什么利益/角色？

**必须追踪**:
- 每个行动者的利益是什么？
- 利益如何重新定义行动者身份？
- 联盟如何形成？

**分析要点**:
- 人类行动者获得什么身份？
- 非人行动者获得什么"代理能力"？
- 利益如何协调/冲突？

### 阶段3: 招募 (Enrolment)

**核心问题**: 行动者如何被招募进入网络？

**必须追踪**:
- 招募策略有哪些？
- 谈判过程如何？
- 抵抗和妥协如何发生？
- 谁被排除？

**分析要点**:
- 成功招募的策略
- 失败的招募尝试
- 抵抗形式

### 阶段4: 动员 (Mobilization)

**核心问题**: 被动员的行动者如何成为代表？

**必须追踪**:
- 代表关系如何建立？
- 代言人如何形成？
- 网络如何稳定化？
- 脆弱点在哪里？

## 输出格式

```yaml
转译过程分析:

问题化:
  核心问题: xxx
  OPP: xxx
  不可或缺者: [列表]
  
利益赋予:
  行动者-利益映射:
    - 行动者A: 利益x
    - 行动者B: 利益y
  
招募:
  成功招募: [列表]
  抵抗/妥协: [列表]
  
动员:
  代表关系: [列表]
  稳定化机制: [列表]
  脆弱点: [列表]
```

## 禁止事项

- ❌ 跳过任何一个阶段
- ❌ 只关注"成功"案例
- ❌ 忽略抵抗和失败
- ❌ 预设结果
- ❌ 功能主义解释
