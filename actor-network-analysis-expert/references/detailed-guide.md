    ↓
【前置检查】任务分解质量验证
  - 粒度是否足够在一次会话中完成？
  - 是否有明确的量化标准？
  - 是否有完成度检查清单？
  - 是否要求子 agent 输出数据清单？
    ↓
1. 行动者识别阶段
   - 开放式识别（人+非人）
   - 对称性检查（是否偏向某类？）
   - 行动者清单制定
   - 【检查点】非人行动者比例≥30%？是否对称？
   - 【禁止】只关注人类？非人作为补充？
    ↓
2. 转译过程追踪阶段
   - 阶段1：问题化（问题如何被定义？）
   - 阶段2：利益赋予（行动者被赋予什么利益？）
   - 阶段3：招募（行动者如何被招募？）
   - 阶段4：动员（行动者如何成为盟友？）
   - 【检查点】4阶段完整？每个行动者都追踪？
   - 【禁止】未追踪到底？预设边界？
    ↓
3. 对称性分析阶段
   - 人与非人的对称性分析
   - 行动者角色的动态分析
   - 权力/影响的网络分析
   - 【检查点】对称性原则遵守？无功能主义？
   - 【禁止】功能主义解释？人为/非人二分？
    ↓
4. 争议识别阶段
   - 识别行动者间的争议
   - 识别转译过程中的冲突
   - 识别失败和替代方案
   - 【检查点】争议完整记录？失败分析？
   - 【禁止】只看成功？忽略意外？
    ↓
5. 网络稳定性分析阶段
   - 网络如何被稳定？
   - 黑箱如何形成？
   - 点心脆弱性在哪？
   - 【检查点】黑箱打开？稳定性分析？
   - 【禁止】接受黑箱？静态观？
    ↓
6. 转译叙事撰写阶段
   - 整合所有追踪数据
   - 撰写转译过程叙事
   - 提供网络地图
   - 【检查点】叙事完整？过程优先？
   - 【禁止】结果导向？功能主义？
    ↓
【最终检查】完成度验证清单（18 项）
    ↓
【承诺宣读】确认未违反任何禁止原则
    ↓
报告完成（仅当全部验证通过）
```

## 📋 任务分解规则

### 分解原则

1. **粒度可控**：每个子任务必须能在一次会话中完成
2. **量化标准**：每个子任务必须有明确的完成标准
3. **独立验证**：每个子任务完成后必须独立验证
4. **数据清单**：子 agent 必须输出完整的数据清单

### 分解模板

```
总任务：[任务名称]
  ├─ 子任务 1: [具体描述]
  │   - 完成标准：[量化指标]
  │   - 输出清单：[行动者清单/转译记录]
  │   - 验证方法：[对称性检查/追踪深度检查]
  │
  ├─ 子任务 2: [具体描述]
  │   - 完成标准：[量化指标]
  │   - 输出清单：[文件列表]
  │   - 验证方法：[检查步骤]
  │
  └─ ...
```

### 委托要求

**必须明确**：
- 任务范围（具体行动者/阶段/情境）
- 完成标准（行动者数量、追踪深度、对称性）
- 输出清单（行动者清单、转译记录、争议清单）
- 验证方法（对称性检查、黑箱检查、追踪深度）

**禁止模糊**：
- "分析一些行动者"
- "看看主要过程"
- "追踪核心链条"
- "深入关键部分"

## 🎓 质量标准

### 分析质量评估矩阵

| 维度 | 合格标准 | 检查方法 |
|------|---------|---------|
| **对称性遵守** | 人与非人完全对称 | 逐项检查 |
| **追踪深度** | 每个行动者都追根溯源 | 追踪链检查 |
| **边界开放性** | 由行动者自己定义 | 边界清单检查 |
| **过程完整性** | 4阶段转译完整 | 阶段检查 |
| **争议记录** | 包含所有争议和失败 | 争议清单检查 |
| **黑箱打开** | 所有黑箱都被打开 | 黑箱清单检查 |
| **叙事质量** | 过程优先，非功能主义 | 叙事检查 |
| **禁止原则遵守** | 完全遵守所有禁止原则 | 逐项检查 |

### 违规后果

| 违规类型 | 后果 | 纠正措施 |
|---------|------|---------|
| 人为/非人二分 | 对称性丢失，网络不完整 | 重新识别所有行动者 |
| 未追踪到底 | 转译链条断裂 | 继续追踪直到完整 |
| 预设边界 | 关键行动者被遗漏 | 重新打开边界 |
| 静态网络观 | 过程信息丢失 | 重新进行过程追踪 |
| 功能主义解释 | 理论深度不足 | 重新撰写叙事 |
| 接受黑箱 | 网络简化，理解不深入 | 打开所有黑箱 |
