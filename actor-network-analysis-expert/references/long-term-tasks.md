# 行动者网络分析 - 长时任务指南

## 长时任务场景

### 场景1：技术系统实施追踪（3-6个月）

**适用情况**：
- 新技术在工作场所的实施
- 技术采用的纵向研究
- 技术-社会的共同演变研究

**任务分解**：

#### Month 1: 初始行动者识别与问题化
```
Week 1-2: 开放式行动者识别
  - 识别所有人类行动者（用户、管理者、技术支持等）
  - 识别所有非人行动者（技术、文档、制度、物理空间）
  - 输出：行动者清单（对称性检查：非人≥30%）

Week 3-4: 转译第一阶段 - 问题化
  - 追踪问题如何被定义
  - 识别核心转译者
  - 定义必要路径（OPP）
  - 输出：问题化分析报告
```

#### Month 2: 利益赋予与招募过程
```
Week 5-6: 转译第二阶段 - 利益赋予
  - 追踪每个行动者被赋予的利益
  - 分析身份如何被重新定义
  - 识别锁定机制
  - 输出：利益赋予分析报告

Week 7-8: 转译第三阶段 - 招募
  - 追踪招募过程
  - 记录谈判和妥协
  - 识别反叛和抵抗
  - 输出：招募过程记录
```

#### Month 3: 动员与网络稳定
```
Week 9-10: 转译第四阶段 - 动员
  - 追踪代表关系的形成
  - 分析网络稳定机制
  - 识别脆弱性
  - 输出：动员分析报告

Week 11-12: 网络稳定性分析
  - 识别黑箱化过程
  - 分析点心脆弱性
  - 评估网络稳定性
  - 输出：稳定性分析报告
```

#### Months 4-6: 持续追踪与争议分析
```
Month 4: 争议识别与追踪
  - 识别持续争议
  - 追踪争议如何处理
  - 分析争议对网络的影响

Month 5: 深度追踪关键行动者
  - 选择3-5个关键行动者
  - 深度追踪其转译过程
  - 分析其网络作用

Month 6: 综合分析与报告撰写
  - 整合所有追踪数据
  - 撰写转译叙事
  - 提供理论贡献
```

### 场景2：政策推行过程研究（4-8个月）

**适用情况**：
- 公共政策的实施研究
- 政策-社会的互动研究
- 政策网络构建研究

**任务分解**：

#### Phase 1: 政策文本分析（Month 1）
```
Week 1-2: 政策作为行动者
  - 政策文件如何定义问题？
  - 政策如何定义相关行动者？
  - 政策如何规定解决方案？

Week 3-4: 政策网络初步识别
  - 识别政策涉及的行动者
  - 分析政策文本中的转译
  - 输出：政策文本分析报告
```

#### Phase 2: 政策实施追踪（Months 2-4）
```
Month 2: 实施初期
  - 追踪政策如何被地方化
  - 识别初期争议和抵抗
  - 分析转译如何开始

Month 3: 实施中期
  - 追踪利益赋予和招募
  - 记录谈判和妥协
  - 分析行动者联盟形成

Month 4: 实施后期
  - 追踪动员过程
  - 分析网络稳定化
  - 识别最终脆弱性
```

#### Phase 3: 多点比较（Months 5-6）
```
Month 5: 选择2-3个实施点
  - 比较不同点的转译过程
  - 分析差异的原因
  - 识别共同模式

Month 6: 深度案例研究
  - 选择1-2个典型案例
  - 深度追踪关键行动者
  - 分析成功/失败因素
```

#### Phase 4: 综合分析（Months 7-8）
```
Month 7: 跨案例分析
  - 比较不同实施点
  - 提炼共同模式
  - 识别关键差异

Month 8: 报告撰写
  - 整合所有数据
  - 撰写转译叙事
  - 提供政策建议
```

### 场景3：跨域网络构建研究（6-12个月）

**适用情况**：
- 跨学科合作研究
- 跨组织联盟研究
- 跨领域创新网络研究

**任务分解**：

#### Phase 1: 边界识别（Months 1-2）
```
Month 1: 识别初始边界
  - 识别涉及的领域/组织
  - 识别边界行动者（边界跨越者）
  - 分析边界如何被定义

Month 2: 追踪边界开放
  - 边界如何被挑战？
  - 行动者如何跨越边界？
  - 新行动者如何进入？
```

#### Phase 2: 转译过程追踪（Months 3-6）
```
Month 3-4: 问题化与利益赋予
  - 跨域问题如何被定义？
  - 不同域的利益如何被协调？
  - 身份如何被重新定义？

Month 5-6: 招募与动员
  - 跨域招募如何进行？
  - 跨域联盟如何形成？
  - 代表关系如何建立？
```

#### Phase 3: 异质工程分析（Months 7-9）
```
Month 7: 识别异质元素
  - 识别不同域的行动者
  - 分析异质性的来源
  - 追踪异质性如何被管理

Month 8: 分析异质性稳定
  - 异质性如何被维持？
  - 同质化压力如何被抵抗？
  - 多样性如何被保护？

Month 9: 评估异质工程成功
  - 跨域网络是否成功？
  - 成功/失败的关键因素？
  - 理论贡献？
```

#### Phase 4: 深度追踪与理论提炼（Months 10-12）
```
Month 10: 深度追踪关键转译者
  - 选择2-3个关键转译者
  - 深度追踪其转译策略
  - 分析其成功因素

Month 11: 争议深度分析
  - 选择2-3个关键争议
  - 深度追踪争议过程
  - 分析争议的解决/延续

Month 12: 理论提炼与报告
  - 整合所有数据
  - 提炼理论贡献
  - 撰写研究报告
```

## 跨对话任务状态保持

### 状态文件结构

```
project/
├── task_plan.md           # 总体任务计划
├── progress.md            # 进度日志
├── findings.md            # 关键发现
├── actors/                # 行动者清单
│   ├── human_actors.md
│   ├── nonhuman_actors.md
│   └── actor_network.md
├── translation/           # 转译记录
│   ├── phase1_problematization.md
│   ├── phase2_interestment.md
│   ├── phase3_enrollment.md
│   └── phase4_mobilization.md
├── controversies/         # 争议记录
│   └── controversy_list.md
└── checkpoints/           # 阶段检查点
    ├── phase1_complete.md
    ├── phase2_complete.md
    └── ...
```

### 会话恢复机制

**每次会话开始时**：
1. 读取 `progress.md` - 了解当前进度
2. 读取 `findings.md` - 了解关键发现
3. 读取最新 checkpoint - 了解已完成的工作
4. 确认下一步任务

**每次会话结束时**：
1. 更新 `progress.md` - 记录新完成的任务
2. 更新 `findings.md` - 记录新发现
3. 创建 checkpoint - 保存当前状态
4. 规划下次会话任务

## 质量保证机制

### 每阶段验证清单

**行动者识别阶段**：
- [ ] 对称性检查（非人≥30%）
- [ ] 行动者类型多样性
- [ ] 行动者清单完整
- [ ] 没有预设边界

**转译追踪阶段**：
- [ ] 4阶段完整
- [ ] 每个行动者都追踪
- [ ] 追踪到底
- [ ] 过程记录详细

**争议分析阶段**：
- [ ] 争议识别完整
- [ ] 争议追踪到底
- [ ] 失败案例记录
- [ ] 抵抗案例记录

**报告撰写阶段**：
- [ ] 过程优先叙事
- [ ] 无功能主义解释
- [ ] 对称性贯穿全文
- [ ] 黑箱全部打开

### 阶段间质量检查

**从行动者识别到转译追踪**：
- 检查：所有行动者是否进入转译分析？
- 验证：转译分析是否覆盖所有行动者？
- 确认：追踪深度是否足够？

**从转译追踪到争议分析**：
- 检查：所有争议是否被识别？
- 验证：争议是否在转译过程中？
- 确认：争议是否被充分分析？

## 常见问题与解决

### Q1: 行动者太多，无法全部追踪
**问题**：行动者数量庞大，追踪困难
**解决**：
- 识别关键行动者（转译者、桥梁、守门人）
- 分层追踪（核心/外围）
- 时间抽样（关键时刻深度追踪）

### Q2: 数据获取困难
**问题**：某些行动者难以访问
**解决**：
- 使用文档和档案
- 寻找替代数据源
- 明确数据局限性
- 使用三角验证

### Q3: 转译过程复杂，难以理清
**问题**：转译过程混乱，难以追踪
**解决**：
- 时间轴方法（按时间重构）
- 关键事件方法（从关键事件逆向追踪）
- 网络地图方法（可视化网络演变）
