# 行动者网络分析质量标准

## 对称性质量（Symmetry Quality）

### 测量对称性

**行动者对称性**：
- 人与非人行动者数量平衡（非人≥30%）
- 人与非人分析深度相同
- 不使用"工具"、"背景"等不对称词汇

**分析对称性检查**：
- [ ] 行动者清单中非人比例≥30%
- [ ] 非人行动者有独立能动性分析
- [ ] 非人行动者有转译过程记录
- [ ] 无不对称语言（如"技术工具"）

### 对称性验证方法

**清单验证**：
```
行动者总数：N
人类行动者：H
非人行动者：NH
对称性指数 = NH / N ≥ 0.3
```

**深度验证**：
- 抽样检查5个非人行动者的分析深度
- 与5个人类行动者的分析深度对比
- 应该相同或更详细

## 追踪完整性（Traceability）

### 追踪深度标准

**每个行动者必须追踪**：
- **起源**：哪里来？如何被定义？
- **关联**：与谁连接？如何被招募？
- **影响**：对谁施加影响？如何施加？
- **变化**：如何变化？为什么变化？
- **终点**：链条延伸到哪里？

### 追踪完整性检验

**链条完整性**：
- [ ] 每个行动者都有完整链条
- [ ] 没有"断裂"的链条
- [ ] 没有"到此为止"的简化
- [ ] 关键行动者有深度追踪

**黑箱检查**：
- [ ] 所有黑箱都被打开
- [ ] 技术物内部转译可见
- [ ] 制度内部过程可见
- [ ] 理论内部争议可见

## 伦理质量（Ethical Quality）

### 知情同意

**人类行动者**：
- 研究目的明确告知
- 参与者知情权保障
- 退出机制清晰
- 数据使用透明

**非人行动者**：
- 技术物使用尊重
- 环境影响考虑
- 可持续性评估

### 负责任研究

**结果解读**：
- 避免污名化特定行动者
- 谨慎解读因果
- 明确研究局限性
- 尊重所有行动者

**利益冲突**：
- 披露资金来源
- 披露研究者立场
- 避免利益驱动结论

### 数据管理

**数据存储**：
- 安全存储协议
- 访问权限控制
- 数据保留期限
- 匿名化处理

**数据共享**：
- 开放数据（如适用）
- 去敏处理
- 使用许可协议

## 理论严谨性（Theoretical Rigor）

### ANT 核心原则遵守

| 原则 | 合格标准 | 检查方法 |
|------|---------|---------|
| **对称性** | 人与非人完全对称 | 对称性指数≥0.3 |
| **广义对称性** | 宏观与微观对称 | 不偏向任一层次 |
| **关联性** | 关系优先于本质 | 关系网络分析完整 |
| **异质性** | 承认多样性 | 异质元素被识别 |
| **过程性** | 过程优先于结果 | 转译过程完整记录 |

### 方法论透明性

**数据来源**：
- 所有数据来源明确
- 数据收集方法透明
- 数据分析过程可追溯

**理论应用**：
- ANT概念使用准确
- 不曲解理论原意
- 引用核心文献

**局限性承认**：
- 明确分析边界
- 承认未追踪部分
- 讨论理论适用性

## 常见质量问题

### 问题1：人为/非人二分
**表现**：
- 只关注人类行动者
- 非人作为"背景"或"工具"
- 非人比例<20%

**后果**：
- 对称性丢失
- 网络不完整
- 理论不符合

**解决**：
- 强制非人行动者比例≥30%
- 对称性检查
- 重新识别行动者

### 问题2：未追踪到底
**表现**：
- 追踪到某个环节就停止
- 链条断裂
- 黑箱未打开

**后果**：
- 转译过程不完整
- 理解不深入
- 关键行动者遗漏

**解决**：
- 每个行动者都追根溯源
- 打开所有黑箱
- 持续追踪

### 问题3：功能主义解释
**表现**：
- 用"功能"、"目的"解释
- 事后理性化
- 忽略意外后果

**后果**：
- 理论深度不足
- 过程信息丢失
- 简化理解

**解决**：
- 关注效果而非目的
- 关注转译而非功能
- 记录意外后果

### 问题4：接受黑箱
**表现**：
- 接受"既定事实"
- 不追问"如何"
- 用专家权威

**后果**：
- 简化理解
- 忽略过程
- 脆弱性被隐藏

**解决**：
- 打开所有黑箱
- 追踪内部转译
- 持续质疑
