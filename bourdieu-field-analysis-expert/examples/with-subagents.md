# 布迪厄场域分析 - 子Agent并行示例

## 示例1: 并行分析多个场域

### 用户请求
```
用布迪厄场域理论分析这10个学术场域
```

### 子Agent分配
```yaml
Agent 1-10: 并行分析10个场域
  - 每个Agent负责1个场域
  - 任务: 场域边界识别、资本分布、习性模式
  - 输出: 场域分析报告

Agent 11: 跨场域比较
  - 任务: 场域间关系、资本转换模式
  - 输入: 10个场域的分析报告
  - 输出: 比较分析报告
```

### 执行时间
- **串行**: 400分钟 (10场域 × 40分钟)
- **并行**: 45分钟 (10场域并行 + 5分钟汇总)
- **加速比**: 8.9x

## 示例2: 资本类型并行分析

### 场景
分析某艺术场域的资本分布

### 并行分解
```yaml
Agent 1: 经济资本分析
  - 识别财富、收入、资产
  - 分析经济资本分布

Agent 2: 文化资本分析
  - 识别教育、资质、文化能力
  - 分析文化资本分布

Agent 3: 社会资本分析
  - 识别关系网络
  - 分析社会资本结构

Agent 4: 符号资本分析
  - 识别声誉、认可
  - 分析符号资本效应
```

### 执行流程
1. 4个Agent同时执行
2. 各自完成资本分析
3. Agent 5汇总形成完整资本分布图

## 示例3: 行动者习性并行识别

### 习性识别任务
```yaml
Agent 1: 核心支配者习性
  - 识别场域主导者的习性模式
  - 分析习性-位置对应关系

Agent 2: 挑战者习性
  - 识别新进入者的习性
  - 分析习性冲突

Agent 3: 边缘者习性
  - 识别边缘行动者习性
  - 分析习性再生产

Agent 4: 习性整体分析
  - 汇总所有习性模式
  - 识别习性分布规律
```

## 示例4: 场域动力学并行追踪

### 动力学分析
```yaml
Agent 1: 位置争夺分析
  - 识别争夺焦点
  - 分析争夺策略

Agent 2: 场域边界变化
  - 追踪边界扩张/收缩
  - 识别新行动者进入

Agent 3: 资本转换过程
  - 追踪资本转换
  - 分析转换损耗

Agent 4: 场域自主性变化
  - 评估自主性水平
  - 识别外部影响

Agent 5: 动力学综合
  - 整合所有动态过程
  - 形成动力学叙事
```

## 回退策略

### 检测条件
```yaml
失败场景:
  - 关系性分析不足
  - 本质化解释
  - 习性识别偏差
  - 资本类型单一

回退行动:
  - 降级到CLI任务队列
  - 强化反本质主义检查
  - 增加关系性分析验证
```

## 质量保证

### 关系性检查
- 是否避免本质主义解释？
- 所有分析是否基于关系？
- 行动者是否在场域关系中理解？

### 资本多样性检查
- 是否分析至少3种资本？
- 每种资本分布是否清晰？
- 资本转换是否考虑损耗？

### 习性分析检查
- 习性是否被识别？
- 习性-位置关联是否明确？
- 是否避免习性本质化？
