# 案例1: 忽视竞争解释的单案例分析

## 案例概述

**案例ID**: case-01-confirmation-bias
**类型**: 负面案例
**研究类型**: 单案例解释性研究
**研究者**: 案例研究专家技能
**时间跨度**: 6个月
**问题类型**: 方法论缺陷

---

## 原始研究设计

### 研究问题
企业创新文化如何影响研发绩效？

### 理论预设
```
创新文化 → 创新行为 → 研发绩效
```

### 案例选择
- 单案例：某科技公司
- 选择理由：该公司以创新文化著称

---

## 研究过程问题

### 问题1: 案例选择偏差

**表现**:
- 仅选择以创新著称的公司
- 未选择创新失败的对照案例
- 预设了"成功"的结果

**后果**:
- 无法验证理论的普遍性
- 确认偏误风险高

### 问题2: 数据收集偏差

**表现**:
```
访谈对象: 仅选择管理层和成功项目成员
文档分析: 仅收集正面报道和成功案例
观察: 仅关注创新活动，忽视常规工作
```

**后果**:
- 数据系统性偏差
- 缺乏三角验证

### 问题3: 忽视竞争解释

**关键事件**:
研究发现该公司研发绩效提升，研究者直接归因于创新文化，但未分析其他可能原因：

| 可能原因 | 是否分析 | 实际影响 |
|---------|----------|----------|
| 创新文化 | ✓ 深入分析 | 部分影响 |
| 市场机会 | ✗ 未分析 | 重要影响 |
| 竞争对手失误 | ✗ 未分析 | 重要影响 |
| 政策红利 | ✗ 未分析 | 中等影响 |
| 人才引进 | ✗ 未分析 | 中等影响 |

### 问题4: 分析深度不足

**表现**:
- 停留在描述层面
- 未进行深度因果分析
- 未建立完整证据链

**缺失的分析**:
```
创新文化的具体内涵是什么？
创新文化如何转化为创新行为？
创新行为如何影响研发绩效？
其他因素如何发挥作用？
```

---

## 后果分析

### 学术后果
1. 研究结论缺乏可信度
2. 无法通过同行评审
3. 理论贡献不足

### 实践后果
1. 结论无法指导实践
2. 忽视了其他重要因素
3. 可能误导决策

---

## 问题根因分析

### 根本原因

| 原因 | 表现 | 后果 |
|------|------|------|
| 确认偏误 | 只寻找支持理论的证据 | 结论不可靠 |
| 方法不严谨 | 未遵循Yin方法论 | 研究质量差 |
| 理论预设过强 | 过早锁定解释 | 忽视竞争解释 |
| 数据收集不足 | 单一来源、选择偏差 | 效度不足 |

---

## 改进方案

### 重新设计

#### 1. 案例选择改进
```
原设计: 单案例（成功案例）
改进设计: 多案例（复制逻辑）

逐案例复制（预期成功）:
- 案例A: 高创新文化公司
- 案例B: 高创新文化公司

理论复制（预期失败）:
- 案例C: 低创新文化公司
- 案例D: 低创新文化公司
```

#### 2. 数据收集改进
```
原设计: 单一来源、选择性收集
改进设计: 多源三角验证

数据来源:
- 高管访谈 + 中层访谈 + 一线员工访谈
- 内部文档 + 外部报告 + 行业数据
- 成功项目 + 失败项目
```

#### 3. 分析方法改进
```
原设计: 描述性归因
改进设计: 模式匹配 + 竞争解释分析

步骤:
1. 列出所有可能的影响因素
2. 收集各因素的证据
3. 比较各因素的解释力
4. 识别核心机制
```

---

## 教训总结

### 核心教训

| 教训 | 原因 | 预防措施 |
|------|------|----------|
| 禁止确认偏误 | 只寻找支持性证据 | 主动寻找反例 |
| 禁止忽视竞争解释 | 过早锁定结论 | 系统分析替代解释 |
| 禁止单一数据源 | 方法不严谨 | 强制三角验证 |
| 禁止便利选择 | 缺乏理论指导 | 使用复制逻辑 |
