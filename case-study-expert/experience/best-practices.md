## 一、研究设计最佳实践

### 1. 问题界定

**实践**:
- 确保问题是"如何"或"为什么"类型
- 问题要与理论对话
- 问题要有实践意义

**常见错误**:
- ❌ 问题过于宽泛
- ❌ 问题不适合案例研究方法
- ❌ 问题与理论脱节

### 2. 理论框架构建

**实践**:
- 明确核心概念定义
- 建立概念间关系假设
- 识别预期模式
- 列出竞争解释

**模板**:
```
理论框架要素:
├── 核心概念定义
├── 概念间关系假设
├── 预期的模式
└── 竞争性解释
```

## 二、案例选择最佳实践

### 1. 单案例选择

| 类型 | 选择标准 | 适用场景 |
|------|----------|----------|
| 关键案例 | 测试理论核心命题 | 理论有明确预测 |
| 极端案例 | 不寻常的极端情况 | 发现边界条件 |
| 典型案例 | 代表常见情况 | 描述性研究 |
| 启示性案例 | 之前无法研究 | 新现象探索 |
| 纵向案例 | 追踪变化过程 | 动态机制研究 |

**注意事项**:
- 明确案例边界定义
- 说明选择理由
- 承认局限性

### 2. 多案例复制逻辑

**复制逻辑 vs 抽样逻辑**:
```
❌ 抽样逻辑(案例研究不适用):
- 代表性抽样
- 概率抽样
- 统计推广

✅ 复制逻辑(案例研究适用):
- 逐案例复制(预测相似结果)
- 理论复制(预测不同结果)
- 分析性推广
```

**设计原则**:
- 逐案例复制至少2个案例
- 理论复制至少1个案例
- 总案例数建议4个以上

## 三、数据收集最佳实践

### 1. 多源证据

**Yin推荐的五种证据来源**:
| 来源 | 优势 | 局限 | 注意事项 |
|------|------|------|----------|
| 档案记录 | 稳定、可重复 | 可能不完整 | 检查来源可靠性 |
| 访谈 | 直接视角 | 回忆偏差 | 多人交叉验证 |
| 直接观察 | 真实行为 | 观察者影响 | 保持客观 |
| 参与观察 | 深入理解 | 主观性强 | 反思性记录 |
| 物理人工制品 | 实物证据 | 解释空间大 | 多角度解读 |

### 2. 三角验证

**四种三角验证**:
```
数据三角: 多来源数据
方法三角: 多种收集方法
研究者三角: 多研究者视角
理论三角: 多理论解释
```

**实施要点**:
- 每个关键结论至少两种来源验证
- 不同来源数据相互印证
- 记录不一致之处

### 3. 证据链建立

**证据链要求**:
- 从研究问题到结论每步可追溯
- 关键决策有依据
- 数据收集有记录
- 分析过程可审计

## 四、数据分析最佳实践

### 1. 模式匹配

**步骤**:
1. 明确理论预测模式
2. 收集经验观察数据
3. 比较观察与预测
4. 判断匹配程度
5. 修正理论或确认支持

**注意事项**:
- 关注不匹配的证据
- 分析匹配和不匹配的原因
- 不要忽视负面证据

### 2. 竞争解释分析

**实践**:
- 列出所有可能的解释
- 收集各解释的证据
- 比较各解释的解释力
- 识别最可能的解释

**模板**:
```
| 解释 | 支持证据 | 不支持证据 | 解释力评估 |
|------|----------|------------|------------|
| 解释A | ... | ... | 高/中/低 |
| 解释B | ... | ... | 高/中/低 |
| 解释C | ... | ... | 高/中/低 |
```

### 3. 跨案例分析

**步骤**:
1. 案例内分析(每个案例独立)
2. 建立比较矩阵
3. 识别共同模式
4. 分析差异原因
5. 构建理论框架

## 五、质量保证最佳实践

### 1. Yin四质量检验

| 检验 | 定义 | 策略 | 技术 |
|------|------|------|------|
| 构念效度 | 测量准确 | 多源证据、证据链 | 三角验证 |
| 内部效度 | 因果正确 | 模式匹配、解释构建 | 因果分析 |
| 外部效度 | 可推广 | 复制逻辑、分析性推广 | 多案例比较 |
| 信度 | 可重复 | 研究草案、数据库 | 文档记录 |

### 2. 质量检查清单

**构念效度**:
- [ ] 是否使用多种证据来源？
- [ ] 是否建立证据链？
- [ ] 是否让参与者审查草稿？

**内部效度**:
- [ ] 是否分析竞争解释？
- [ ] 是否使用模式匹配？
- [ ] 是否建立因果链条？

**外部效度**:
- [ ] 是否使用复制逻辑？
- [ ] 是否说明推广边界？
- [ ] 是否与理论对话？

**信度**:
- [ ] 是否有研究草案？
- [ ] 是否建立数据库？
- [ ] 是否记录分析过程？

## 六、常见错误与避免

### 1. 案例选择错误

| 错误 | 后果 | 避免方法 |
|------|------|----------|
| 便利抽样 | 代表性不足 | 使用理论驱动选择 |
| 边界模糊 | 分析混乱 | 明确定义案例边界 |
| 案例过少 | 验证不足 | 多案例使用复制逻辑 |

### 2. 数据收集错误

| 错误 | 后果 | 避免方法 |
|------|------|----------|
| 单一来源 | 效度不足 | 强制三角验证 |
| 确认偏误 | 结论偏颇 | 主动寻找反例 |
| 证据链断裂 | 可信度降低 | 完整记录过程 |

### 3. 数据分析错误

| 错误 | 后果 | 避免方法 |
|------|------|----------|
| 描述性止步 | 理论贡献不足 | 深度理论抽象 |
| 忽视竞争解释 | 论证不足 | 系统分析替代解释 |
| 过度推广 | 结论不可靠 | 明确推广边界 |

## 八、报告撰写最佳实践

### 结构建议

```
案例研究报告:
├── 摘要
├── 引言(问题与意义)
├── 文献综述(理论框架)
├── 研究方法(设计、选择、收集、分析)
├── 案例描述(背景与关键事件)
├── 案例分析(模式匹配与理论对话)
├── 讨论(贡献、意义、局限)
└── 结论
```

### 写作原则

1. **厚描述**: 提供丰富的情境描述
2. **证据支撑**: 每个论断有证据
3. **透明性**: 详细说明过程
4. **诚实性**: 报告不符合预期的发现
