## 成功经验

### 1. 案例选择的关键性
**经验**: 案例选择决定了研究的效度和推广性。
**证据**: 使用复制逻辑选择的多案例研究比便利抽样更可信。
**建议**: 单案例选择关键/极端/典型案例；多案例使用复制逻辑。

### 2. 三角验证的重要性
**经验**: 单一数据源不足以支撑有力的结论。
**证据**: 使用3种以上证据来源的研究构念效度更高。
**建议**: 每个案例至少使用访谈+文档+观察三种数据源。

### 3. 证据链的建立
**经验**: 完整的证据链增强研究的可审计性。
**证据**: 有完整证据链的研究更容易通过同行评审。
**建议**: 从研究问题到结论的每一步都要有证据链接。

### 4. 模式匹配的深度
**经验**: 表层模式匹配不足以产生理论贡献。
**证据**: 深度模式匹配能发现意外模式和竞争解释。
**建议**: 寻找支持和不支持理论的证据。

## 失败教训

### 1. 案例选择偏差
**教训**: 便利抽样导致代表性不足。
**原因**: 选择容易接触的案例而非理论上有意义的案例。
**预防**: 使用明确的案例选择标准和复制逻辑。

### 2. 数据收集不足
**教训**: 数据不足导致分析浅薄。
**原因**: 预设了结论，只收集支持性证据。
**预防**: 系统收集正反两方面的证据。

### 3. 忽视竞争解释
**教训**: 只寻找支持理论的证据导致确认偏误。
**原因**: 研究者过于相信自己的理论假设。
**预防**: 主动寻找和分析竞争性解释。

### 4. 分析深度不足
**教训**: 描述性案例研究缺乏理论贡献。
**原因**: 停留在描述层面，未进行理论抽象。
**预防**: 从描述到分析，从案例到理论。

## 模式库

### 模式1: 单案例解释性设计
```
关键案例选择 → 理论预设 → 多源数据收集 → 模式匹配 → 理论修正
```
**适用**: 验证或修正现有理论

### 模式2: 多案例探索性设计
```
多案例选择(复制逻辑) → 开放式数据收集 → 跨案例比较 → 理论构建
```
**适用**: 构建新理论

### 模式3: 嵌入式多案例设计
```
整体案例 → 嵌入分析单元 → 多层分析 → 综合结论
```
**适用**: 复杂现象的多层面研究

## 方法论知识

### Yin四质量检验

| 检验 | 策略 | 技术 |
|------|------|------|
| 构念效度 | 多证据来源、证据链 | 三角验证 |
| 内部效度 | 模式匹配、解释构建 | 因果分析 |
| 外部效度 | 复制逻辑、分析性推广 | 多案例比较 |
| 信度 | 案例研究草案、数据库 | 可重复性 |

### 案例选择类型
- **关键案例**: 测试理论的核心命题
- **极端案例**: 不寻常的极端情况
- **典型案例**: 代表常见情况
- **启示性案例**: 之前无法研究的情况
- **纵向案例**: 追踪变化过程
