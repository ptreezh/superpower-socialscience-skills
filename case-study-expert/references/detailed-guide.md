# 案例研究详细指南

## 理论基础

### 案例研究的定义

案例研究是一种在真实情境中深入探究当代现象的研究方法，特别适用于研究"如何"和"为什么"类型的问题，当研究者对事件缺乏控制时(Yin, 2018)。

### 核心理论框架

#### 1. Yin的实证主义案例研究
- 强调研究设计的严谨性
- 注重效度和信度检验
- 使用复制逻辑进行案例选择
- 模式匹配作为分析技术

#### 2. Stake的建构主义案例研究
- 强调对案例的深入理解
- 注重参与者的意义建构
- 自然推广而非分析性推广
- 主题分析作为分析技术

#### 3. Eisenhardt的理论构建
- 从案例构建理论
- 多案例的复制逻辑
- 深入案例内分析
- 跨案例模式识别

## 研究设计

### 一、研究问题界定

#### 适合案例研究的问题类型
- **如何(How)问题**: 过程、机制研究
- **为什么(Why)问题**: 因果、解释研究

#### 不适合的问题类型
- "多少"问题 → 使用调查
- "是否"问题 → 使用实验

### 二、理论框架构建

```
理论框架要素:
├── 核心概念定义
├── 概念间关系假设
├── 预期的模式
└── 竞争性解释
```

### 三、案例类型确定

| 类型 | 目的 | 特点 | 适用场景 |
|------|------|------|---------|
| 探索性 | 生成假设 | 开放性 | 现象新颖 |
| 描述性 | 描述现象 | 完整性 | 已有理论框架 |
| 解释性 | 解释因果 | 验证性 | 因果机制研究 |

### 四、案例选择

#### 单案例选择标准

1. **关键案例(Critical Case)**
   - 测试理论的核心命题
   - "如果在这个案例中成立，则在所有案例中都成立"

2. **极端案例(Extreme Case)**
   - 不寻常的极端情况
   - 揭示通常难以观察的机制

3. **典型案例(Typical Case)**
   - 代表常见情况
   - 用于描述性研究

4. **启示性案例(Revelatory Case)**
   - 之前无法研究的情况
   - 新现象的首次观察

5. **纵向案例(Longitudinal Case)**
   - 追踪变化过程
   - 研究动态机制

#### 多案例复制逻辑

**复制逻辑 vs 抽样逻辑**:
```
抽样逻辑(不适用):
- 代表性抽样
- 概率抽样
- 统计推广

复制逻辑(适用):
- 逐案例复制: 预测相似结果
- 理论复制: 预测不同结果
- 分析性推广
```

**复制逻辑设计示例**:
```
理论: 领导风格影响组织创新

逐案例复制(预期高创新):
- 案例A: 变革型领导 + 高创新
- 案例B: 变革型领导 + 高创新

理论复制(预期低创新):
- 案例C: 事务型领导 + 低创新
- 案例D: 事务型领导 + 低创新
```

## 数据收集

### 一、多证据来源

| 数据来源 | 具体方法 | 优势 | 局限 |
|---------|---------|------|------|
| 档案记录 | 文件、报告、记录 | 稳定、可重复 | 可能不完整 |
| 访谈 | 开放式、焦点小组 | 直接视角 | 回忆偏差 |
| 直接观察 | 参与式、非参与式 | 真实行为 | 观察者影响 |
| 参与观察 | 研究者参与 | 深入理解 | 主观性强 |
| 物理人工制品 | 工具、产品 | 实物证据 | 解释空间大 |

### 二、三角验证策略

```
三角验证类型:
├── 数据三角: 多来源数据
├── 方法三角: 多种收集方法
├── 研究者三角: 多研究者视角
└── 理论三角: 多理论解释
```

### 三、证据链建立

```
证据链示例:
研究问题 → 理论命题 → 案例选择 → 数据收集 → 分析技术 → 结论
    ↓           ↓           ↓           ↓           ↓         ↓
  文档记录    命题文档    选择备忘    收集协议    分析记录   结论备忘
```

### 四、案例研究数据库

```
数据库结构:
├── 原始数据/
│   ├── 访谈转录/
│   ├── 观察记录/
│   └── 文档资料/
├── 编码数据/
│   ├── 开放编码/
│   └── 轴心编码/
└── 分析备忘/
    ├── 案例内分析/
    └── 跨案例分析/
```

## 数据分析

### 一、模式匹配

```
模式匹配过程:
1. 确定理论预测模式
2. 收集经验数据
3. 比较经验模式与预测模式
4. 判断匹配程度
5. 修正理论或确认支持
```

**模式匹配示例**:
```
理论预测: A → B → C

案例1观察: A → B → C (完全匹配)
案例2观察: A → B' → C (部分匹配)
案例3观察: A' → B → C' (不匹配)

结论: 理论得到部分支持，需修正
```

### 二、解释构建

```
解释构建步骤:
1. 分析原始数据
2. 识别关键事件和因素
3. 构建因果链条
4. 检验竞争解释
5. 形成最终解释
```

### 三、时间序列分析

```
时间序列设计:
事前测量 → 干预 → 事后测量

关键点:
- 多个时间点测量
- 识别趋势和变化
- 分析变化原因
```

### 四、逻辑模型

```
逻辑模型示例:
投入 → 活动 → 产出 → 结果
  ↓        ↓        ↓        ↓
资源     行为     产品     影响
```

### 五、跨案例分析

```
跨案例分析步骤:
1. 案例内分析(每个案例独立分析)
2. 识别案例间模式
3. 寻找共同特征
4. 分析差异原因
5. 构建理论框架
```

## 质量保证

### 一、构念效度

**策略**:
- 多源三角验证
- 建立证据链
- 关键信息提供者审查

**检验清单**:
- [ ] 是否使用了多种数据来源？
- [ ] 是否建立了完整的证据链？
- [ ] 是否让参与者审查草稿？

### 二、内部效度

**策略**:
- 模式匹配
- 解释构建
- 时间序列分析
- 竞争性解释分析

**检验清单**:
- [ ] 是否检验了竞争解释？
- [ ] 是否建立了因果链条？
- [ ] 是否使用了模式匹配？

### 三、外部效度

**策略**:
- 复制逻辑(多案例)
- 分析性推广
- 理论框架

**检验清单**:
- [ ] 是否使用了复制逻辑？
- [ ] 是否明确推广边界？
- [ ] 是否与理论对话？

### 四、信度

**策略**:
- 案例研究草案
- 案例研究数据库
- 研究者培训

**检验清单**:
- [ ] 是否有研究草案？
- [ ] 是否建立了数据库？
- [ ] 是否记录了分析过程？

## 报告撰写

### 一、报告结构

```
案例研究报告:
├── 引言
│   ├── 研究问题
│   ├── 研究意义
│   └── 报告概览
├── 文献综述
│   ├── 理论背景
│   ├── 相关研究
│   └── 理论框架
├── 研究方法
│   ├── 研究设计
│   ├── 案例选择
│   ├── 数据收集
│   └── 数据分析
├── 案例描述
│   ├── 案例背景
│   └── 关键事件
├── 案例分析
│   ├── 模式匹配
│   └── 理论对话
├── 讨论
│   ├── 理论贡献
│   ├── 实践意义
│   └── 研究局限
└── 结论
```

### 二、写作原则

1. **厚描述**: 提供丰富的情境描述
2. **证据支撑**: 每个论断都要有证据
3. **透明性**: 详细说明研究过程
4. **诚实性**: 报告不符合预期的发现
