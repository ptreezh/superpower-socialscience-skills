# 负面案例001: p值崇拜的错误

## ❌ 错误做法

### p值崇拜

**研究问题**: 哪些因素预测学业成绩?

**数据**: N=100名学生，10个预测变量

### 错误1: 只报告p值

**分析**:
```yaml
回归分析（10个预测变量）

报告:
  - "X1显著（p=0.03）"
  - "X2显著（p=0.04）"
  - "X3不显著（p=0.15）"
  - ...

问题:
  ❌ 没有报告系数
  ❌ 没有报告标准误
  ❌ 没有报告效应量
  ❌ 没有报告置信区间
  ❌ 无法评估实际重要性
```

### 错误2: p-hacking

**过程**:
```yaml
原始分析:
  - 10个变量，只有2个显著（p<0.05）

研究者策略:
  1. 尝试不同的变量组合
  2. 删除"不显著"的变量
  3. 添加交互项
  4. 转换变量
  5. 删除异常值

经过10次尝试:
  - 现在有5个变量显著！
  - 报告这5个变量
  - 不提失败的尝试

问题:
  - p值累积错误（10次尝试→α≈0.40）
  - 选择性报告
  - 结果不可复现
```

### 错误3: 忽视效应量

**结果**:
```yaml
变量X1:
  - p = 0.03（显著）
  - β = 0.05（极小）
  - R²增加 = 0.01

结论:
  ❌ "X1是重要预测因素"

正确解读:
  ✅ "X1统计显著，但效应极小（β=0.05），
      实际意义有限"
```

### 错误4: "p>0.05 = 没有效果"

**结果**:
```yaml
变量X2:
  - p = 0.08（不显著）
  - β = 0.35（中等效应）
  - 95% CI [-0.03, 0.73]

结论:
  ❌ "X2不影响结果"

正确解读:
  ✅ "X2效应中等（β=0.35），但统计不显著（p=0.08）。
      可能是因为样本量不足（N=100）。
      95% CI包含0，但也包含有意义的效应。
      建议扩大样本重新检验。"
```

### 错误5: HARKing

**Hypothesizing After Results are Known**:

**过程**:
```yaml
探索性分析:
  - 尝试20个变量
  - 发现X3、X7、X10显著（p<0.05）

论文中:
  ❌ "基于理论，我们假设X3、X7、X10会影响结果"
  ❌ 伪装成验证性研究

正确做法:
  ✅ 明确说明是探索性分析
  ✅ 承认需要验证性研究 replication
```

---

## ✅ 正确做法

### 1. 完整报告

```yaml
正确报告:
  变量X1:
    - β = 0.05, SE = 0.02
    - t = 2.50, p = 0.03
    - 95% CI [0.01, 0.09]
    - 半偏R² = 0.01

  解读:
    "X1统计显著（p=0.03），但效应极小（β=0.05, R²=0.01），
     实际意义可能有限。"
```

### 2. 避免p-hacking

```yaml
预注册:
  - 事先假设
  - 事先分析计划
  - 事后报告所有分析

多重比较校正:
  - Bonferroni: α' = 0.05/10 = 0.005
  - 只有p<0.005才算显著
```

### 3. 关注效应量

```yaml
解释框架:
  1. 统计显著性（p值）
  2. 效应量（β, R², Cohen's d等）
  3. 置信区间（精确性）
  4. 实际意义（理论/实践重要性）

平衡判断:
  - p<0.05 但效应小 → 可能不重要
  - p>0.05 但效应大 → 可能值得进一步研究
```

### 4. 透明报告

```yaml
完整透明:
  - 报告所有分析（包括不显著的）
  - 报告所有尝试（如探索性分析）
  - 共享数据和代码
  - 明确区分探索性与验证性
```

---

## 🎯 关键教训

### p值不是一切

1. **p值**: "这个结果如果H0为真，有多大概率出现"
   - 不告诉你效应大小
   - 不告诉你实际重要性
   - 受样本量影响

2. **效应量**: "这个结果有多重要"
   - 独立于样本量
   - 实际意义指标

3. **置信区间**: "这个结果有多精确"
   - 提供更多信息
   - 包含效应量范围

### 好的实践

✅ 总是报告效应量
✅ 总是报告置信区间
✅ 讨论实际意义
✅ 预注册或明确探索性
✅ 多重比较校正
✅ 完全透明
