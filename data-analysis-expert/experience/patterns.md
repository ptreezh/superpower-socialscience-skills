## 🎯 高频分析模式

### Pattern 1: 问卷数据分析标准流程

**频率**: 高频 (15次/月)
**成功率**: 95%
**平均时长**: 3.5小时

#### 标准任务序列

```yaml
Phase 1: 数据准备 (1 hour)
  Task 1.1: 数据导入与完整性检查 (10 min)
    - 检查变量数、案例数
    - 检查数据类型
    - 识别导入错误

  Task 1.2: 数据探索 (20 min)
    - 描述统计（均值、SD、范围）
    - 分布图（直方图、Q-Q图）
    - 缺失值模式分析
    - 异常值检测（箱线图、Z分数）

  Task 1.3: 数据清洗 (30 min)
    - 处理缺失值（删除/插补）
    - 处理异常值（Winsorize/删除）
    - 变量类型转换
    - 创建清洗日志

Phase 2: 质量评估 (30 min)
  Task 2.1: 描述统计 (15 min)
    - 生成完整描述统计表
    - 按组别对比

  Task 2.2: 信度分析 (15 min)
    - Cronbach's α计算
    - Item-total correlation
    - α < 0.7时的改进建议

Phase 3: 假设检验 (1.5 hours)
  Task 3.1: 检验前假设检查 (30 min)
    - 正态性检验（Shapiro-Wilk）
    - 方差齐性检验（Levene）
    - 样本量平衡性检查

  Task 3.2: 主要假设检验 (30-45 min)
    - 选择适当检验方法
    - 执行检验
    - 计算效应量
    - 计算置信区间

  Task 3.3: 补充分析 (可选，15-30 min)
    - 亚组分析
    - 敏感性分析

Phase 4: 结果报告 (30 min)
  Task 4.1: 结果表格 (15 min)
    - APA格式结果表
    - 包含效应量、CI

  Task 4.2: 可视化 (15 min)
    - 箱线图（分布对比）
    - 森林图（效应量）
    - 森林图（置信区间）
```

#### 质量检查清单

```yaml
数据质量:
  - [ ] 数据完整性检查
  - [ ] 异常值已识别并处理
  - [ ] 缺失值已适当处理
  - [ ] 变量类型正确

分析质量:
  - [ ] 假设检验已完成
  - [ ] 正态性已检验
  - [ ] 方差齐性已检验
  - [ ] 效应量已报告
  - [ ] 置信区间已报告

报告质量:
  - [ ] 结果表格完整
  - [ ] 图表清晰准确
  - [ ] 统计符号正确
  - [ ] 实际意义已讨论
```

#### 常见变体

```yaml
小样本（n < 50）:
  - 使用非参数检验（Mann-Whitney U）
  - 报告中位数而非均值
  - Bootstrap置信区间

大样本（n > 200）:
  - 关注效应量而非p值
  - 考虑实际显著性
  - 警惕过度显著

多组比较:
  - 使用ANOVA而非多次t检验
  - Bonferroni校正
  - 事后检验（Tukey HSD）

重复测量:
  - 使用配对t检验
  - 关注个体内变化
  - 报告相关系数
```

#### 成功因素

✅ **提前规划**
- 明确每个假设的检验方法
- 预判假设违反的处理

✅ **逐步验证**
- 每步完成后验证
- 提早发现问题

✅ **完整报告**
- 不只p值
- 效应量 + 置信区间 + 实际意义

✅ **可视化辅助**
- 箱线图展示分布
- 森林图展示效应量

#### 常见陷阱

❌ **忽视正态性检验**
- 后果：第一类错误
- 预防：先Shapiro-Wilk检验

❌ **只报告p值**
- 后果：不完整的信息
- 预防：总是报告效应量

❌ **多重比较不校正**
- 后果：假阳性累积
- 预防：Bonferroni或FDR校正

❌ **过度推断因果**
- 后果：误导性结论
- 预防：观察研究用谨慎语言

### Pattern 2: 实验数据分析流程

**频率**: 中频 (10次/月)
**成功率**: 92%
**平均时长**: 2.5小时

#### 标准任务序列

```yaml
Phase 1: 数据准备 (45 min)
  Task 1.1: 数据导入与条件检查 (15 min)
  Task 1.2: 数据探索 (15 min)
  Task 1.3: 数据清洗 (15 min)

Phase 2: 描述统计 (30 min)
  Task 2.1: 按条件描述统计 (15 min)
  Task 2.2: 可视化探索 (15 min)

Phase 3: 主要分析 (1 hour)
  Task 3.1: 方差分析 (45 min)
    - One-way ANOVA或Two-way ANOVA
    - 效应量（η²或偏η²）
    - 事后检验（Tukey HSD）

  Task 3.2: 简单效应分析（可选，15 min）

Phase 4: 结果报告 (30 min)
  Task 4.1: 结果表格 (15 min)
  Task 4.2: 可视化 (15 min)
```

#### 关键差异

```yaml
与问卷分析的区别:
  1. 组别是实验操纵的
     - 可推断因果
     - 不用担心混淆变量

  2. 样本量通常较小
     - 更需要关注效应量
     - 可能需要非参数替代

  3. 关心交互效应
     - Two-way ANOVA
     - 简单效应分析
```

### Pattern 3: 二次数据分析流程

**频率**: 中频 (8次/月)
**成功率**: 90%
**平均时长**: 4-6小时

#### 标准任务序列

```yaml
Phase 1: 数据获取与理解 (1-2 hours)
  Task 1.1: 数据下载与导入 (30 min)
  Task 1.2: 文档阅读与变量理解 (30-60 min)
  Task 1.3: 数据探索 (30 min)

Phase 2: 研究问题定义 (30 min)
  Task 2.1: 文献回顾 (15 min)
  Task 2.2: 假设生成 (15 min)

Phase 3: 数据准备 (1 hour)
  Task 3.1: 变量选择与创建 (30 min)
  Task 3.2: 数据清洗 (30 min)

Phase 4: 分析 (1-2 hours)
  Task 4.1: 描述统计 (30 min)
  Task 4.2: 推断统计 (30-90 min)

Phase 5: 结果报告 (1 hour)
  Task 5.1: 结果表格 (30 min)
  Task 5.2: 讨论 (30 min)
```

#### 关键差异

```yaml
与其他分析的区别:
  1. 数据已存在
     - 需要理解原始研究设计
     - 受限于已有变量

  2. 需要更多前期工作
     - 阅读文档
     - 理解变量编码

  3. 可能需要复杂加权
     - 调查权重
     - 抽样权重
```

## 🔍 常见检验方法选择模式

### t检验选择树

```yaml
独立样本t检验:
  条件1: 两组独立
  条件2: 正态分布
  条件3: 方差齐 → Student's t
  条件4: 方差不齐 → Welch's t

配对样本t检验:
  条件1: 同一被试前后测
  条件2: 配对设计
  条件3: 差值正态分布

非参数替代:
  - Mann-Whitney U test (替代独立t)
  - Wilcoxon signed-rank (替代配对t)
```

### ANOVA选择树

```yaml
One-way ANOVA:
  条件: 单因素多组（>2）
  事后: Tukey HSD

Two-way ANOVA:
  条件: 两因素
  关注: 主效应 + 交互效应

Repeated Measures ANOVA:
  条件: 重复测量
  注意: 球形假设

非参数替代:
  - Kruskal-Wallis (替代One-way)
  - Friedman (替代Repeated Measures)
```

## 📊 效应量解释模式

### Cohen's d

```yaml
解释标准:
  d = 0.2: 小效应
  d = 0.5: 中等效应
  d = 0.8: 大效应

注意事项:
  - 社会科学通常d < 0.5
  - 不要只看数值大小
  - 要考虑实际意义
```

### η² (ANOVA)

```yaml
解释标准:
  η² = 0.01: 小效应
  η² = 0.06: 中等效应
  η² = 0.14: 大效应

报告格式:
  "主效应显著，F(2, 57) = 4.56, p = .015, η² = .14"
```

### r (相关)

```yaml
解释标准:
  r = 0.1: 小相关
  r = 0.3: 中等相关
  r = 0.5: 大相关

注意事项:
  - 不要混淆相关与因果
  - 考虑样本量影响
  - 看散点图
```
