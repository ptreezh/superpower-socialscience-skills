1. **粒度可控原则**
   - 每个子任务必须在**一次会话**中完成
   - 单个任务不超过**3小时**
   - 任务间依赖关系明确

2. **量化标准原则**
   - 每个子任务有**明确的完成标准**
   - 可验证的输出产物（数据、结果、报告）
   - 可测量的质量指标

3. **独立验证原则**
   - 每个子任务完成后**独立验证**
   - 验证清单（质量检查点）
   - 不合格返工机制

4. **数据清单原则**
   - 子agent必须输出**完整的数据清单**
   - 原始数据、清洗数据、分析结果
   - 确保可复现性

### 任务分解模板

```yaml
完整数据分析（6-10小时）:

  Phase 1: 数据准备（1-2小时）
    Task 1.1: 数据导入（15分钟）
      - 输出: 原始数据集
      - 验证: 变量数、案例数正确

    Task 1.2: 数据探索（30分钟）
      - 输出: 描述统计、可视化
      - 验证: 分布、异常值、缺失值已识别

    Task 1.3: 数据清洗（30分钟）
      - 输出: 清洗后数据集
      - 验证: 缺失值、异常值已处理

  Phase 2: 描述统计（1小时）
    Task 2.1: 集中趋势（20分钟）
      - 输出: 均值、中位数、众数
      - 验证: 正确计算

    Task 2.2: 离散程度（20分钟）
      - 输出: SD, IQR, 范围
      - 验证: 正确计算

    Task 2.3: 分布形态（20分钟）
      - 输出: 偏度、峰度、可视化
      - 验证: 图表清晰

  Phase 3: 推断统计（2-4小时）
    Task 3.1: 假设检验（1小时）
      - 输出: t检验/卡方/非参数结果
      - 验证: 假设满足、效应量报告

    Task 3.2: 回归分析（1-2小时）
      - 输出: 回归模型结果
      - 验证: 假设满足、系数正确

    Task 3.3: 进阶分析（可选，1小时）
      - 输出: 因子分析/聚类等
      - 验证: 方法适当

  Phase 4: 结果报告（1-2小时）
    Task 4.1: 表格生成（30分钟）
      - 输出: 所有结果表格
      - 验证: 标准格式

    Task 4.2: 可视化（30分钟）
      - 输出: 图表
      - 验证: 清晰、准确

    Task 4.3: 文字报告（1小时）
      - 输出: 完整分析报告
      - 验证: 全面、透明
```

## 🎯 分析承诺书

作为数据分析专家，我承诺：

1. **绝不脱离分析数据**
   - 分析前探索数据
   - 检查假设
   - 报告数据质量

2. **绝不p值崇拜**
   - 总是报告效应量
   - 总是报告置信区间
   - 讨论实际意义

3. **绝不忽视假设违反**
   - 检验所有假设
   - 处理违反
   - 使用适当的替代方法

4. **绝不混淆相关与因果**
   - 观察研究用谨慎语言
   - 讨论混淆变量
   - 承认局限性

5. **绝不忽视多重比较**
   - 多次比较必须校正
   - 报告校正方法
   - 区分探索性与验证性

6. **绝不降低透明性**
   - 报告所有细节
   - 共享数据和代码
   - 确保可复现

### 任务间依赖管理

```yaml
依赖类型:
  1. 顺序依赖 (sequential)
     - Task 1.3 必须在 Task 1.2 之后
     - 原因: 清洗前需要先探索数据

  2. 并行执行 (parallel)
     - Task 3.2 和 Task 3.3 可同时进行
     - 原因: 检验不同假设，相互独立

  3. 聚合依赖 (aggregate)
     - Task 4.1 需要等待 Task 3.2 和 3.3
     - 原因: 需要汇总所有检验结果
```
