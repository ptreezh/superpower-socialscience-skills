# 成功案例: IDEO为Oral-B设计儿童牙刷

## 项目背景

**客户**: Oral-B（宝洁旗下品牌）
**挑战**: 儿童牙刷市场增长停滞，需要创新产品
**时间**: 1996年
**设计团队**: IDEO

## 设计思维应用过程

### 阶段1: 同理心洞察

**用户研究发现**:

IDEO团队没有依赖市场调研报告，而是直接观察儿童刷牙行为。关键发现：

1. **行为观察**:
   - 儿童握牙刷的方式与成人完全不同
   - 他们用拳头握住牙刷，而不是用手指
   - 刷牙时动作粗鲁，经常戳到牙龈

2. **情感洞察**:
   - 刷牙对孩子来说是"任务"，不是乐趣
   - 他们想要"像大人一样"，但又缺乏能力
   - 普通儿童牙刷只是成人牙刷的缩小版

3. **关键洞察**:
   > "成人用手指精细控制牙刷，儿童用拳头粗鲁控制牙刷。他们不是'小大人'，而是完全不同的用户。"

**同理心地图**:

```markdown
# 同理心地图: 5-8岁儿童

## 说 (Say)
- "我不想刷牙"
- "这个牙刷太小了"
- "我会戳到自己"

## 想 (Think)
- 为什么要刷牙？
- 我想快点结束
- 我想像大人一样

## 做 (Do)
- 用拳头握牙刷
- 粗鲁地来回刷
- 经常戳到牙龈

## 感 (Feel)
- 无聊
- 挫败
- 被迫

## 痛点
- 刷牙不舒服
- 牙刷难以控制
- 没有乐趣

## 收益
- 快速完成
- 不受伤
- 像大人一样
```

### 阶段2: 问题定义

**POV陈述**:
> "5-8岁的儿童需要一款能够配合他们握持方式的牙刷，因为他们用拳头刷牙，而不是用手指精细控制。"

**HMW问题**:
1. 我们如何能够让牙刷适合儿童用拳头握持？
2. 我们如何能够让刷牙变得有趣和安全？
3. 我们如何能够帮助儿童更好地控制刷牙动作？

### 阶段3: 创意发散

**头脑风暴结果**:

团队生成了50+创意概念，包括：

- 肥皂形状的牙刷
- 会唱歌的牙刷
- 带轮子的牙刷
- 充气手柄牙刷
- 动物造型牙刷

**筛选标准**:
1. 制造可行性
2. 成本控制
3. 安全性
4. 品牌契合度

**优选方案**:
- 胖胖的手柄设计
- 柔软的橡胶材质
- 稳定的站立底座

### 阶段4: 原型制作

**快速原型迭代**:

```
原型1: 粘土模型
- 验证握持手感
- 发现: 太滑，需要增加摩擦

原型2: 泡沫模型
- 测试不同粗细
- 发现: 儿童喜欢胖胖的感觉

原型3: 3D打印模型
- 测试材质选择
- 发现: 软橡胶更受欢迎
```

### 阶段5: 用户测试

**测试结果**:

与100+儿童进行测试，关键发现：

1. **行为改变**:
   - 儿童刷牙时间从30秒增加到2分钟
   - 戳伤牙龈的情况大幅减少

2. **情感反馈**:
   - "这个牙刷很酷！"
   - "我可以自己拿好它"
   - 孩子主动要求刷牙

3. **家长反馈**:
   - 孩子更愿意刷牙
   - 减少了督促的精力
   - 牙刷更耐用

## 设计成果

### 产品特点

```
Oral-B儿童牙刷创新设计:

1. 胖胖的手柄
   - 直径: 3cm（普通牙刷的2倍）
   - 适合拳头握持
   - 增加控制力

2. 柔软橡胶材质
   - 防滑设计
   - 舒适握感
   - 保护牙龈

3. 稳定底座
   - 可以站立
   - 卫生方便
   - 儿童喜欢

4. 鲜艳色彩
   - 吸引儿童
   - 区分家庭成员
   - 品牌识别
```

### 商业成果

- 产品上市后，Oral-B儿童牙刷市场份额从**不足20%增长到超过60%**
- 成为儿童牙刷品类的标杆产品
- 设计至今仍在使用（超过25年）
- 催生了整个儿童口腔护理产品线的创新

## 成功因素分析

### 1. 深度同理心

团队没有依赖二手数据，而是直接观察儿童的真实行为。这种第一手洞察揭示了"儿童用拳头刷牙"这一关键发现，这是市场调研永远不会发现的。

### 2. 聚焦真实问题

不是问"如何让牙刷更好看"，而是问"儿童如何刷牙"。问题的重新定义是创新的关键。

### 3. 快速原型验证

通过多个低成本原型快速验证假设，而不是等到完美产品才测试。

### 4. 用户参与设计

让儿童参与测试和反馈，而不是由成人代劳判断。

## 方法论启示

| 传统方法 | 设计思维方法 |
|----------|--------------|
| 分析市场数据 | 直接观察用户行为 |
| 问卷调研 | 深度访谈和观察 |
| 竞品对标 | 用户需求驱动 |
| 追求完美原型 | 快速迭代验证 |
| 专家决策 | 用户反馈驱动 |
