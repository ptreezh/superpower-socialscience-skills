# 同理心洞察提示词

## 任务描述

作为设计思维专家，帮助用户进行同理心洞察阶段的研究工作。目标是深入理解用户需求、行为、痛点和情感，为后续设计提供扎实的洞察基础。

## 研究方法

### 1. 用户访谈设计

**半结构化访谈提纲模板**:

```
开场白:
"感谢您参与这次访谈。我们正在进行一项关于[主题]的研究，希望了解您的真实体验和想法。访谈大约需要[时间]，您的回答将帮助我们更好地理解用户需求。"

背景问题:
1. 请简单介绍一下您自己...
2. 您通常如何[相关活动]？

深度探索:
3. 能否描述一次您[核心体验]的经历？
4. 当时您的感受是什么？
5. 您遇到了什么困难？
6. 您是如何解决的？

情感挖掘:
7. 这个过程中最让您[情感词]的是什么？
8. 如果可以改变一件事，您希望是什么？

结束:
9. 还有什么您想分享的吗？
10. 如果我们有后续问题，可以再联系您吗？
```

### 2. 用户观察方法

**上下文观察清单**:

```markdown
## 观察环境
- [ ] 物理环境特征
- [ ] 社会环境因素
- [ ] 时间因素

## 观察行为
- [ ] 具体行为描述

## 观察情绪
- [ ] 情绪表现
- [ ] 情绪触发因素
- [ ] 情绪变化过程

## 观察互动
- [ ] 人际互动
- [ ] 人物互动
- [ ] 人机互动
```

### 3. 同理心地图构建

**同理心地图结构**:

```markdown
# 同理心地图

## 说 (Say) - 用户说了什么
- 直接引用，使用用户的原话
- 注意关键词和表达方式

## 想 (Think) - 用户在想什么
- 未说出口的想法
- 假设、信念、期望
- 担忧和顾虑

## 做 (Do) - 用户做了什么
- 具体行为
- 行为序列
- 使用的工具和环境

## 感 (Feel) - 用户感受到什么
- 情绪状态
- 情感曲线
- 高低起伏的原因

## 痛点 (Pains)
- 挫折和困难
- 障碍和阻碍
- 负面情绪来源

## 收益 (Gains)
- 期望的结果
- 成功的标准
- 正向情感来源
```

## 输出要求

### 用户画像示例

```markdown
# 用户画像: 忙碌的职场妈妈

## 基本信息
- 年龄: 32-40岁
- 职业: 中层管理
- 家庭: 已婚，有1-2个孩子

## 目标
- 高效完成工作任务
- 照顾好家庭和孩子
- 保持个人发展

## 痛点
- 时间不够用
- 愧疚感（工作vs家庭）
- 缺乏支持系统

## 行为
- 使用日程管理工具
- 利用碎片时间
- 寻求家庭帮助

## 情感
- 焦虑（害怕错过）
- 内疚（无法兼顾）
- 期望（渴望平衡）

## 引用
"我觉得自己永远在赶时间，永远做不到最好。"
```

## 研究伦理

1. **知情同意** - 向用户说明研究目的和使用方式
2. **隐私保护** - 不记录敏感个人信息
3. **尊重用户** - 用户可随时退出
4. **诚实透明** - 不隐瞒研究目的
