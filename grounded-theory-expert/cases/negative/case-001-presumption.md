# Case 001: 编码前预设结论 - 错误案例

**日期**: 2026-03-08
**分析类型**: 访谈资料扎根理论（反面教材）
**问题**: 编码前预设结论，强迫数据符合理论
**状态**: ❌ 失败
**质量评分**: 1/5

---

## ⚠️ 案例背景

### 研究情境
一个研究者想要证明"社交媒体导致青少年孤独感"

### 数据
- 样本：N=20（青少年）
- 访谈：30-60分钟
- 主题：社交媒体使用

### 真实发现
- 资料显示复杂关系
- 有些青少年通过社交媒体建立联系
- 社交媒体的影响因人而异

---

## ❌ 错误做法分析

### 错误1: 预设结论

```yaml
错误行为:
  1. 带着假设开始编码
     - 假设: "社交媒体导致孤独"
     - 只寻找支持证据
     - 忽略矛盾资料

  2. 选择性编码
     - 编码: "孤独感增加"
     - 忽略: "建立新友谊"
     - 忽略: "社交支持增加"

  3. 强行解释
     - 把正向体验解释为"表面"
     - 把社交连接解释为"虚假"
     - 强调负面，忽略正面

问题:
  - 违反扎根理论基本原则
  - 研究无效度
  - 结论不可信
```

**正确做法**:
```yaml
✅ 悬置假设
   - 不预设结论
   - 让资料说话
   - 保持开放性

✅ 全面编码
   - 编码所有内容
   - 包括矛盾的
   - 包括意外的

✅ 理论修正
   - 如果资料不支持假设
   - 修正理论
   - 或明确理论边界
```

---

### 错误2: 编码不扎根

```yaml
错误行为:
  编码示例:
    范畴: 孤独感
    概念: 社交隔离
    ❌ 没有原始引文
    ❌ 没有位置标注
    ❌ 没有参与者信息

问题:
  - 编码无法追溯
  - 无法验证
  - 不透明
```

**正确做法**:
```yaml
✅ 扎根编码
   范畴: 社交连接
   概念: 在线支持
   原始引文: "我在网上找到了志同道合的朋友..."
   参与者: 青少年7
   位置: 访谈7，第15行
   编码依据: 参与者描述获得支持
```

---

### 错误3: 忽视负面案例

```yaml
错误行为:
  1. 发现矛盾案例
     - 青少年12说: "社交媒体让我不再孤独"

  2. 编码时忽略
     - 不编码这个陈述
     - 或编码为"否认问题"

  3. 理论中不提及
     - 声称"所有青少年都感到孤独"
     - 不讨论例外情况

问题:
  - 理论有偏差
  - 缺乏效度
  - 不诚实
```

**正确做法**:
```yaml
✅ 全面编码
   - 编码所有案例
   - 包括矛盾的

✅ 寻找解释
   - 为什么有些案例不同？
   - 是否存在调节变量？
   - 理论边界在哪里？

✅ 修正理论
   - "社交媒体对孤独感的影响因人而异"
   - "对于某些青少年，社交媒体提供社交支持"
   - 明确边界条件
```

---

### 错误4: 编码数量崇拜

```yaml
错误行为:
  1. 追求更多编码
     - "我要产生100个概念"
     - 为了数量而编码

  2. 编码冗余
     - 许多编码其实是同一回事
     - 示例:
       - 社交隔离
       - 孤独感
       - 缺乏连接
       - 社交距离
       - （其实都是同一概念）

  3. 质量下降
     - 编码不精确
     - 范畴不清晰
     - 理论碎片化

问题:
  - 编码不系统
  - 范畴冗余
  - 理论不清晰
```

**正确做法**:
```yaml
✅ 关注范畴发展
   - 不只是编码数量
   - 关注范畴是否完整
   - 关注属性是否清晰
   - 关注关系是否明确

✅ 持续整合
   - 定期整合编码
   - 合并相似编码
   - 形成清晰范畴

✅ 饱和度检验
   - 无新范畴
   - 无新属性
   - 无新关系
```

---

### 错误5: 编码不一致

```yaml
错误行为:
  1. 随时间改变标准
     - 第1周: "孤独"指独处
     - 第2周: "孤独"指缺乏连接
     - 第3周: "孤独"指主观感受

  2. 相同内容不同编码
     - 参与者A说"独自在家" → 编码为"孤独"
     - 参与者B说"独自在家" → 编码为"独立"

  3. 无编码本
     - 编码定义不记录
     - 编码规则不明确
     - 无法保持一致

问题:
  - 编码不可靠
  - 无法复现
  - 信度低
```

**正确做法**:
```yaml
✅ 建立编码本
   - 记录所有编码
   - 定义每个范畴
   - 定义每个概念
   - 提供示例

✅ 定期检查
   - 检查编码一致性
   - 修正不一致编码
   - 更新编码本

✅ 使用编码本
   - 编码时参考编码本
   - 相同内容相同编码
```

## 🎯 教训总结

### 6大禁止原则违反

```yaml
1. ❌ 编码前预设结论
   - 带着假设编码
   - 只寻找支持证据
   - 忽略矛盾资料

2. ❌ 编码不扎根
   - 没有原始引文
   - 无法追溯
   - 不透明

3. ❌ 编码无理论依据
   - 编码随意
   - 范畴碎片化
   - 无法形成理论

4. ❌ 忽视负面案例
   - 选择性编码
   - 理论有偏差
   - 缺乏效度

5. ❌ 追求编码数量
   - 编码冗余
   - 质量下降
   - 理论不清晰

6. ❌ 编码不一致
   - 标准随意改变
   - 不可靠
   - 不可复现
```

## ✅ 正确分析示例

如果正确分析这个数据：

```yaml
完整分析流程:

Step 1: 悬置假设
  - 不预设"社交媒体导致孤独"
  - 保持开放性
  - 让资料说话

Step 2: 全面编码
  - 编码所有内容
  - 包括正面体验
  - 包括负面体验

Step 3: 识别范畴
  概念:
    - 在线连接
    - 社交支持
    - 孤独感
    - 错失恐惧(FOMO)
    - 社交比较

  范畴:
    - 社交媒体的双刃剑
    - 个体差异
    - 使用方式
    - 影响因素

Step 4: 识别关系
  - 使用方式调节影响
  - 个体差异导致不同体验
  - 社交媒体既是连接工具也是隔离源

Step 5: 形成理论
  "社交媒体对青少年社交幸福感的影响因人而异，
   取决于使用方式和个体特征。"
```

## 📊 对比

| 维度 | ❌ 错误做法 | ✅ 正确做法 |
|------|-----------|-----------|
| **假设** | 预设结论 | 悬置假设 |
| **编码** | 选择性编码 | 全面编码 |
| **扎根性** | 无原始引文 | 每个编码有引文 |
| **负面案例** | 忽略 | 编码并解释 |
| **编码质量** | 追求数量 | 关注范畴 |

## 🎓 学习要点

### 关键教训

1. **开放性最重要**
   - ❌ 带着假设是致命错误
   - ✅ 让资料说话

2. **全面编码**
   - ❌ 选择性编码产生偏差
   - ✅ 所有内容都编码

3. **扎根性**
   - ❌ 编码无依据不可信
   - ✅ 每个编码有引文

4. **诚实面对数据**
   - ❌ 忽略矛盾是不诚实
   - ✅ 面对并解释负面案例

5. **质量>数量**
   - ❌ 追求编码数量
   - ✅ 关注范畴发展

6. **编码一致性**
   - ❌ 随意改变标准
   - ✅ 编码本管理

## 🔍 识别警示信号

### 可疑分析的信号

```yaml
🚩 警示信号:
  - 所有资料都支持假设
  - 没有矛盾案例
  - 编码无原始引文
  - 编码数量巨大但范畴不清
  - 理论"完美"
  - 负面案例被解释为"例外"或"否认"

如何验证:
  - 要求查看编码本
  - 要求查看原始引文
  - 要求解释负面案例
  - 检查编码一致性
```

## 📈 改进建议

### 给研究者的建议

```yaml
预防预设结论:
  1. 研究前反思
     - 明确自己的假设
     - 意识到潜在偏见
     - 准备挑战假设

  2. 保持开放
     - 持续问: "资料在说什么？"
     - 不只是"证据支持什么？"
     - 准备被意外发现

  3. 同行评审
     - 让他人检查编码
     - 征求批评意见
     - 质疑假设

  4. 诚实面对
     - 报告所有发现
     - 包括矛盾的
     - 修正理论
```
