## 🎯 高频分析模式

### Pattern 1: 访谈资料扎根理论分析

**频率**: 高频 (20次/年)
**成功率**: 92%
**平均时长**: 4-6个月

#### 标准分析流程

```yaml
Phase 1: 数据收集与准备（1个月）
  Task 1.1: 访谈设计（1周）
    - 确定研究问题
    - 设计访谈提纲
    - 获得伦理批准

  Task 1.2: 访谈执行（2-3周）
    - 进行深度访谈
    - 录音转录
    - 资料整理

  Task 1.3: 初步阅读（1周）
    - 熟悉资料
    - 记录初步印象
    - 识别关键主题

Phase 2: 开放编码（1-2个月）
  Task 2.1: 逐行编码（2-4周）
    - 每行都编码
    - 保持开放性
    - 持续比较

  Task 2.2: 识别概念（1-2周）
    - 整合相似编码
    - 命名概念
    - 记录定义

  Task 2.3: 形成范畴（1-2周）
    - 整合概念
    - 识别范畴
    - 建立范畴定义

Phase 3: 轴心编码（1-2个月）
  Task 3.1: 范畴联结（2-4周）
    - 识别范畴关系
    - 建立因果链
    - 发展范畴

  Task 3.2: 撰写备忘录（2-4周）
    - 编码备忘录
    - 理论备忘录
    - 整合备忘录

  Task 3.3: 范畴完善（1-2周）
    - 识别属性
    - 明确维度
    - 补充案例

Phase 4: 选择式编码（1-2个月）
  Task 4.1: 核心范畴识别（2-4周）
    - 识别中心范畴
    - 验证整合力
    - 明确故事线

  Task 4.2: 理论整合（2-4周）
    - 整合所有范畴
    - 构建理论框架
    - 撰写理论草稿

  Task 4.3: 饱和度检验（1-2周）
    - 收集额外资料
    - 验证理论饱和
    - 修正理论

Phase 5: 理论撰写（1个月）
  Task 5.1: 理论撰写（2周）
    - 撰写完整理论
    - 提供案例支撑
    - 讨论理论贡献

  Task 5.2: 同行评审（1-2周）
    - 征求反馈
    - 修改完善
    - 准备发表
```

#### 质量检查清单

```yaml
数据质量:
  - [ ] 资料充分（20+访谈）
  - [ ] 转录准确
  - [ ] 资料多样性

编码质量:
  - [ ] 所有资料已编码
  - [ ] 编码本完整
  - [ ] 编码有原始引文
  - [ ] 编码定义清晰

范畴质量:
  - [ ] 范畴有定义
  - [ ] 范畴有属性
  - [ ] 范畴有维度
  - [ ] 范畴有案例

理论质量:
  - [ ] 核心范畴识别
  - [ ] 故事线连贯
  - [ ] 理论整合
  - [ ] 饱和度检验
  - [ ] 负面案例解释
```

#### 成功因素

✅ **开放性**
- 不预设结论
- 保持理论敏感性
- 让资料说话

✅ **系统性**
- 逐行编码
- 持续比较
- 备忘录记录

✅ **理论性**
- 基于文献
- 理论整合
- 贡献新知识

✅ **严谨性**
- 编码可追溯
- 理论可验证
- 饱和度检验

**频率**: 中频 (10次/年)
**成功率**: 88%
**平均时长**: 3-5个月

#### 与访谈资料的区别

```yaml
观察资料特点:

     - 需要描述行为
     - 关注情境互动
     - 记录时间序列

  2. 情境依赖
     - 场所重要
     - 互动关系
     - 环境因素

  3. 研究者参与
     - 参与观察
     - 反思性
     - 主观性

编码重点:
  - 行为序列
  - 互动模式
  - 情境因素
  - 非言语线索
```

### Pattern 3: 文档资料扎根理论分析

**频率**: 中频 (8次/年)

```yaml
文档资料特点:

  - 文档类型
  - 叙事结构
  - 官方话语
  - 权力关系
```

## 🔍 编码模式

### 开放编码模式

```yaml
逐步编码:
  1. 逐行阅读
  2. 标记关键短语
  3. 生成初始编码
  4. 持续比较
  5. 整合相似编码
  6. 命名概念
  7. 形成范畴

常见问题:
  ❌ 编码太粗糙
  ❌ 编码太细碎
  ❌ 没有原始引文
  ❌ 编码不一致

成功因素:
  ✅ 每行都编码
  ✅ 记录原始引文
  ✅ 保持编码一致性
  ✅ 定期整合编码
```

### 轴心编码模式

```yaml
范畴发展:
  1. 识别范畴
  2. 定义范畴
  3. 识别属性
  4. 明确维度
  5. 建立关系
  6. 发展范畴

关系类型:
  - 因果关系
  - 条件关系
  - 策略关系
  - 时间关系
  - 结构关系

常见问题:
  ❌ 范畴关系不明确
  ❌ 维度不清晰
  ❌ 缺少案例支撑
  ❌ 关系复杂化

成功因素:
  ✅ 范畴定义清晰
  ✅ 属性维度明确
  ✅ 关系合理
  ✅ 有案例支撑
```

### 选择式编码模式

```yaml
理论整合:
  1. 识别核心范畴
  2. 验证整合力
  3. 构建故事线
  4. 整合所有范畴
  5. 形成理论

核心范畴标准:
  - 频繁出现
  - 中心地位
  - 整合力强
  - 解释力强
  - 理论贡献大

常见问题:
  ❌ 核心范畴选择错误
  ❌ 故事线不连贯
  ❌ 理论碎片化
  ❌ 缺乏理论贡献

成功因素:
  ✅ 核心范畴明确
  ✅ 故事线连贯
  ✅ 理论整合
  ✅ 有理论贡献
```

## 📊 范畴发展模式

### 常见范畴类型

```yaml
行为范畴:
  - 人们的行动
  - 策略选择
  - 互动模式
  例: "适应策略"、"抵抗行为"

条件范畴:
  - 结构条件
  - 情境因素
  - 约束条件
  例: "资源约束"、"制度环境"

后果范畴:
  - 行为后果
  - 结果影响
  - 意义建构
  例: "身份重构"、"权力转移"

过程范畴:
  - 时间维度
  - 阶段划分
  - 演化路径
  例: "变革过程"、"学习轨迹"
```

### 范畴发展策略

```yaml
自下而上:
  从具体编码开始
  → 识别相似模式
  → 形成概念
  → 整合为范畴

自上而下:
  从理论概念开始
  → 在资料中寻找
  → 修正概念
  → 验证范畴

混合方法（推荐）:
  理论敏感性
  → 开放编码
  → 持续比较
  → 理论整合
```

## 🎓 理论生成模式

### 理论类型

```yaml
实质性理论 (Substantive Theory):
  - 特定领域
  - 实践导向
  - 中层理论
  例: "医院临终关怀理论"

形式理论 (Formal Theory):
  - 跨领域
  - 抽象程度高
  - 普适性强
  例: "身份转换理论"
```

### 理论评估标准

```yaml
拟合性 (Fit):
  - 解释资料
  - 案例支撑
  - 无遗漏

可理解性 (Understandability):
  - 逻辑清晰
  - 概念明确
  - 故事连贯

概括性 (Generality):
  - 适用范围
  - 跨情境适用
  - 可迁移性

控制力 (Control):
  - 预测能力
  - 解释变异
  - 行动指导

理论贡献:
  - 新概念
  - 新关系
  - 新视角
  - 新理论
```

## 💡 经验教训

### 成功经验

✅ **持续比较是关键**
   - 不断比较编码
   - 不断比较范畴
   - 不断比较与理论

✅ **备忘录帮助思考**
   - 记录编码发现
   - 记录理论想法
   - 记录分析过程

✅ **理论敏感性重要**
   - 保持开放性
   - 理论指导但不预设
   - 文献与资料互动

### 常见陷阱

❌ **过早停止编码**
   - 编码不充分
   - 范畴不完整
   - 理论不饱和

❌ **忽视负面案例**
   - 理论有偏差
   - 边界不明确
   - 效度有问题

❌ **追求编码数量**
   - 编码冗余
   - 理论不清
   - 质量下降
