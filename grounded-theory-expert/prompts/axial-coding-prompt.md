# Axial Coding Prompt - 轴心编码

## 任务描述

执行扎根理论的轴心编码分析，在开放性编码基础上建立范畴间的联系，构建编码范式模型。

## 轴心编码原则

### 1. 建立联系
- 识别范畴间的因果关系
- 发现脉络和情境条件
- 识别干预条件
- 追踪行动策略
- 记录结果

### 2. 使用范式模型
 Strauss & Corbin 的范式模型包含 6 个要素：
1. **因果条件** - 导致现象发生的因素
2. **核心现象** - 研究的中心事件或问题
3. **脉络** - 情境条件（时间、地点、文化）
4. **干预条件** - 促进或阻碍行动的因素
5. **行动/互动策略** - 个体或群体的应对行为
6. **结果** - 行动的后果

### 3. 多层次分析
- 主范畴（核心范畴）
- 副范畴（次要范畴）
- 属性（范畴的特征）
- 维度（属性的变化范围）

## 编码步骤

### Step 1: 范畴聚类
将开放性编码中的概念按主题聚类，形成范畴。

### Step 2: 识别关系
对每个范畴：
1. 问："这个范畴与其他范畴有什么关系？"
2. 识别因果关系
3. 识别脉络关系
4. 识别干预关系

### Step 3: 构建范式模型
1. 确定核心现象
2. 识别因果条件
3. 描述脉络条件
4. 识别干预条件
5. 记录行动策略
6. 追踪结果

### Step 4: 定义属性和维度
对每个范畴：
1. 列出属性（特征）
2. 定义维度（变化范围）
3. 提供数据示例

## 输出格式

```json
{
  "categories": [
    {
      "name": "范畴名称",
      "subcategories": ["副范畴 1", "副范畴 2"],
      "properties": [
        {
          "name": "属性名称",
          "dimensions": ["维度 1", "维度 2"]
        }
      ],
      "data_examples": ["示例 1", "示例 2"]
    }
  ],
  "paradigm_model": {
    "causal_conditions": ["条件 1", "条件 2"],
    "phenomenon": "核心现象",
    "context": ["脉络 1", "脉络 2"],
    "intervening_conditions": ["干预条件 1", "干预条件 2"],
    "strategies": ["策略 1", "策略 2"],
    "consequences": ["结果 1", "结果 2"]
  },
  "relationships": [
    {
      "from": "范畴 A",
      "to": "范畴 B",
      "type": "causal|contextual|intervening|sequential",
      "description": "关系描述",
      "data_support": ["数据示例"]
    }
  ]
}
```

## 质量检查

轴心编码完成后，检查：
- [ ] 范畴体系层级清晰（至少 2 层）
- [ ] 每个范畴都有属性和维度
- [ ] 因果关系有数据支持
- [ ] 范式模型 6 要素完整
- [ ] 关系网络逻辑一致

## 示例

### 范畴示例
```json
{
  "name": "应对策略",
  "subcategories": ["主动应对", "被动应对", "回避应对"],
  "properties": [
    {
      "name": "应对强度",
      "dimensions": ["高强度", "中强度", "低强度"]
    },
    {
      "name": "应对时机",
      "dimensions": ["即时", "延迟"]
    }
  ],
  "data_examples": [
    "我就直接去找领导谈了（主动应对）",
    "我想先看看情况再说（被动应对）"
  ]
}
```

### 范式模型示例
```json
{
  "causal_conditions": ["工作压力大", "资源不足"],
  "phenomenon": "职业倦怠",
  "context": ["互联网行业", "一线城市", "35 岁以下"],
  "intervening_conditions": ["社会支持", "个人韧性"],
  "strategies": ["寻求支持", "调整期望", "离职倾向"],
  "consequences": ["心理健康下降", "工作绩效下降", "离职"]
}
```

## 注意事项

1. **不要过度简化**: 因果关系可能很复杂
2. **允许多重因果**: 一个结果可能有多个原因
3. **关注过程**: 行动策略是动态的
4. **记录例外**: 注意不符合模式的案例
