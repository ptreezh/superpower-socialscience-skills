# Open Coding Prompt - 开放性编码

## 任务描述

执行扎根理论的开放性编码分析，将文本数据分解为独立的概念单元。

## 编码原则

### 1. 保持开放
- 悬置先入为主的观念
- 让概念从数据中自然涌现
- 避免强制归类

### 2. 贴近数据
- 使用受访者原话（in vivo 编码）
- 保持概念与数据的紧密联系
- 每个编码必须有数据支持

### 3. 行动导向命名
- 使用动词 + 名词的格式
- 例如："寻求支持"、"规避风险"、"建构身份"
- 避免静态的名词

### 4. 细致分析
- 逐行或逐段分析
- 不遗漏重要信息
- 允许矛盾共存

## 编码步骤

### Step 1: 数据分段
将文本按意义单元分段，每段表达一个完整的意思。

### Step 2: 逐段编码
对每个分段：
1. 仔细阅读 2-3 遍
2. 问自己："这段数据在说什么？"
3. 提取 1-3 个核心概念
4. 为每个概念命名

### Step 3: 概念定义
对每个编码：
1. 写出清晰的定义（50-100 字）
2. 提供 2-3 个数据示例
3. 记录出现频次

### Step 4: 生成备忘录
对重要或有趣的编码：
1. 记录分析洞见
2. 联想相关理论
3. 提出研究问题

## 输出格式

```json
{
  "codes": [
    {
      "name": "编码名称",
      "definition": "编码定义",
      "data_examples": [
        "原始数据引用 1",
        "原始数据引用 2"
      ],
      "frequency": 5,
      "memo": "相关备忘录"
    }
  ],
  "reliability": {
    "cohens_kappa": 0.75,
    "agreement_rate": 0.85,
    "status": "acceptable"
  },
  "memos": [
    {
      "type": "code",
      "content": "备忘录内容",
      "timestamp": "2026-03-03T10:00:00Z"
    }
  ]
}
```

## 质量检查

编码完成后，检查：
- [ ] 每个编码都有明确定义
- [ ] 每个编码至少有 2 个数据示例
- [ ] 编码命名是行动导向的
- [ ] 没有遗漏重要数据
- [ ] 允许矛盾编码共存

## 示例

### 数据片段
> "我当时很困惑，不知道该怎么办。后来我想到了找朋友聊聊，他们给了我很多建议。"

### 编码结果
```json
{
  "name": "寻求社会支持",
  "definition": "个体在面临困境时主动向他人寻求帮助和建议的行为",
  "data_examples": [
    "我当时很困惑，不知道该怎么办。后来我想到了找朋友聊聊",
    "我就打电话给我最好的朋友，她总是能给我好的建议"
  ],
  "frequency": 3,
  "memo": "这似乎是一个重要的应对策略，可能与后续的决策质量有关"
}
```

## 注意事项

1. **不要过度抽象**: 保持与数据的联系
2. **不要过早归类**: 开放性编码阶段允许概念重叠
3. **不要忽略负面案例**: 记录与主流模式不符的数据
4. **记录分析过程**: 写下你的思考和疑问
