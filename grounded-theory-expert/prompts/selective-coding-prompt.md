# Selective Coding Prompt - 选择式编码

## 任务描述

执行扎根理论的选择式编码分析，确定核心范畴，整合理论框架，构建故事线，生成理论命题。

## 选择式编码原则

### 1. 核心范畴的特征
核心范畴必须满足以下条件：
- **中心性**: 与其他所有范畴都有联系
- **解释力**: 能解释大部分数据变异
- **整合力**: 能整合其他范畴
- **频繁出现**: 在数据中频繁出现
- **理论相关性**: 有理论意义

### 2. 故事线构建
- 用一句话概括研究的核心发现
- 故事线应该连贯、简洁
- 包含现象、行动、结果

### 3. 理论命题
- 命题应该是可验证的陈述
- 命题之间应该有逻辑联系
- 命题应该能解释现象

## 编码步骤

### Step 1: 确定核心范畴
1. 列出所有主要范畴
2. 评估每个范畴的整合力
3. 选择最具整合力的范畴
4. 验证核心范畴的标准

### Step 2: 整合理论框架
1. 将核心范畴与其他范畴联系
2. 建立系统的理论框架
3. 检查框架的逻辑一致性

### Step 3: 构建故事线
1. 用一句话概括核心发现
2. 描述现象→行动→结果的链条
3. 确保故事线连贯

### Step 4: 生成理论命题
1. 基于理论框架生成命题
2. 确保命题可验证
3. 提供数据支持

## 输出格式

```json
{
  "core_category": {
    "name": "核心范畴名称",
    "definition": "核心范畴定义",
    "integration_power": 9,
    "explanatory_power": 8,
    "selection_rationale": "选择理由",
    "data_examples": ["示例 1", "示例 2"]
  },
  "theoretical_framework": {
    "description": "理论框架描述",
    "categories_relationship": [
      {
        "category": "范畴名称",
        "relationship_to_core": "与核心范畴的关系"
      }
    ]
  },
  "story_line": "故事线内容（200-500 字）",
  "propositions": [
    {
      "id": "P1",
      "statement": "命题陈述",
      "supporting_evidence": ["数据示例 1", "数据示例 2"],
      "theoretical_basis": "理论基础"
    }
  ]
}
```

## 质量检查

选择式编码完成后，检查：
- [ ] 核心范畴满足所有标准
- [ ] 核心范畴整合力强（评分≥8/10）
- [ ] 故事线连贯、简洁
- [ ] 理论命题可验证
- [ ] 命题有数据支持
- [ ] 框架逻辑一致

## 示例

### 核心范畴示例
```json
{
  "name": "身份重构",
  "definition": "个体在面临重大生活转变时，通过意义建构和行为调整，重新建立自我认同的过程",
  "integration_power": 9,
  "explanatory_power": 8,
  "selection_rationale": "该范畴能整合所有其他范畴，解释了受访者在失业后的适应过程，在数据中出现频率最高，具有重要的理论意义",
  "data_examples": [
    "我开始重新思考自己到底想要什么",
    "失业让我有机会重新定义成功"
  ]
}
```

### 故事线示例
> 本研究的核心发现是：个体在面临失业这一重大生活事件时，会经历一个"身份重构"的过程。这一过程始于失业冲击带来的身份危机（因果条件），在家庭支持和社会网络的影响下（干预条件），个体通过意义建构、技能提升、社交拓展等策略（行动策略），逐步重建自我认同（结果）。整个过程受到个人韧性、经济压力、社会支持等因素的调节（脉络条件）。

### 理论命题示例
```json
{
  "id": "P1",
  "statement": "社会支持越强，个体身份重构的速度越快",
  "supporting_evidence": [
    "我有家人朋友的支持，所以很快就走出来了",
    "要是没有我老婆，我可能现在还消沉呢"
  ],
  "theoretical_basis": "社会支持理论"
}
```

## 注意事项

1. **避免过度概括**: 命题应该有数据支持
2. **承认局限性**: 说明理论的适用范围
3. **保持开放性**: 允许后续研究修正
4. **反身性**: 说明研究者的立场和假设
