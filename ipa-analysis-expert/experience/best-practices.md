# IPA分析最佳实践

## 版本信息
- **创建时间**: 2026-03-15
- **方法论标准**: Smith & Osborn (2015), Smith et al. (2009)

---

## 一、研究设计阶段

### 1.1 研究问题设计

**好的IPA研究问题特征**：
- 关注"如何体验"
- 关注"意味着什么"
- 关注"如何理解"

**示例对比**：

| 好的问题 | 不好的问题 |
|---------|-----------|
| 抑郁症患者如何体验日常生活？ | 抑郁症患者有什么症状？ |
| 成为父母对男性意味着什么？ | 男性育儿的障碍有哪些？ |
| 慢性疼痛患者如何理解"康复"？ | 哪些因素影响康复效果？ |

### 1.2 抽样策略

**同质样本原则**：
- 经验相似性（都经历过X）
- 人口学相似性有助于深入比较
- 但也要考虑变异的必要性

**样本量决策**：
| 研究目的 | 建议样本 | 理由 |
|---------|---------|------|
| 探索性研究 | 3-6 | 初步了解 |
| 硕士论文 | 6-8 | 深度与可行性平衡 |
| 博士论文 | 10-15 | 深度与广度兼顾 |
| 发表研究 | 6-10 | 深度优先 |

## 二、数据收集阶段

### 2.1 访谈技巧

**建立关系的策略**：
- 开场说明研究目的
- 强调没有"正确"答案
- 表达真正的兴趣
- 保持非评判态度

**追问技术**：

| 技术 | 示例 | 目的 |
|------|------|------|
| 澄清 | "您能详细说说吗？" | 明确含义 |
| 深化 | "那时候您有什么感受？" | 情感维度 |
| 具体化 | "能举个例子吗？" | 具体情境 |
| 意义追问 | "这对您意味着什么？" | 意义建构 |

### 2.2 访谈时长

- 建议：60-90分钟
- 过短：深度不足
- 过长：参与者疲劳
- 灵活：根据参与者状态调整

### 2.3 转录要点

**必须保留**：
- 原话（逐字）
- 停顿和沉默
- 非语言表达
- 情感表达

**转录符号系统**：
```
(3s)      - 3秒停顿
(.)       - 短暂停顿
[叹气]    - 非语言表达
[笑]      - 笑声
...       - 犹豫
"..."     - 强调
(??)      - 听不清
```

## 三、分析阶段

### 3.1 单案例分析顺序

```
1. 完整阅读（不标记）
      ↓
2. 重读并记录印象
      ↓
3. 逐行注释（三类并行）
      ↓
4. 发展主题
      ↓
5. 构建主从结构
      ↓
6. 整理主题表
```

### 3.2 注释实践要点

**描述性注释**：
- 用参与者语言
- 不添加诠释
- 关注"说了什么"

**语言性注释**：
- 关注隐喻
- 注意停顿和犹豫
- 分析强调和重复

**概念性注释**：
- 使用心理学术语
- 明确标注研究者视角
- 连接理论概念

### 3.3 主题发展原则

**从注释到主题**：
```
注释 → 新兴主题 → 心理学术语命名

示例：
注释："被困在盒子里"
    ↓
新兴主题："被困的感觉"
    ↓
心理学术语："受限自我的体验"
```

**命名要求**：
- 使用心理学术语
- 反映经验本质
- 与原文有清晰连接

### 3.4 主从结构构建

**主主题特征**：
- 覆盖经验的核心维度
- 有多个从主题支撑
- 有理论意义

**从主题特征**：
- 具体化主主题
- 有原文引语支撑
- 呈现经验细节

## 四、跨案例分析

### 4.1 分析原则

**必须遵守**：
- 每个案例单独分析
- 不预设主题
- 允许新主题涌现
- 保持个体差异

### 4.2 模式识别

**共同模式**：
- 多个参与者出现的主题
- 核心经验维度

**个体差异**：
- 每个参与者的独特性
- 经验轨迹的不同

### 4.3 整合策略

```
案例1主题 ←→ 案例2主题 ←→ 案例3主题
     ↓              ↓              ↓
     └──────────────┴──────────────┘
                    ↓
             共同主题模式
             + 个体差异展示
```

## 五、质量保证

### 5.1 Yardley四原则

| 原则 | 具体体现 | 如何实现 |
|------|---------|---------|
| 环境敏感性 | 情境描述充分 | 详细描述参与者背景和访谈情境 |
| 承诺与严谨 | 深入分析 | 逐行注释、案例展示 |
| 透明性与连贯性 | 过程清晰 | 方法描述详细、论证有据 |
| 影响与重要性 | 贡献明确 | 理论对话、实践意义 |

### 5.2 技术策略

**成员检验**：
- 返回转录稿确认
- 返回分析结果反馈
- 根据反馈修正

**同侪审查**：
- 独立研究者审查部分样本
- 比较主题发现
- 讨论分歧

**审计追踪**：
- 保存所有分析记录
- 记录决策过程
- 展示主题发展轨迹

### 5.3 自反性实践

**自反日志内容**：
```
日期: [日期]
分析内容: [案例/段落]
我的反应: [...]
可能的影响: [...]
需要进一步探索: [...]
```

## 六、常见问题

### 6.1 主题太多怎么办？

**解决策略**：
- 尝试合并相似主题
- 识别主从关系
- 聚焦核心经验
- 确保每个主题有充分证据

### 6.2 分析时"卡住"怎么办？

**解决策略**：
- 回到原文重读
- 记录困惑作为分析的一部分
- 尝试不同的主题命名
- 寻求同侪讨论

### 6.3 如何处理矛盾数据？

**正确做法**：
- 承认矛盾存在
- 不要强行统一
- 探索矛盾的意义
- 可能是经验的复杂性

### 6.4 研究者偏见如何处理？

**策略**：
- 承认而非消除偏见
- 使用自反日志
- 同侪审查
- 成员检验

## 七、报告撰写

### 7.1 引语使用原则

- 大量使用原文引语
- 引语与分析交织
- 引语支撑主题
- 标注参与者编号

### 7.2 展示分析过程

**必要内容**：
- 注释示例
- 主题发展轨迹
- 分析决策说明
- 研究者反思

### 7.3 个体差异呈现

- 使用表格展示
- 在叙述中提及
- 保持与共同主题的对话
