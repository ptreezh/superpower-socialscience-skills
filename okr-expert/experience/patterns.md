# OKR经验模式

## 已识别的模式

### 模式1: 目标数字化

**现象**：目标中包含数字，变成"KR型目标"

**根因**：
- 习惯性使用KPI思维
- 对"激励性"理解不足
- 难以提炼定性描述

**解决方案**：
- 目标必须回答"我们要成为什么"
- 使用"动词+名词"格式
- 删除所有数字，保留激励性

**示例**：
```
❌ 目标：提高客户满意度到90%
✅ 目标：打造让客户爱不释手的产品体验
```

### 模式2: 关键结果过多

**现象**：一个目标关联10+关键结果，失去焦点

**根因**：
- 不愿放弃任何指标
- 对"关键"理解偏差
- 缺乏优先级判断

**解决方案**：
- 严格限制3-5个KR
- 使用MoSCoW优先级法
- 移除的KR放入观察列表

### 模式3: OKR与绩效挂钩

**现象**：OKR完成度直接影响绩效考核和奖金

**根因**：
- 管理层短视
- 缺乏信任文化
- 混淆OKR与KPI

**解决方案**：
- 明确OKR不用于绩效考核
- 使用单独的绩效评估机制
- 建立"挑战性目标"文化

### 模式4: 瀑布式对齐

**现象**：等待上级OKR确定后才能制定下级OKR，导致延迟

**根因**：
- 过度强调垂直对齐
- 忽视自下而上创新
- 审批流程冗长

**解决方案**：
- 70%自上而下 + 30%自下而上
- 使用OKR草稿并行制定
- 建立快速对齐机制

## 成功模式

### 成功模式1: 公开透明

**特征**：
- 全员可见所有OKR
- 定期公开进度更新
- 鼓励跨团队协作

### 成功模式2: 周检查机制

**特征**：
- 每周检查OKR进度
- 及时识别障碍
- 快速调整行动

### 成功模式3: 季度复盘

**特征**：
- 季度末全面评分
- 分析达成/未达成原因
- 提取经验教训

## 失败模式

### 失败模式1: 设后不管

**表现**：季度初设定OKR后，直到季度末才查看

**预防**：
- 强制每周检查机制
- 自动进度提醒
- OKR仪表盘可视化

### 失败模式2: 橡皮筋目标

**表现**：目标范围不断变化，难以评估完成度

**预防**：
- 季度中不允许大幅修改
- 变更需正式流程
- 保留历史版本对比

### 失败模式3: 保守目标

**表现**：所有目标都是100%可达成，失去激励作用

**预防**：
- 强制60-70%完成率预期
- 区分承诺型OKR和挑战型OKR
- 奖励尝试而非仅奖励成功
