# OKR经典文献

## 核心文献

### 1. John Doerr - Measure What Matters (2018)
**《这就是OKR：让谷歌、亚马逊实现爆炸性增长的工作法》**

核心概念：
- Objectives：目标（定性、激励性）
- Key Results：关键结果（定量、可度量）
- Alignment：对齐（垂直对齐、水平对齐）
- Transparency：透明（全员可见）
- Stretch Goals：延伸目标（挑战性）

引用规范：
> "Ideas are easy. Execution is everything." - John Doerr

### 2. Andy Grove - High Output Management (1983)
**《高产出管理》**

OKR的历史根源：
- Intel的i386项目首次应用OKR
- Andy Grove被称为"OKR之父"
- John Doerr在Intel学习OKR

### 3. Rick Klau - How Google Sets Goals (2013)
**Google OKR实践指南**

关键要点：
- 60-70%完成率是理想状态
- OKR不是绩效考核工具
- 公开透明是核心

## OKR框架详解

### 目标 (Objectives)

特征：
- **定性的**：不包含数字
- **激励性的**：鼓舞人心
- **有时间限制**：通常季度
- **团队可执行**：团队可控

示例：
```
❌ 目标：提高用户满意度到90%（包含数字）
✅ 目标：打造让用户爱不释手的产品体验
```

### 关键结果 (Key Results)

特征：
- **定量的**：必须有数字
- **可度量**：有明确衡量标准
- **有挑战性**：延伸目标
- **3-5个**：每个目标

示例：
```
✅ KR1：用户NPS评分从45提升到60
✅ KR2：日活跃用户增长30%
✅ KR3：用户流失率降低至5%以下
```

## OKR评分系统

### 评分标准

| 分数 | 状态 | 含义 |
|------|------|------|
| 0.0-0.3 | 红色 | 未达成，需要分析原因 |
| 0.4-0.6 | 黄色 | 有进展，但未达预期 |
| 0.7-0.9 | 绿色 | 良好达成，目标设置合理 |
| 1.0 | 蓝色 | 可能过于简单，考虑提高 |

### 理想完成率

- 目标：0.6-0.7 (60-70%)
- 1.0意味着目标太简单
- 0.4以下需要分析原因

## OKR与KPI的区别

| 特征 | OKR | KPI |
|------|-----|-----|
| 性质 | 挑战目标 | 绩效指标 |
| 完成度 | 60-70%理想 | 100%期望 |
| 用途 | 目标管理 | 绩效考核 |
| 透明度 | 全员公开 | 通常保密 |
| 频率 | 季度 | 月度/季度 |

## OKR实施周期

```
季度前2周：OKR制定
季度中：周检查、月回顾
季度后：评分与回顾
```
