## 🎯 核心优化内容

### 1. 绝对禁止原则（6项）- MSQCA特化

| # | 禁止项 | 核心要求 | 量化标准 |
|---|--------|---------|---------|
| 1 | 机械校准 | 理论指导 | 每个条件有明确校准标准 |
| 2 | 忽视案例独特性 | 回到案例 | 每个组合的案例列表 |
| 3 | 过度简化 | 保守最小化 | 至少2种解 |
| 4 | 忽视时间维度 | 时序明确 | 条件先于结果 |
| 5 | 忽视稳健性 | 检验敏感性 | 至少3种参数、2种校准 |
| 6 | 统计思维套用 | 集合论思维 | 使用集合论语言 |

### 2. 任务分解规则（4原则）

```yaml
分解原则:
  1. 粒度可控: 每个子任务必须在一次会话中完成
  2. 量化标准: 每个子任务必须有明确的完成标准
  3. 独立验证: 每个子任务完成后必须独立验证
  4. 数据清单: 子 agent 必须输出完整的数据清单
```

### 3. 渐进式加载结构（3层）

```
第一层：核心执行规则（SKILL.md）

第三层：案例文档（cases/）
  - positive/: 正确示范
  - negative/: 错误警示
```

## 📚 知识贡献

### 方法学贡献

1. **集合论因果分析框架**
   - 多重并发因果（多个条件组合）
   - 非对称因果（X→Y ≠ Y→X）
   - 等效性（多条路径）

2. **理论指导的校准方法**
   - 基于理论的锚点设定
   - 校准函数（logistic/logit）
   - 跨研究可比性

3. **案例导向的分析**
   - 回到具体案例
   - 过程追踪验证
   - 跨案例比较

4. **稳健性检验标准**
   - 参数敏感性（3种设置）

   - 明确稳健性/脆弱性
