# QCA分析模式与经验总结

## QCA三大分析方法

### 1. csQCA (crisp-set QCA)

**特征**: 二元集合（0或1）

**适用场景**:
- 明确的二分变量
- 制度分析（存在/不存在）
- 政策采用（是/否）

**示例**: 民主转型研究
```yaml
条件:
  C1: 经济发展 (0=低, 1=高)
  C2: 中产阶级 (0=弱, 1=强)
  C3: 外部压力 (0=无, 1=有)

结果:
  Y: 民主转型 (0=否, 1=是)

解:
  民主转型 = C1*C2 + C1*C3
  解释:
    - 路径1: 经济发展 + 中产阶级
    - 路径2: 经济发展 + 外部压力
```

**优势**:
- 简单直观
- 易于解释
- 适合小样本

**局限**:
- 损失信息（连续变量→二元）
- 校准主观性

---

### 2. mvQCA (multi-value QCA)

**特征**: 多值集合（超过两个类别）

**适用场景**:
- 多类别变量
- 政体类型（威权/混合/民主）
- 政策类型（激进/温和/保守）

**示例**: 政策类型分析
```yaml
条件:
  C1: 政体类型 (0=威权, 0.5=混合, 1=0民主)
  C2: 经济水平 (0=低, 0.5=中, 1=0高)
  C3: 社会动员 (0=低, 0.5=中, 1=0高)

结果:
  Y: 政策类型 (0=紧缩, 0.5=维持, 1=0扩张)

解:
  扩张政策 = C1[1]*C3[1] + C1[0.5]*C2[1]
  解释:
    - 路径1: 民主 + 高动员
    - 路径2: 混合政体 + 高经济水平
```

**优势**:
- 保留更多类别信息
- 适合多类别现象
- 比 csQCA 更精细

**局限**:
- 类别划分主观性
- 不如 fsQCA 精确

---

### 3. fsQCA (fuzzy-set QCA)

**特征**: 模糊集（连续隶属度 0.0-1.0）

**适用场景**:
- 连续变量
- 程度差异
- 最常用、最灵活

**示例**: 社会运动成功
```yaml
条件:
  C1: 政治机会 (0.0-1.0)
  C2: 资源动员 (0.0-1.0)
  C3: 文化框架 (0.0-1.0)

结果:
  Y: 运动成功 (0.0-1.0)

校准:
  使用 logistic 函数
  锚点: 0.0, 0.5, 1.0

解:
  成功 = C1*C3 + C2*C3
  解释:
    - 路径1: 政治机会 + 文化框架
    - 路径2: 资源动员 + 文化框架
```

**优势**:
- 保留连续信息
- 校准灵活
- 适合大多数研究

**局限**:
- 校准主观性
- 结果解释复杂

## 真值表分析模式

### 模式1: 有限多样性问题

**问题**: 2^k 个逻辑组合，有些组合没有案例

**示例** (4个条件 → 16个组合):
```yaml
理论组合: 16
实际案例: 20
空组合: 5 (无案例)

问题:
  - 如何处理空组合?
  - 是否使用逻辑余项?
```

**处理策略**:
1. **保守策略**: 不使用逻辑余项
   - 只分析有案例的组合
   - 结果更保守

2. **激进策略**: 使用所有逻辑余项
   - 简化表达式
   - 结果可能过度简化

3. **中间策略**: 只使用合理的逻辑余项
   - 基于理论判断
   - 平衡简洁与准确性

### 模式2: 逻辑矛盾案例

**问题**: 同一条件组合，既有成功也有失败

**示例**:
```yaml
组合: C1*C2*C3 = 1

案例:
  案例1: 成功 (Y=1)
  案例2: 失败 (Y=0)
  案例3: 成功 (Y=1)

一致性: 0.67 (低于阈值 0.75)

处理:
  选项1: 降低一致性阈值
  选项2: 删除该组合
  选项3: 深入分析矛盾案例
```

**最佳实践**:
- 首选: 深入分析矛盾案例
- 理解为什么相同组合有不同结果
- 可能遗漏了重要条件

## 校准与阈值设定模式

### 模式1: 理论驱动校准

**原则**: 基于理论和知识设定锚点

**示例**: "经济发展水平"
```yaml
理论依据:
  - 世界银行收入分类
  - 经济发展理论

锚点设定:
  0.0: 低收入 (< $1,045 GNI per capita)
  0.5: 中等收入 ($4,095 GNI per capita)
  1.0: 高收入 (> $13,205 GNI per capita)

校准函数:
  使用 logistic 函数
  避免机械分位数
```

### 模式2: 数据驱动校准（不推荐但常用）

**方法**: 基于数据分布

**示例**:
```yaml
使用分位数:
  - 33% 分位数 → 0.0
  - 67% 分位数 → 1.0

问题:
  - 不同样本不可比较
  - 缺乏理论依据

仅在以下情况使用:
  - 理论不明确
  - 探索性研究
  - 明确说明局限
```

### 模式3: 混合校准

**方法**: 理论 + 数据

**步骤**:
```yaml
Step 1: 理论设定锚点
  - 基于文献和理论

Step 2: 数据检验
  - 检查锚点在数据中的位置
  - 调整明显不合理的锚点

Step 3: 报告依据
  - 说明理论依据
  - 说明调整原因
```

## 必要/充分条件识别模式

### 模式1: 必要条件检验

**问题**: Y 是否需要 X?

**检验**: Y ⊆ X (所有 Y 都是 X)

**示例**:
```yaml
问题: "革命成功是否需要高压统治?"

检验:
  一致性 = |Y ∩ X| / |Y|
  = |成功案例 ∩ 高压| / |成功案例|

结果:
  一致性 = 0.92 (≥0.90)
  → 高压是必要条件

解释:
  - 92% 的成功案例有高压
  - 但高压不一定导致成功
```

**常见必要条件**:
- 民主转型需要政治危机
- 社会运动需要组织
- 政策采用需要议程设置

### 模式2: 充分条件检验

**问题**: X 是否导致 Y?

**检验**: X ⊆ Y (所有 X 都是 Y)

**示例**:
```yaml
问题: "高压 + 经济危机是否导致革命?"

检验:
  一致性 = |X ∩ Y| / |X|
  = |高压*危机 ∩ 成功| / |高压*危机|

结果:
  一致性 = 0.78 (≥0.75)
  覆盖度 = 0.45
  → 充分但不完整

解释:
  - 78% 的高压*危机案例革命成功
  - 但只解释了 45% 的成功案例
```

### 模式3: 必要且充分条件

**问题**: X 是否是 Y 的充要条件?

**检验**:
```yaml
必要性: Y ⊆ X
充分性: X ⊆ Y

充要: X = Y

示例:
  X = "总统制"
  Y = "行政主导"

  如果:
    所有行政主导国家都是总统制
    所有总统制国家都是行政主导

  则: 总统制 ⟷ 行政主导
```

## 组合复杂度管理

### 模式1: 条件数选择

**原则**: 2-7 个条件

**考虑因素**:
```yaml
案例数:
  - 10-30: 2-4 个条件
  - 30-100: 4-6 个条件
  - 100-200: 5-7 个条件

理论依据:
  - 每个条件有理论支持
  - 避免为数据而条件

可解释性:
  - 条件越多，组合越复杂
  - 案例分散到更多组合
```

**错误示例**:
```yaml
过度复杂:
  10 个条件 → 1024 个理论组合
  只有 50 个案例
  → 绝大多数组合空置

正确做法:
  减少到 4-5 个核心条件
  → 16-32 个理论组合
  → 案例更集中
```

### 模式2: 条件组合优化

**策略**: 删除冗余条件

**示例**:
```yaml
原始条件:
  C1: GDP per capita
  C2: GDP per capita squared
  C3: 经济增长率
  C4: 经济发展水平

问题: C1, C2, C3, C4 高度相关

解决:
  只保留 C4 (经济发展水平)
  其他作为敏感性检验
```

## 有限多样性应对策略

### 策略1: 增加案例

**方法**: 收集更多案例填补空组合

**示例**:
```yaml
空组合: C1*~C2*C3

解决:
  寻找符合 C1*~C2*C3 的案例
  - 历史案例
  - 跨国案例
  - 不同时期案例
```

### 策略2: 使用逻辑余项

**方法**: 基于理论简化空组合

**示例**:
```yaml
理论:
  "C1 和 C4 不可能同时存在"

空组合: C1*C4

处理:
  将 C1*C4 视为逻辑余项
  删除该组合
```

### 策略3: 报告局限

**方法**: 明确说明未观察的组合

**示例**:
```yaml
报告:
  "分析基于实际观察的组合
   未观察组合: C1*~C2*C3, ~C1*C2*~C4
   这些组合的理论推断需谨慎"
```

**功能**:
- fsQCA 标准软件

**功能**:

- 支持所有 QCA 类型

**优势**:
- 跨平台
- 现代界面
- 在线协作

**局限**:
- 需要网络
- 功能不如 fs/QCA 完整

- 需要编程

## 常见错误与修正

### 错误1: 机械校准

**错误**:
```yaml
使用统计分位数:
  - 33% 分位数 → 0.0
  - 67% 分位数 → 1.0
```

**修正**:
```yaml
使用理论锚点:
  - 基于文献定义
  - 基于知识设定
  - 报告校准依据
```

### 错误2: 忽视案例

**错误**:
```yaml
只报告逻辑表达式:
  "成功 = C1*C2 + C3"

不分析具体案例
```

**修正**:
```yaml
回到具体案例:
  1. 识别每个组合的案例
  2. 深入分析代表性案例
  3. 理解因果机制
```

### 错误3: 过度简化

**错误**:
```yaml
只报告最简解:
  "成功 = C1 + C2"

损失信息:
  - 不知道具体组合
  - 案例分散
```

**修正**:
```yaml
报告多种解:
  1. 复杂解（保留所有组合）
  2. 中间解（删除部分逻辑余项）
  3. 最简解（删除所有逻辑余项）
  4. 基于理论选择
```
