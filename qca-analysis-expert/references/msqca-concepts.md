# MSQCA核心概念详解

## MSQCA理论框架

### 什么是MSQCA?

**MSQCA** (Multi-Value Qualitative Comparative Analysis):
- 基于集合论和布尔代数的比较方法
- 分析因果复杂性（多重并发、非对称、等效性）
- 适用于中小样本（10-200个案例）

**vs 回归分析**:
| 维度 | 回归分析 | MSQCA |
|------|---------|-------|
| 因果模型 | 净效应（独立变量） | 因果组合（交互作用） |
| 对称性 | 对称（X→Y = Y→X相关） | 非对称（X→Y ≠ Y→X） |
| 等效性 | 单一路径 | 多条路径 |
| 案例 | 案例是观测单位 | 案例是核心 |

## 集合论基础

### 模糊集 (Fuzzy Sets)

**定义**: 元素以0.0-1.0的隶属度属于集合。

**示例**: "高收入国家"
- 美国: 0.95（完全属于）
- 中国: 0.70（较多属于）
- 印度: 0.30（较少属于）
- 马拉维: 0.05（几乎不属于）

**锚点** (Anchors):
- **0.0**: 完全不属于（full membership）
- **0.5**: 交叉点（crossover point）
- **1.0**: 完全属于（full non-membership）

### 多值集 (Multi-Value Sets)

**定义**: 超过两个类别的集合。

**示例**: "政体类型"
- 0.0: 完全威权
- 0.3: 威权主导
- 0.5: 混合政体
- 0.7: 民主主导
- 1.0: 完全民主

## 核心分析步骤

### Step 1: 校准 (Calibration)

**目标**: 将连续变量转化为集合隶属度。

**错误做法**（机械校准）:
```yaml
使用统计分位数:
  - 33%分位数 → 0.0
  - 67%分位数 → 1.0

问题:
  - 没有理论依据
  - 不同样本不可比较
```

**正确做法**（理论指导）:
```yaml
理论定义锚点:
  条件: "经济发展水平"
  0.0: 低收入国家定义（< $1,000 GNI per capita）
  0.5: 中等收入国家定义（$4,000 GNI per capita）
  1.0: 高收入国家定义（> $12,000 GNI per capita）

使用校准函数:
  - logit函数: log(x/(1-x))
  - logistic函数: 1/(1+e^(-x))
```

### Step 2: 真值表 (Truth Table)

**定义**: 列出所有可能的条件组合及其结果。

**示例**（3个条件: A, B, C）:
| A | B | C | 案例数 | 结果 | 一致性 |
|---|---|---|--------|------|--------|
| 0 | 0 | 0 | 5 | 0 | - |
| 0 | 0 | 1 | 3 | 1 | 0.95 |
| 0 | 1 | 0 | 2 | 0 | - |
| 0 | 1 | 1 | 8 | 1 | 0.92 |
| 1 | 0 | 0 | 1 | 0 | - |
| 1 | 0 | 1 | 4 | 1 | 0.88 |
| 1 | 1 | 0 | 6 | 1 | 0.90 |
| 1 | 1 | 1 | 10 | 1 | 0.98 |

**逻辑行**: 有足够案例数且一致性高的组合。

### Step 3: 布尔最小化 (Boolean Minimization)

**目标**: 简化逻辑表达式，保留核心因果组合。

**示例**:
```
原始: A*B*C + A*B*~C + A*~B*C
简化: A*B + A*C

解释:
- A*B: 当A和B存在时，结果发生（无论C）
- A*C: 当A和C存在时，结果发生（无论B）
```

**三种解**:
1. **复杂解** (Complex Solution): 保留所有逻辑余项
2. **简化解** (Parsimonious Solution): 删除所有逻辑余项
3. **中间解** (Intermediate Solution): 只删除合理的逻辑余项

### Step 4: 必要性/ sufficiency分析

**充分性** (Sufficiency):
- 问题: "X是否导致Y?"
- 检验: X ⊆ Y （X是Y的子集）
- 指标: 一致性（≥0.75）、覆盖度

**必要性** (Necessity):
- 问题: "Y是否需要X?"
- 检验: Y ⊆ X （Y是X的子集）
- 指标: 一致性（≥0.90）、覆盖度

**示例**:
```yaml
充分性: A*B → 革命成功
  - 一致性: 0.85（85%的A*B案例革命成功）
  - 原始覆盖度: 0.40（40%的革命案例由A*B解释）

必要性: 革命成功 → A
  - 一致性: 0.95（95%的革命案例有A）
  - 覆盖度: 0.70（70%的A案例发生革命）
```

## 质量指标

### 一致性 (Consistency)

**充分一致性**:
```
Consistency(X → Y) = |X ∩ Y| / |X|

含义: X案例中Y的比例
阈值: ≥0.75
```

**必要一致性**:
```
Consistency(Y → X) = |X ∩ Y| / |Y|

含义: Y案例中X的比例
阈值: ≥0.90
```

### 覆盖度 (Coverage)

**原始覆盖度** (Raw Coverage):
```
Raw Coverage = |X ∩ Y| / |Y|

含义: X解释的Y案例比例
```

**唯一覆盖度** (Unique Coverage):
```
Unique Coverage = |X_only ∩ Y| / |Y|

含义: X独占解释的Y案例比例
```

## 常见错误

### 错误1: 机械校准

**问题**: 使用统计分位数（33%, 67%）
**正确**: 基于理论设定锚点

### 错误2: 忽视案例

**问题**: 只看真值表和逻辑表达式
**正确**: 回到具体案例，深入理解

### 错误3: 过度简化

**问题**: 只报告最简解
**正确**: 报告复杂解、简解开、最简解

### 错误4: 统计思维套用

**问题**: 寻找"显著性"、"效应大小"
**正确**: 使用集合论语言（包含、子集）
