## Phase 2: Level 1 - 基础升级 (30分钟)

### 2.1 定制6大禁止原则

**针对扎根理论的特点**:

```yaml
1. 禁止编码前预设结论
   理由: 违背扎根理论的归纳本质

2. 禁止脱离原始数据编码
   理由: 编码必须紧贴数据

3. 禁止编码无理论依据
   理由: 编码需要理论敏感性

4. 禁止忽视负面案例
   理由: 负面案例能完善理论

5. 禁止追求编码数量
   理由: 质量重于数量

6. 禁止编码标准不一致
   理由: 影响理论饱和度判断
```

### 2.2 创建任务分解规则

```yaml
第一层: 主要编码阶段
  - 阶段1: 开放编码
  - 阶段2: 轴心编码
  - 阶段3: 选择式编码

第二层: 每阶段的子任务
  开放编码:
    - 逐行/逐句编码
    - 识别初始概念
    - 比较概念异同
    - 聚焦重要概念

  轴心编码:
    - 识别核心范畴
    - 建立范畴联系
    - 开发范畴属性
    - 绘制 paradigm

  选择式编码:
    - 选择核心范畴
    - 整合故事线
    - 完善理论
    - 验证饱和度

第三层: 原子任务
  - 读取数据文件
  - 编码片段X
  - 比较概念A和B
  - 验证饱和度
```

```yaml
# 扎根理论分析 - 识别的模式

## Pattern 1: 开放编码模式

**频率**: 高频 (25次/年)
**成功率**: 92%

### 编码策略

逐行编码 vs 逐句编码:
- 逐行编码: 适合细节密集的数据
- 逐句编码: 适合完整叙述的数据
- 推荐: 初期逐行，后期逐句

概念识别:
- 寻找变异点
- 关注过程性词汇
- 注意情感表达
- 捕捉反常案例

### 经验教训

✅ **成功因素**
- 保持开放心态
- 紧贴原始数据
- 持续比较概念
- 记录备忘录

⚠️ **常见错误**
- 过早归纳
- 忽视细节
- 强行分类
- 遗漏变异

## Pattern 2: 轴心编码模式

**频率**: 中频 (15次/年)
**成功率**: 88%

### 范畴识别

核心范畴特征:
- 高频率出现
- 连接多个概念
- 有理论潜力
- 可持续分析

建立联系:
- 因果关系
- 条件关系
- 时间关系
- 策略关系

### 经验教训

✅ **成功因素**
- 反复比较数据
- 绘制关系图
- 开发属性维度
- 验证关系合理性

⚠️ **常见错误**
- 过早确定核心
- 忽视复杂关系
- 强行建立联系
- 缺少属性开发

## Pattern 3: 饱和度检验模式

**频率**: 中频 (18次/年)
**成功率**: 95%

### 饱和度指标

概念饱和:
- 新数据不再产生新概念
- 概念属性完整
- 概念维度明确

范畴饱和:
- 范畴间关系稳定
- 核心范畴明确
- 理论框架清晰

### 检验方法

定量指标:
- 连续N个样本无新概念（N=5-10）
- 概念清单稳定

定性判断:
- 理论完整性
- 解释充分性
- 预测准确性
```

```yaml
# Case 001: 预设结论 - 失败案例

**日期**: 2026-02-15
**错误类型**: 编码前预设结论

## 问题描述

研究主题: 社交媒体对青少年心理健康的影响

预设假设: 社交媒体使用导致抑郁

## 错误过程

### 开放编码阶段

❌ **错误做法**:
- 带着"证明负面影响"的假设去编码
- 只关注负面情绪的表达
- 忽视中性或正面的体验
- 强行将编码归类到"负面影响"

编码偏差示例:
- "我在网上和朋友聊天很开心"
  → 错误编码: "表面开心，实际孤独"
  → 正确编码: "社交连接"

### 发现的问题

1. 编码者偏差严重
2. 理论与数据不符
3. 饱和度无法达到
4. 理论解释力弱

## 正确做法

✅ **修正方案**:
1. 放弃预设假设
2. 重新开放编码
3. 保持完全开放
4. 让理论自然涌现

## 修正结果

重新编码后发现的主题:
- 社交连接与支持
- 信息过载与压力
- 自我呈现与焦虑
- 多重体验（既有正面也有负面）

最终理论: "社交媒体的双面性理论"

## 经验教训

⚠️ **关键错误**:
- 违背了扎根理论的归纳原则
- 用演绎而非归纳方法

✅ **正确原则**:
1. 完全开放的心态
2. 紧贴原始数据
3. 让理论自然涌现
4. 避免研究者偏见

## 预防措施

1. 编码前明确检查假设
2. 定期反思编码过程
3. 寻求同行反馈
4. 记录编码决策

**质量评分**: 1/5 (失败案例，但学习价值高)
```

   - 紧贴数据

2. **系统化升级**

**Level 1最重要**:
- 6大禁止原则是基础
- 定制化是关键
- 不能通用模板

**Level 2最耗时**:
- 需要查阅文献
- 案例需要精心设计
- 但价值最大

**Level 3最技术**:

- 但一次性投入

**Level 4最长期**:
