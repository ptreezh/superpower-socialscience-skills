# 负面案例002: 忽略网络权重的错误

## 案例描述

研究者分析某公司内部邮件通信网络（200名员工，5000封邮件），试图识别关键沟通人物。

**数据特征**：邮件边有**权重**（邮件数量），权重范围从1到150。

---

## ❌ 错误做法

### 1. 忽略权重，将网络视为无权

**研究者使用无权网络指标**：

```python
# 错误：将加权网络转换为无权
G_unweighted = nx.Graph()
G_unweighted.add_edges_from(G.edges())  # 忽略权重

# 计算度中心性（无权）
degree_centrality = nx.degree_centrality(G_unweighted)
```

**错误结果**：

| 员工 | 无权度中心性 | 邮件数（权重） | 实际重要性 |
|------|-------------|---------------|-----------|
| 张三 | 0.45 (高) | 平均10封 | 被高估 |
| 李四 | 0.15 (低) | 与3人共1500封 | 被严重低估 |
| 王五 | 0.30 (中) | 与5人共750封 | 被低估 |

**为什么错误**：
- 张三给45个人各发1封邮件 → 度中心性高，但实际影响小
- 李四与3个人频繁沟通（每人500封） → 度中心性低，但实际影响大
- **无权指标完全忽略了沟通强度**

### 2. 基于错误指标得出错误结论

**错误报告**：
> "张三是公司的沟通枢纽，直接与45%的员工有联系。李四只是边缘人物，仅与3人沟通。建议加强李四的沟通网络。"

**这个结论是错误的**，因为李四虽然连接少，但沟通强度大，实际是关键人物。

---

## ✅ 正确做法

### 1. 使用加权中心性指标

```python
# 正确：保留权重
G_weighted = nx.Graph()
G_weighted.add_weighted_edges_from([(u, v, d['weight'])
                                     for u, v, d in G.edges(data=True)])

# 计算加权度中心性
weighted_degree = dict(G_weighted.degree(weight='weight'))

# 归一化
max_weight = max(weighted_degree.values())
weighted_degree_centrality = {n: w/max_weight
                             for n, w in weighted_degree.items()}
```

### 2. 同时考虑结构和强度

**综合分析**：

| 员工 | 无权度 | 加权度 | 解读 |
|------|--------|--------|------|
| 张三 | 0.45 (高) | 0.12 (低) | **广播型**：连接广但浅 |
| 李四 | 0.15 (低) | 0.85 (高) | **深度型**：连接窄但深 |
| 王五 | 0.30 (中) | 0.50 (中) | **平衡型**：连接和强度中等 |

**策略建议**：
- **张三**：适合信息广播，但不适合深度协调
- **李四**：适合深度合作和关键任务，但需扩展沟通面
- **王五**：平衡型，适合一般协调任务

### 3. 使用多种加权指标

```python
# 加权介数中心性
betweenness_weighted = nx.betweenness_centrality(G_weighted, weight='weight')

# 加权接近中心性
closeness_weighted = nx.closeness_centrality(G_weighted, distance='weight')
```

**注意**：对于接近中心性，权重应解释为"距离"（权重越大，距离越小）

### 4. 基于正确指标得出正确结论

**正确报告**：
> "综合无权和加权分析，张三是广播型沟通者（连接广但浅），适合信息发布；李四是深度型沟通者（连接窄但深），适合关键任务协调；王五是平衡型。建议加强李四的沟通广度，同时保持其深度沟通优势。"

---

## 🎯 关键教训

### 1. 如果数据有权重，必须使用
❌ **错误**：忽略权重，当作无权网络处理
✅ **正确**：使用加权中心性指标

### 2. 权重改变一切
❌ **错误**：假设无权和加权的排名相似
✅ **正确**：无权和加权可能完全不同，必须分别分析

### 3. 综合结构和强度
❌ **错误**：只看加权或只看无权
✅ **正确**：同时考虑，分类讨论（广播型/深度型/平衡型）

### 4. 明确权重的含义
❌ **错误**：不解释权重代表什么
✅ **正确**：明确权重=邮件数量，权重越大，沟通越频繁

## 防范措施

**数据分析前自检**：
- [ ] 数据是否有权重属性？
- [ ] 权重的含义是什么？（频次、强度、相似度...）
- [ ] 权重分布如何？（是否有极端值）
- [ ] 是否应该使用加权指标？

**权重处理决策树**：
```
数据有权重？
  ├─ 是 → 权重重要吗？
  │       ├─ 是 → 使用加权指标
  │       └─ 否 → 权重可能是噪音，考虑无权
  └─ 否 → 使用无权指标
```

**同时报告无权和加权结果**：
- 无权：网络结构（谁与谁连接）
- 加权：关系强度（连接多重要）
- 综合：分类讨论（广播型/深度型/平衡型）
