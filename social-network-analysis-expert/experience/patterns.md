## 🎯 高频分析模式

### Pattern 1: 研究合作网络分析

**频率**: 高频 (25次/年)
**成功率**: 95%
**平均时长**: 2-3小时

#### 标准分析流程

```yaml
Phase 1: 数据准备（30分钟）
  Task 1.1: 数据验证（10分钟）
    - 检查边列表格式
    - 验证节点ID
    - 去除重复边

  Task 1.2: 数据清洗（10分钟）
    - 移除孤立点
    - 移除自环
    - 处理缺失值

  Task 1.3: 网络构建（10分钟）
    - 构建NetworkX图
    - 计算基础指标
    - 验证网络连通性

Phase 2: 中心性分析（1小时）
  Task 2.1: 度中心性（15分钟）
    - 计算度中心性
    - 识别核心节点
    - 排名分析

  Task 2.2: 介数中心性（15分钟）
    - 计算介数中心性
    - 识别经纪人
    - 关键路径分析

  Task 2.3: 接近中心性（15分钟）
    - 计算接近中心性
    - 识别中心节点
    - 效率分析

  Task 2.4: 特征向量中心性（15分钟）
    - 计算特征向量
    - 识别影响力节点
    - 与其他指标对比

Phase 3: 社群检测（30分钟）
  Task 3.1: Louvain算法（15分钟）
    - 检测社群
    - 计算模度值
    - 社群分配

  Task 3.2: 社群质量（15分钟）
    - 社群内密度
    - 社群间连接
    - 验证划分

Phase 4: 结构洞分析（30分钟）
  Task 4.1: 约束系数（15分钟）
    - 计算约束系数
    - 识别结构洞
    - 经纪人分析

  Task 4.2: 桥接节点（15分钟）
    - 识别桥接节点
    - 关键连接
    - 瓶颈分析

Phase 5: 可视化与报告（30分钟）
  Task 5.1: 网络可视化（20分钟）
    - 节点大小=中心性
    - 颜色=社群
    - 布局优化

  Task 5.2: 分析报告（10分钟）
    - 汇总发现
    - 关键节点
    - 社群特征
```

#### 质量检查清单

```yaml
数据质量:
  - [ ] 边列表格式正确
  - [ ] 无孤立点（或合理）
  - [ ] 无自环
  - [ ] 网络连通

分析质量:
  - [ ] 所有中心性指标已计算
  - [ ] 社群检测完成
  - [ ] 结构洞分析完成
  - [ ] 指标有解释

报告质量:
  - [ ] 关键节点识别
  - [ ] 社群特征描述
  - [ ] 网络图清晰
  - [ ] 发现有洞察
```

#### 成功因素

✅ **多种中心性指标**
   - 不只看度中心性
   - 结合介数、接近、特征向量
   - 综合分析

✅ **社群验证**
   - 不只依赖模度值
   - 检查社群内密度
   - 验证社群间连接

✅ **可视化优化**
   - 节点大小反映中心性
   - 颜色区分社群
   - 布局清晰

**成功率**: 90%
**平均时长**: 2小时

#### 与合作网络的区别

```yaml
社交媒体网络特点:
  1. 规模更大
     - 节点数通常1000+
     - 边数密集
     - 计算复杂度高

  2. 有向图
     - 关注关系有方向
     - 入度 vs 出度
     - 不对称连接

  3. 动态性
     - 时间维度
     - 演化模式
     - 影响扩散

分析重点:
  - 影响力节点（PageRank）
  - 信息传播
  - 社群形成
  - 关键意见领袖
```

## 🔍 中心性指标模式

### 度中心性 (Degree Centrality)

```yaml
定义:
  节点的连接数

适用场景:
  - 识别最活跃节点
  - 快速评估
  - 初步分析

解释:
  - 高度中心性: 节点很活跃，有很多连接
  - 低度中心性: 节点较孤立

常见模式:
  合作网络:
    - 高度: 研究者很合作
    - 低度: 研究者较独立
```

### 介数中心性 (Betweenness Centrality)

```yaml
定义:
  节点在最短路径上的频率

适用场景:
  - 识别经纪人
  - 识别瓶颈
  - 信息流动分析

解释:
  - 高介数: 节点控制信息流
  - 低介数: 节点不在关键路径

常见模式:
  合作网络:
    - 高介数: 跨领域合作者
    - 关键中介: 连接不同社群
```

### 接近中心性 (Closeness Centrality)

```yaml
定义:
  节点到所有其他节点的平均距离的倒数

适用场景:
  - 识别中心位置
  - 效率分析
  - 影响力传播

解释:
  - 高接近: 节点可以快速到达其他节点
  - 低接近: 节点较边缘

常见模式:
  合作网络:
    - 高接近: 核心研究者
    - 快速信息传播
```

### 特征向量中心性 (Eigenvector Centrality)

```yaml
定义:
  节点的重要性基于其邻居的重要性

适用场景:
  - 识别影响力节点
  - 考虑网络结构
  - 长期影响

解释:
  - 高特征向量: 连接重要节点
  - 低特征向量: 连接不重要节点

常见模式:
  合作网络:
    - 高特征向量: 连接知名研究者
    - 学术影响力
```

## 📊 社群检测模式

### Louvain算法

```yaml
特点:
  - 快速
  - 可扩展
  - 无需预设社群数

适用场景:
  - 大型网络
  - 未知社群数
  - 快速检测

质量评估:
  - 模度值 (Modularity): >0.3较好
  - 社群内密度: 高
  - 社群间连接: 低
```

### 其他算法

```yaml
标签传播 (Label Propagation):
  - 快速
  - 适合大型网络
  - 结果不稳定

 Girvan-Newman:
  - 基于边介数
  - 适合小型网络
  - 计算密集

 Infomap:
  - 信息流
  - 适合有向图
  - 较慢
```

## 🕳️ 结构洞模式

### 约束系数 (Constraint)

```yaml
定义:
  节点在社群中的冗余连接程度

解释:
  - 低约束: 有结构洞机会
  - 高约束: 冗余连接多

应用:
  - 识别经纪人
  - 创新机会
  - 信息优势
```

### 经纪人角色

```yaml
类型:
  1. 协调者 (Coordinator)
     - 同一群体内连接

  2. 顾问 (Consultant)
     - 不同群体内连接

  3. 门卫 (Gatekeeper)
     - 群体间信息控制

  4. 代表 (Representative)
     - 对外代表群体

识别:
  - 高介数 + 低约束 = 经纪人
  - 连接不同社群
```

## 💡 经验教训

### 成功经验

✅ **多种指标结合**
   - 不只看单一指标
   - 综合分析
   - 交叉验证

✅ **可视化辅助**
   - 帮助理解
   - 发现模式
   - 传达结果

✅ **理论联系**
   - 结合社会学理论
   - 解释网络结构
   - 提供洞察

### 常见陷阱

❌ **过度解读中心性**
   - 高中心性≠高影响力
   - 需要结合其他指标
   - 需要考虑网络类型

❌ **忽视网络规模**
   - 大网络vs小网络
   - 指标可比较性
   - 稀疏vs密集

❌ **忽略社群质量**
   - 不只看模度值
   - 验证社群划分
   - 考虑业务意义
