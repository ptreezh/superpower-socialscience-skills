## ⚠️ 五大核心原则 (必须严格遵守)

### 1. 结构决定论

**核心观点**: 网络位置决定行为和机会

**必须做到**:
- 分析行为时首先考虑网络位置
- 用结构特征解释个体差异
- 识别关键位置（中心、桥梁、边缘）

**量化标准**:
- 必须计算至少3种中心性指标
- 必须识别Top-K关键节点

### 2. 关系数据优先

**核心观点**: 关系本身是分析对象，不是个体属性的附属

**必须做到**:
- 数据收集以关系为核心
- 分析重点在边而非节点
- 记录关系强度和方向

**数据格式要求**:
- 边列表: `(source, target, weight?)`
- 邻接矩阵: 方阵，元素表示关系强度

### 3. 网络位置解释

**核心观点**: 位置相似性解释行为相似性

**必须做到**:
- 使用结构等价性分析
- 识别位置相似的节点群
- 解释位置与行为的相关性

### 4. 方法论一致性

**核心观点**: 理论与方法严格对应

**必须做到**:
- 选择方法时说明理论依据
- 不混用冲突的理论框架
- 解释结果时回溯理论假设

### 5. 边界界定规范

**核心观点**: 网络边界必须明确且有理论依据

**必须做到**:
- 明确说明边界界定标准
- 讨论边界对结果的影响
- 报告孤立节点处理方式

## 📊 中心性指标解释标准

### 度中心性 (Degree Centrality)

**含义**: 节点的直接连接数量

**解释框架**:
- 高度中心性 = 信息获取优势、可见度高
- 低度中心性 = 边缘位置、信息劣势

**公式**: `C_D(v) = d(v) / (n-1)`

**适用场景**: 
- 信息传播分析
- 影响力识别
- 资源获取能力评估

### 介数中心性 (Betweenness Centrality)

**含义**: 节点处于其他节点最短路径上的程度

**解释框架**:
- 高介数 = 桥梁位置、信息控制力
- 低介数 = 边缘或冗余位置

**公式**: `C_B(v) = Σ σ_{st}(v) / σ_{st}`

**适用场景**:
- 信息流通分析
- 关键节点识别
- 网络脆弱性评估

### 接近中心性 (Closeness Centrality)

**含义**: 节点到其他所有节点的平均距离

**解释框架**:
- 高接近 = 信息传播快、独立性
- 低接近 = 信息滞后、依赖他人

**公式**: `C_C(v) = (n-1) / Σ d(v, u)`

**适用场景**:
- 信息传播效率分析
- 独立性评估
- 时效性要求高的场景

### 特征向量中心性 (Eigenvector Centrality)

**含义**: 连接到重要节点的程度

**解释框架**:
- 高特征向量 = 连接核心、影响力传导
- 低特征向量 = 连接边缘、影响力有限

**适用场景**:
- 影响力分析
- 核心节点识别
- 递归影响评估

## 📋 任务分解规则

### 分解原则

1. **粒度可控**: 每个子任务必须在一次会话中完成
2. **量化标准**: 每个子任务必须有明确的完成标准
3. **独立验证**: 每个子任务完成后必须独立验证
4. **数据清单**: 子Agent必须输出完整的数据清单

```yaml

├─ 子任务2: 中心性分析
│   - 完成标准: 所有中心性指标计算完成
│   - 输出清单: centrality_results.json, ranking.json
│   - 验证方法: 检查值范围[0,1]
├─ 子任务3: 社群检测
│   - 完成标准: 社群划分完成，Q值计算完成
│   - 输出清单: communities.json, modularity.json
│   - 验证方法: Q值>0.3
└─ 子任务4: 报告生成
    - 完成标准: 分析报告生成
    - 输出清单: report.md
    - 验证方法: 检查报告完整性
```

## 🔧 分析流程

### 标准分析流程

1. **数据准备**
   - 解析边列表或邻接矩阵
   - 验证数据格式
   - 处理缺失值和异常值

2. **网络构建**
   - 创建NetworkX图对象
   - 设置有向/无向属性
   - 添加权重（如有）

3. **基础指标计算**
   - 节点数、边数、密度
   - 连通性检查
   - 平均度数

4. **中心性分析**
   - 计算度中心性
   - 计算介数中心性
   - 计算接近中心性
   - 计算特征向量中心性
   - 生成综合排名

5. **社群检测**
   - 执行Louvain算法
   - 计算模块化Q值
   - 分析社群结构

6. **结构洞分析**
   - 计算约束指标
   - 计算有效规模
   - 识别经纪人

7. **报告生成**
   - 整合所有结果
   - 生成可视化
   - 撰写解释文本

## ⛔ 常见错误警示

### 数据层面
- ❌ 混淆有向/无向网络
- ❌ 忽略权重信息
- ❌ 不处理孤立节点

### 方法层面
- ❌ 中心性值超出[0,1]范围未检查
- ❌ Q值<0.3仍声称发现社群结构
- ❌ 网络不连通时仍计算接近中心性

### 解释层面
- ❌ 中心性解释脱离网络背景
- ❌ 过度解读中心性排名
- ❌ 忽略网络规模对指标的影响
