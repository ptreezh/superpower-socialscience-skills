# 社会网络分析方法详解

## 中心性分析

### 度中心性 (Degree Centrality)

**定义**：节点在网络中的直接连接数量

**计算**：
```
DC(i) = degree(i) / (n - 1)
```

**解读**：
- 高度中心性：网络中的"明星"，直接连接众多
- 低度中心性：边缘节点，连接较少

**应用场景**：识别关键影响者、核心人物

---

### 介数中心性 (Betweenness Centrality)

**定义**：节点位于其他节点最短路径上的频率

**计算**：
```
BC(i) = Σ (σ_st(i) / σ_st)
```
其中 σ_st 是节点 s 到 t 的最短路径数量，σ_st(i) 是经过节点 i 的最短路径数量

**解读**：
- 高介数中心性：网络中的"桥梁"、"守门人"
- 控制信息流动的关键节点

**应用场景**：识别信息中介、资源控制者

---

### 接近中心性 (Closeness Centrality)

**定义**：节点到其他所有节点的平均距离的倒数

**计算**：
```
CC(i) = (n - 1) / Σ distance(i, j)
```

**解读**：
- 高接近中心性：能够快速到达网络中的其他节点
- 信息传播效率高

**应用场景**：识别高效传播者、独立影响力

---

### 特征向量中心性 (Eigenvector Centrality)

**定义**：节点的重要性取决于其连接节点的重要性

**计算**：
```
Ax = λx
```
其中 A 是邻接矩阵，x 是特征向量

**解读**：
- 高特征向量中心性：与重要人物连接的人
- 间接影响力强

**应用场景**：识别隐性影响者、权力网络

## 社群检测

### Louvain 算法

**原理**：基于模块度优化的层次聚类

**步骤**：
1. 初始化：每个节点为一个社群
2. 迭代：移动节点到相邻社群以最大化模块度
3. 聚合：将每个社群缩为一个超级节点
4. 重复步骤2-3直到模块度不再增加

**优势**：
- 速度快，适合大规模网络
- 能发现层次结构
- 模块度评分直观

**评估**：
- 模块度 (Modularity) > 0.3：有意义社群结构
- 模块度 > 0.5：强社群结构

---

### 标签传播算法 (Label Propagation)

**原理**：基于邻居标签的迭代传播

**步骤**：
1. 初始化：每个节点赋予唯一标签
2. 迭代：每个节点采用邻居中最常见的标签
3. 收敛：当标签不再变化时停止

**优势**：
- 无需预设社群数量
- 计算简单快速
- 适合动态网络

**局限**：
- 可能不稳定（多次运行结果不同）
- 倾向于形成巨型社群

---

## 结构洞分析

### Burt 约束指标 (Constraint)

**定义**：节点在网络中的冗余连接程度

**计算**：
```
C(i) = Σ [p(i,j) + Σ p(i,q) p(q,j)]²
```
其中 p(i,j) 是节点 i 投入在 j 上的关系比例

**解读**：
- 高约束 (>0.5)：网络封闭，冗余连接多
- 低约束 (<0.3)：结构洞丰富，机会多

**策略意义**：
- 低约束节点：适合创业、创新
- 高约束节点：适合稳定合作、信任建立

---

### 有效规模 (Effective Size)

**定义**：非冗余连接的数量

**计算**：
```
ES(i) = n - Σ t * p(i,j)
```
其中 t 是 j 的其他连接中与 i 连接的比例

**解读**：
- 高有效规模：连接多样，信息源丰富
- 低有效规模：连接冗余，回声室效应

---

### 桥接角色识别

**类型**：
1. **桥梁 (Bridge)**：连接不同社群的唯一路径
2. **中介 (Broker)**：在不同社群间传递信息
3. **边界跨越者 (Boundary Spanner)**：连接多个社群

**识别方法**：
- 边介数中心性高的边
- 跨社群连接的节点
- 低约束、高有效规模的节点

## 网络可视化

### 力导向布局 (Force-Directed Layout)

**原理**：将节点模拟为带电粒子，边模拟为弹簧

**参数**：
- 引力强度：节点间吸引力
- 斥力强度：节点间排斥力
- 弹簧长度：边的理想长度

**优势**：
- 直观展示网络结构
- 自然的聚类效果
- 交互式探索友好

**注意事项**：
- 大型网络（>1000节点）会变慢
- 多次运行可能不同（初始位置随机）

### 可视化编码

**节点大小**：度中心性或其他指标
**节点颜色**：社群归属或节点属性
**边粗细**：权重或连接强度
**标签位置**：避免重叠，确保可读性

## 质量检查点

### 数据质量检查
- [ ] 所有节点都有唯一ID
- [ ] 边列表无重复
- [ ] 无孤立点（除非有特殊原因）
- [ ] 网络连通性合理

### 分析质量检查
- [ ] 所有中心性指标已计算
- [ ] 社群检测有模块度评估
- [ ] 结构洞指标完整
- [ ] 结果有统计描述

### 可视化质量检查
- [ ] 布局清晰，无过度拥挤
- [ ] 标签可读（字体大小合适）
- [ ] 颜色有意义（图例清晰）
- [ ] 交互式功能正常

## 常见错误与避免

### 错误1：只看度中心性
**问题**：忽略介数、接近、特征向量中心性
**后果**：遗漏关键桥梁节点、隐性影响者
**解决**：必须计算全部4项指标

### 错误2：忽视网络权重
**问题**：将所有边视为相同权重
**后果**：强连接和弱连接无区分
**解决**：使用加权中心性指标

### 错误3：社群检测单一算法
**问题**：只运行一种社群检测算法
**后果**：社群划分可能不稳定
**解决**：至少使用2种算法并比较

### 错误4：过度解读小样本
**问题**：对小型网络（<50节点）进行过度分析
**后果**：统计意义有限，结论不可靠
**解决**：谨慎解读，明确样本限制
