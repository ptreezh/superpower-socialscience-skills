# 社会网络分析 - 长时任务指南

## 长时任务场景

### 场景1：大型网络分阶段分析（2-4周）

**适用情况**：
- 网络规模 > 1000 个节点
- 分析维度多（中心性 + 社群 + 结构洞）
- 需要深度解读和验证

**任务分解**：

#### Week 1: 数据准备与基础分析
```
Day 1-2: 数据收集与清洗
  - 输出：清洗后的边列表、节点属性表
  - 验证：数据完整性检查

Day 3-4: 网络构建与描述性分析
  - 输出：网络对象、基础指标（密度、直径、聚类系数）
  - 验证：网络构建成功，指标计算完整

Day 5-7: 中心性分析（第一批节点）
  - 输出：中心性指标表格、初步排名
  - 验证：覆盖所有节点，有初步解读
```

#### Week 2: 深度中心性分析与社群检测
```
Day 8-10: 中心性分析（第二批节点）与综合
  - 输出：完整中心性分析报告
  - 验证：所有节点覆盖，深度解读完成

Day 11-14: 社群检测与验证
  - 输出：社群划分结果、模块度报告、社群特征
  - 验证：至少2种算法，结果稳定
```

#### Week 3: 结构洞分析与可视化
```
Day 15-17: 结构洞分析
  - 输出：约束指标、有效规模、桥接角色识别
  - 验证：指标完整，识别准确

Day 18-21: 可视化与报告撰写
  - 输出：网络可视化图、综合分析报告
  - 验证：可视化清晰，报告完整
```

#### Week 4: 验证与完善
```
Day 22-24: 结果验证
  - 与领域专家验证
  - 补充敏感性分析
  - 交叉验证结果

Day 25-28: 报告完善与交付
  - 补充方法论说明
  - 完善策略建议
  - 最终报告交付
```

## 跨对话任务状态保持

### 状态文件结构

```
project/
├── task_plan.md           # 总体任务计划
├── progress.md            # 进度日志
├── findings.md            # 关键发现
├── data/
│   ├── network.gml        # 当前网络数据
│   └── results/           # 分析结果
└── checkpoints/
    ├── phase1_complete.md
    ├── phase2_complete.md
    └── ...
```

### 会话恢复机制

**每次会话开始时**：
1. 读取 `progress.md` - 了解当前进度
2. 读取 `findings.md` - 了解关键发现
3. 读取最新 checkpoint - 了解已完成的工作
4. 确认下一步任务

**每次会话结束时**：
1. 更新 `progress.md` - 记录新完成的任务
2. 更新 `findings.md` - 记录新发现
3. 创建 checkpoint - 保存当前状态
4. 规划下次会话任务

## 质量保证机制

### 每阶段验证清单

**数据准备阶段**：
- [ ] 数据完整性验证
- [ ] 数据一致性检查
- [ ] 网络构建成功

**分析执行阶段**：
- [ ] 所有指标计算完成
- [ ] 结果文件可读
- [ ] 初步解读存在

**报告生成阶段**：
- [ ] 方法论说明完整
- [ ] 结果呈现清晰
- [ ] 策略建议有依据

### 阶段间质量检查

**从数据准备到分析执行**：
- 检查：数据质量是否满足分析要求？
- 验证：网络构建是否正确？
- 确认：分析计划是否可行？

**从分析执行到报告生成**：
- 检查：所有分析是否完成？
- 验证：结果是否可靠？
- 确认：解读是否有依据？

## 常见问题与解决

### Q1: 大型网络分析超时
**问题**：网络太大，一次性分析超时
**解决**：
- 采样分析（随机采样10%子网络验证）
- 分批分析（将网络分成多个连通分量）
- 增量分析（先分析核心子网络）

### Q2: 长时任务中断
**问题**：分析过程中断，需要重新开始
**解决**：
- 使用 checkpoint 机制
- 定期保存中间结果
- 设计可恢复的工作流

### Q3: 跨对话结果不一致
**问题**：不同对话的分析结果不同
**解决**：
- 固定随机种子
- 使用确定性算法
- 记录所有参数设置
