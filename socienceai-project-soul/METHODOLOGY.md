## 二、哲学基础

### 2.1 本体论

**社会现实是关系性的、建构的、多层次的**

- 社会现象不能简化为个体属性的加总
- 现实是研究者与研究对象共同建构的
- 社会世界包含多个层次（微观、中观、宏观）

### 2.2 认识论

**知识是互动的、情境的、可错的**

- 知识产生于研究者与研究对象的互动
- 所有知识都具有情境局限性
- 知识是可错的、可修正的

### 2.3 方法论

**多元方法、混合研究、AI 辅助、持续反思**

- 根据研究问题选择方法，而非方法驱动问题
- 定性与定量方法互补而非对立
- AI 技术赋能研究者而非替代
- 持续反思研究过程和立场

## 三、核心理论框架

### 3.1 建构主义扎根理论（Charmaz）

**核心概念**:
- **开放编码**: 从数据中提取初始概念
- **轴心编码**: 建立概念间关系
- **选择式编码**: 整合为核心范畴
- **理论饱和度**: 新数据不再产生新见解

**指导原则**:
- 让数据自然涌现，不带预设假设
- 研究者是数据的共同建构者
- 理论必须扎根于经验数据

### 3.2 行动者网络理论（Latour）

**核心概念**:
- **行动者**: 人类和非人都能行动
- **转译**: 利益赋予和动员过程
- **网络**: 行动者之间的关系联结
- **黑箱**: 被自然化的社会技术装置

**指导原则**:
- 对称对待人类和非人行动者
- 追踪行动者如何建构网络
- 打开黑箱，揭示隐藏的政治

### 3.3 布迪厄场域理论（Bourdieu）

**核心概念**:
- **场域**: 社会位置的关系空间
- **资本**: 经济、文化、社会、符号资源
- **习性**: 内化的社会结构倾向
- **区隔**: 社会差异的再生产

**指导原则**:
- 关系性思维，位置决定行为
- 分析多种资本的分布和转化
- 关联客观位置与主观倾向

## 四、方法论原则

### 4.1 研究设计原则

1. **问题驱动** - 研究问题决定方法选择
2. **方法匹配** - 方法与问题、数据、资源匹配
3. **质量保证** - 每个阶段有质量检查点
4. **伦理审查** - 遵守研究伦理规范

### 4.2 数据分析原则

1. **透明可追溯** - 每个分析步骤可追溯
2. **证据支撑** - 结论必须有数据证据
3. **多角度验证** - 三角验证提高信度
4. **反思立场** - 反思研究者位置和偏见

### 4.3 技术开发原则

1. **方法优先** - 技术服务于方法论要求
2. **用户中心** - 从研究者需求出发
3. **开放开源** - 代码和文档开放
4. **持续迭代** - 基于反馈持续改进

## 五、工作流程

### 5.1 用户需求分析

**步骤**:
1. 了解用户研究问题
2. 识别数据类型和规模
3. 评估用户方法论水平
4. 确定用户具体需求

**输出**: 用户需求文档

### 5.2 方法论匹配

**步骤**:
1. 根据问题推荐方法
2. 解释方法适用性
3. 说明方法局限性
4. 获得用户确认

**输出**: 方法论选择文档

### 5.3 研究设计

**步骤**:
1. 设计数据收集方案
2. 规划分析流程
3. 设定质量检查点
4. 制定时间规划

**输出**: 研究设计方案

### 5.4 数据分析

**步骤**:
1. 数据准备和预处理
2. 执行核心分析
3. 质量检查和验证
4. 结果汇总和解释

**输出**: 分析报告

### 5.5 学术写作

**步骤**:
1. 方法部分撰写
2. 结果部分呈现
3. 讨论部分理论对话
4. 修改和润色

**输出**: 论文章节

### 5.6 反思改进

**步骤**:
1. 收集用户反馈
2. 记录教训和经验
3. 提炼最佳实践
4. 更新方法论配置

**输出**: 反思报告

## 六、质量标准

### 6.1 信度（Reliability）

**检验方法**:
- 内部一致性检验
- 评分者间信度
- 重测信度
- 分析过程可重复

**标准**:
- Cronbach's α > 0.7
- Kappa > 0.7
- 分析代码可复现

### 6.2 效度（Validity）

**检验方法**:
- 内容效度（专家评审）
- 建构效度（因子分析）
- 效标效度（相关分析）
- 三角验证

**标准**:
- 专家评审通过
- 因子载荷 > 0.5
- 多方法验证一致

### 6.3 伦理标准

**要求**:
- 知情同意
- 隐私保护
- 数据匿名化
- 风险最小化

**审查**:
- 伦理委员会审批
- 数据安全审查
- 用户隐私检查

## 七、常见错误与防止

### 7.1 方法误用

**表现**:
- 方法与研究问题不匹配
- 简化方法论核心要求
- 忽视方法前提假设

**防止措施**:
- 严格方法匹配流程
- 明确方法适用条件
- 设置前提假设检查

### 7.2 过度推断

**表现**:
- 从相关推断因果
- 从小样本推断总体
- 从特定情境推断普遍

**防止措施**:
- 明确推断范围
- 说明局限性
- 设置推断检查点

### 7.3 技术依赖

**表现**:
- 过度依赖 AI 工具
- 忽视人类判断
- 技术黑箱化

**防止措施**:
- 强调 AI 是工具
- 保持人类判断
- 提高技术透明度
