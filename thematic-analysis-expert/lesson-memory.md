## 成功模式

### 模式1：六步骤严格执行

**场景**: 任何主题分析

**方法**:
- 按顺序完成六个步骤
- 每步都有明确的产出
- 不跳过任何步骤

**效果**: 确保分析的系统性和严谨性。

**适用条件**: 所有主题分析研究。

---

### 模式2：主题审查迭代

**场景**: 主题发展阶段

**方法**:
1. 形成候选主题
2. 检验主题内部一致性
3. 检验主题间差异性
4. 合并、拆分或删除主题
5. 重复检验直到满意

**效果**: 提升主题质量和边界清晰度。

**适用条件**: 主题数量较多或边界模糊时。

---

### 模式3：负面案例搜索

**场景**: 主题审查阶段

**方法**:
- 主动寻找与主题不符的数据
- 分析不符合的原因
- 调整主题定义或边界

**效果**: 增强分析的严谨性和可信度。

**适用条件**: 所有主题分析研究。

## 失败教训

### 教训1：跳过编码直接预设主题

**场景**: 时间压力下的分析

**错误做法**:
- 根据研究问题预设主题
- 直接从数据中寻找支持
- 跳过系统编码过程

**后果**:
- 主题无法反映数据全貌
- 忽视意外发现
- 违背方法论原则

**正确做法**:
- 必须系统编码
- 让主题从编码中涌现
- 保持开放性

**教训标签**: #预设主题 #跳过编码

---

### 教训2：用引文代替分析

**场景**: 报告撰写阶段

**错误做法**:
- 大量堆砌引文
- 缺乏阐释和分析
- 引文本身成为"发现"

**后果**:
- 分析缺乏深度
- 没有理论贡献
- 质量评估不达标

**正确做法**:
- 引文是证据
- 需要阐释引文的意义
- 分析要有理论视角

**教训标签**: #浅层分析 #堆砌引文

---

### 教训3：主题重叠边界模糊

**场景**: 多主题研究

**错误做法**:
- 主题之间存在大量重叠
- 同一内容可以归入多个主题
- 主题边界不清晰

**后果**:
- 分析冗余
- 读者困惑
- 质量不达标

**正确做法**:
- 明确每个主题的独特贡献
- 检验主题间差异
- 必要时合并或拆分

**教训标签**: #主题重叠 #边界模糊

## 质量标准基准

### 主题质量标准

| 评分 | 标准 |
|------|------|
| 优秀 | 内部一致、边界清晰、覆盖充分、分析深入 |
| 良好 | 基本一致、边界较清、覆盖较全 |
| 及格 | 有一定一致性、边界基本清晰 |
| 不及格 | 内部混乱、边界模糊、覆盖不足 |

### 编码质量标准

| 评分 | 标准 |
|------|------|
| 优秀 | 覆盖全部数据、有意义、足够精细 |
| 良好 | 覆盖大部分数据、有意义 |
| 及格 | 有基本覆盖 |
| 不及格 | 覆盖不足或编码无意义 |
