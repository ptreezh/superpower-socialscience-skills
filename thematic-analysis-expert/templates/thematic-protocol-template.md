## 三、分析过程

### 3.1 Braun & Clarke 六步骤规划

#### Step 1: 熟悉数据

**计划活动**:
- [ ] 反复阅读转录文本
- [ ] 记录初步想法和观察
- [ ] 标记潜在关注点
- [ ] 建立数据整体理解

**产出**:
- 初步观察笔记
- 数据概览文档

#### Step 2: 生成初始代码

**编码策略**:
- 编码单位: [句子/段落/意义单元]
- 编码方式: [手工/软件辅助]
- 编码粒度: [描述/阐释]

**计划活动**:
- [ ] 对全部数据进行编码
- [ ] 确保编码覆盖数据
- [ ] 记录编码决策

**产出**:
- 编码列表
- 编码定义表

#### Step 3: 搜索主题

**主题发展策略**:
- 代码聚类方法
- 主题候选标准

**计划活动**:
- [ ] 将代码聚类成潜在主题
- [ ] 检查每个主题的支持数据
- [ ] 形成主题地图

**产出**:
- 候选主题列表
- 主题-代码关联图

#### Step 4: 审查主题

**审查标准**:
- 内部一致性
- 外部异质性
- 数据支持充分性

**计划活动**:
- [ ] 检验主题内部一致性
- [ ] 检验主题间差异
- [ ] 调整主题(合并/拆分/删除)
- [ ] 对照原始数据验证

**产出**:
- 修订后的主题列表
- 主题审查记录

#### Step 5: 定义和命名主题

**命名原则**:
- 准确反映主题内容
- 简洁易懂
- 有理论意义(如适用)

**计划活动**:
- [ ] 为每个主题撰写定义
- [ ] 确定主题命名
- [ ] 识别子主题
- [ ] 选择代表性引文

**产出**:
- 主题定义表
- 最终主题结构

#### Step 6: 撰写报告

**报告结构**:
1. 引言
2. 方法
3. 发现
4. 讨论

**计划活动**:
- [ ] 撰写分析发现
- [ ] 选择支持性引文
- [ ] 讨论理论意义
- [ ] 说明研究局限

**产出**:
- 分析报告

## 五、分析工具

### 5.1 编码工具

| 工具 | 用途 |
|------|------|
| code_extractor.py | 代码提取 |
| theme_cluster.py | 主题聚类 |
| saturation_checker.py | 饱和度检查 |
| quality_assessor.py | 质量评估 |

### 5.2 使用方法

```bash
# 代码提取
python tools/code_extractor.py --input data/transcripts/ --output codes/

# 主题聚类
python tools/theme_cluster.py --codes codes/ --output themes/

# 饱和度检查
python tools/saturation_checker.py --data data/ --codes codes/

# 质量评估
python tools/quality_assessor.py --themes themes/ --output report/
```
