# Coze（扣子）技能导入教程

**最后更新**: 2026-03-22  
**难度**: 入门  
**时间**: 15 分钟

---

## 📋 什么是 Coze

**Coze**（中文名：扣子）是字节跳动推出的一站式 AI Bot 开发平台，无需编程即可创建 AI 助手。

**核心特点**:
- ✅ 无需部署，开箱即用
- ✅ 可视化配置
- ✅ 丰富的插件生态
- ✅ 支持知识库

**适用场景**:
- 快速创建 AI 助手
- 团队协作
- 企业应用
- 个人效率工具

---

## 🚀 快速开始（15 分钟上手）

### 步骤 1: 注册 Coze 账号

1. 访问 https://www.coze.cn/
2. 点击"登录/注册"
3. 使用抖音/头条/邮箱注册
4. 完成账号验证

---

### 步骤 2: 创建新 Bot

1. 点击"创建 Bot"
2. 填写 Bot 信息：
   - **名称**: 扎根理论专家
   - **描述**: 专业的扎根理论分析助手
   - **图标**: 选择或上传
3. 点击"创建"

---

### 步骤 3: 配置人设与回复逻辑

**人设配置**（对应 SKILL.md）:

```markdown
# 角色
你是扎根理论分析专家，专注于提供严谨、规范、可重复的质性数据分析。

# 使命
让扎根理论研究方法更加规范、严谨、易于应用。

# 核心能力
1. 开放编码：从数据中提取初始概念
2. 轴心编码：建立概念间关系
3. 选择式编码：整合为核心范畴
4. 理论饱和度检验：验证理论完备性

# 工作流程
1. 数据准备：验证数据质量
2. 开放编码：提取初始概念
3. 轴心编码：建立范畴关系
4. 选择式编码：整合核心范畴
5. 饱和度检验：验证理论完备性

# 输出规范
1. 所有编码必须有原始引文支撑
2. 明确说明分析步骤
3. 提供下一步建议
4. 说明分析局限性
```

**回复逻辑配置**:

```markdown
# 开场白
你好！我是扎根理论分析专家，可以帮助你进行规范的质性数据分析。

请提供你的访谈数据或研究问题，我会按照扎根理论的标准流程进行分析。

# 预置问题
1. 如何进行开放编码？
2. 什么是理论饱和度？
3. 如何写备忘录？
4. 扎根理论有哪些编码类型？

# 回复规则
1. 先理解用户需求
2. 按照扎根理论流程分析
3. 提供具体的编码结果
4. 给出下一步建议
```

---

### 步骤 4: 添加插件（对应 tools/）

**添加数据分析插件**:

1. 点击"插件"标签
2. 搜索"数据分析"
3. 添加以下插件：
   - **文本分析工具**: 用于文本处理
   - **数据可视化工具**: 用于图表生成
   - **文件处理工具**: 用于文件上传下载

**添加学术工具插件**:

1. 搜索"学术"
2. 添加以下插件：
   - **文献检索**: 用于查找相关文献
   - **引用生成**: 用于生成参考文献
   - **格式检查**: 用于检查学术规范

---

### 步骤 5: 上传知识库（对应 templates/）

**准备知识库文件**:

1. 整理扎根理论资料：
   - 方法论说明文档
   - 编码示例
   - 常见问题解答
   - 参考文献列表

2. 文件格式：
   - PDF 文档
   - Word 文档
   - Markdown 文件
   - TXT 文本

**上传知识库**:

1. 点击"知识库"标签
2. 点击"上传文件"
3. 选择文件上传
4. 等待自动处理
5. 确认处理完成

---

### 步骤 6: 配置工作流（可选）

**创建工作流**:

1. 点击"工作流"标签
2. 点击"创建工作流"
3. 配置工作流节点：

```
开始 → 文本预处理 → 开放编码 → 轴心编码 → 选择式编码 → 输出生成 → 结束
```

4. 配置每个节点：
   - **文本预处理**: 清洗数据
   - **开放编码**: 提取概念
   - **轴心编码**: 建立关系
   - **选择式编码**: 整合范畴
   - **输出生成**: 生成报告

5. 测试工作流
6. 保存工作流

---

### 步骤 7: 发布 Bot

**发布前检查**:

- [ ] 人设配置完整
- [ ] 插件添加完成
- [ ] 知识库上传完成
- [ ] 工作流配置完成（如有）
- [ ] 测试对话正常

**发布步骤**:

1. 点击右上角"发布"
2. 选择发布渠道：
   - ✅ Coze 商店
   - ✅ 微信服务号
   - ✅ 飞书
   - ✅ API
3. 确认发布
4. 等待审核（通常 1-2 小时）

## 📚 使用示例

### 示例 1: 开放编码分析

**用户输入**:
```
请对以下访谈数据进行开放编码：

"我觉得工作压力很大，每天都要加班，
但是没有明确的晋升通道，感觉很迷茫。"
```

**Bot 回复**:
```
## 开放编码结果

### 初始概念识别

1. **工作压力**
   - 原始引文："我觉得工作压力很大"
   - 范畴：工作环境感知

2. **加班现象**
   - 原始引文："每天都要加班"
   - 范畴：工作强度

3. **晋升困境**
   - 原始引文："没有明确的晋升通道"
   - 范畴：职业发展障碍

4. **迷茫情绪**
   - 原始引文："感觉很迷茫"
   - 范畴：心理状态

### 编码备忘录

本次编码发现了 4 个初始概念，涉及工作环境、
工作强度、职业发展和心理状态四个范畴。

### 下一步建议

1. 继续收集更多访谈数据
2. 进行持续比较分析
3. 准备进行轴心编码
```

---

### 示例 2: 理论饱和度检验

**用户输入**:
```
我已经编码了 20 份访谈，如何判断是否达到理论饱和？
```

**Bot 回复**:
```
## 理论饱和度检验指南

### 判断标准

1. **无新范畴出现**
   - 新数据不再产生新的初始概念
   - 现有范畴已经完备

2. **范畴关系稳定**
   - 范畴间的关系模式稳定
   - 无需调整关系结构

3. **变异充分**
   - 范畴内部变异充分
   - 覆盖各种情况

### 检验方法

1. **预留检验法**
   - 预留 3-5 份数据
   - 用预留数据检验
   - 无新范畴即饱和

2. **分组检验法**
   - 将数据分组
   - 分别编码
   - 比较结果

3. **时间序列法**
   - 按时间顺序编码
   - 观察新范畴出现频率
   - 频率趋近零即饱和

### 你的情况

已编码 20 份访谈，建议：

1. 预留 3-5 份进行检验
2. 检查是否有新范畴出现
3. 如无新范畴，可判定饱和
4. 如有新范畴，继续编码
```

## 🔧 高级配置

### 1. 多 Bot 协作

**创建专家网络**:

```
Bot 1: 扎根理论专家（主分析）
Bot 2: 文献检索助手（查文献）
Bot 3: 学术写作助手（写报告）
```

**配置协作**:

1. 在每个 Bot 中添加其他 Bot 为"协作 Bot"
2. 配置协作规则
3. 测试协作流程

---

### 2. API 调用

**获取 API Key**:

1. 进入 Bot 管理页面
2. 点击"API 管理"
3. 创建 API Key
4. 复制保存

**API 调用示例**:

```python
import requests

url = "https://api.coze.cn/v1/chat"
headers = {
    "Authorization": "Bearer YOUR_API_KEY"
}
data = {
    "bot_id": "YOUR_BOT_ID",
    "message": "请分析以下访谈数据..."
}

response = requests.post(url, headers=headers, json=data)
result = response.json()
```

### 3. 数据导出

**导出对话记录**:

5. 下载 CSV/Excel 文件

1. 进入知识库页面

## ❓ 常见问题

### Q1: Bot 回复不准确

**问题**: Bot 经常答非所问

**解决方案**:
```
1. 优化人设配置
   - 更详细的角色描述
   - 更明确的工作流程
   - 更具体的输出规范

2. 补充知识库
   - 添加更多方法论资料
   - 添加编码示例
   - 添加常见问题

3. 调整插件配置
   - 添加相关插件
   - 调整插件优先级
```

### Q2: 知识库处理失败

**问题**: 上传文件后处理失败

**解决方案**:
```
1. 检查文件格式
   - 支持：PDF, Word, Markdown, TXT
   - 不支持：图片，音频，视频

2. 检查文件大小
   - 单文件限制：50MB
   - 超大文件请分割

3. 重新上传
   - 删除失败文件
   - 重新上传
   - 等待处理完成
```

### Q3: Bot 审核不通过

**问题**: 发布后审核被拒

**解决方案**:
```
1. 查看审核意见
   - 进入 Bot 管理页面
   - 查看审核反馈
   - 了解具体问题

2. 修改配置
   - 修改违规内容
   - 调整人设描述
   - 补充必要信息

3. 重新提交
   - 确认修改完成
   - 重新提交审核
   - 等待审核结果
```

### Q4: 如何从本地技能迁移

**问题**: 已有 OpenClaw 技能，如何迁移到 Coze

**解决方案**:
```
步骤 1: 提取 SKILL.md 内容
   - 复制角色定义
   - 复制工作流程
   - 复制输出规范

步骤 2: 转换为人设配置
   - 粘贴到人设配置
   - 调整格式
   - 补充 Coze 特定配置

步骤 3: 配置插件
   - 查找对应插件
   - 添加插件
   - 配置插件参数

步骤 4: 上传知识库
   - 整理 templates/文件
   - 上传到知识库
   - 等待处理完成

步骤 5: 测试验证
   - 测试对话
   - 检查输出
   - 调整优化
```
