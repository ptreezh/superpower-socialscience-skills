## 描述

**价值主张设计专家** - 支持**复杂任务分解**和**长时任务执行**的价值创新方法论技能。

基于Strategyzer的价值主张设计方法论，提供系统化的产品市场匹配分析框架。价值主张设计是一种以客户为中心的创新方法，通过深入理解客户需求，设计能够创造客户价值的产品和服务。

## 6大绝对禁止原则

1. **禁止自说自话** - 必须从客户视角出发，而非从产品视角出发
2. **禁止忽视负面情绪** - 痛点和挫折是创新的关键来源
3. **禁止泛泛而谈的客户** - 必须具体到可识别的客户细分
4. **禁止假设客户需求** - 必须通过真实客户访谈验证
5. **禁止忽视匹配验证** - 设计后必须验证价值匹配程度
6. **禁止一成不变** - 价值主张需要根据市场反馈持续迭代

## CRCT思维链

- **C (Constant Comparison)**: 持续比较价值主张与客户期望的匹配程度
- **R (Record)**: 记录客户访谈和验证实验的结果
- **C (Chain)**: 建立从客户洞察到产品设计的完整链条
- **T (Trace)**: 追踪每个价值假设的验证过程

### 客户画像三要素

```
客户画像 (Customer Profile):
├── 客户工作 (Customer Jobs)
│   ├── 功能性工作: 客户需要完成的任务
│   ├── 社会性工作: 客户希望在社会中的形象
│   └── 情感性工作: 客户的情感需求
│
├── 痛点 (Pains)
│   ├── 障碍: 阻碍完成工作的因素
│   ├── 风险: 客户担心的负面结果
│   └── 挫折: 完成工作过程中的烦恼
│
└── 收益 (Gains)
    ├── 必需收益: 客户期望的基本价值
    ├── 期望收益: 客户希望获得的额外价值
    ├── 意外收益: 超出客户期望的价值
    └── 潜在收益: 客户可能需要的未来价值
```

### 价值主张三要素

```
价值主张 (Value Proposition):
├── 产品服务 (Products & Services)
│   ├── 核心产品: 主要产品或服务
│   ├── 支持服务: 辅助性服务
│   └── 增值服务: 额外价值服务
│
├── 痛点解决方案 (Pain Relievers)
│   ├── 消除障碍: 帮助客户克服障碍
│   ├── 降低风险: 减少客户的担忧
│   └── 缓解挫折: 减少客户的烦恼
│
└── 收益创造方案 (Gain Creators)
    ├── 创造必需收益: 满足基本期望
    ├── 创造期望收益: 提供额外价值
    └── 创造意外收益: 超出期望的价值
```

### 价值匹配类型

```
价值匹配类型:
├── 问题-方案匹配 (Problem-Solution Fit)
│   - 验证: 我们是否找到了值得解决的问题？
│   - 阶段: 客户发现
│   - 方法: 客户访谈、问题验证
│
├── 产品-市场匹配 (Product-Market Fit)
│   - 验证: 我们是否构建了客户想要的产品？
│   - 阶段: 客户验证
│   - 方法: MVP测试、数据验证
│
└── 规模化匹配 (Scale Fit)
    - 验证: 我们是否可以规模化增长？
    - 阶段: 客户创造
    - 方法: 增长实验、渠道优化
```

## 分析阶段

### 阶段1: 客户细分识别

**目标**: 识别和定义目标客户细分

**输入**:
- 市场研究
- 竞争分析
- 企业资源

**活动**:
1. 市场细分分析
2. 客户群体识别
3. 细分吸引力评估
4. 目标客户选择

**输出**:
- 客户细分列表
- 目标客户定义
- 细分优先级

### 阶段2: 客户画像构建

**目标**: 深入理解目标客户

**输入**:
- 目标客户定义
- 客户访谈
- 行为数据

**活动**:
1. 客户工作分析
2. 痛点识别
3. 收益期望分析
4. 画像验证

**输出**:
- 客户工作清单
- 痛点清单
- 收益清单
- 客户画像文档

### 阶段3: 价值主张设计

**目标**: 设计匹配客户需求的价值主张

**输入**:
- 客户画像
- 企业能力
- 竞争格局

**活动**:
1. 产品服务设计
2. 痛点解决方案设计
3. 收益创造方案设计
4. 差异化定位

**输出**:
- 产品服务清单
- 解决方案清单
- 收益创造清单
- 价值主张文档

### 阶段4: 价值匹配验证

**目标**: 验证价值主张与客户需求的匹配

**输入**:
- 客户画像
- 价值主张

**活动**:
1. 匹配度分析
2. 差距识别
3. 优先级排序
4. 验证实验设计

**输出**:
- 匹配度评估
- 差距分析
- 验证计划

### 阶段5: 迭代优化

**目标**: 基于验证结果优化价值主张

**输入**:
- 验证结果
- 市场反馈

**活动**:
1. 结果分析
2. 问题识别
3. 方案调整
4. 新假设生成

**输出**:
- 优化方案
- 新的价值主张
- 迭代计划
